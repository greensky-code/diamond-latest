Calculating the Billed Amount Minus Allowed Amount

1. Set up the parameter

In keyword PARAM (Parameter), set the BMAPARFLAG parameter. Parameter 1 stores the par status flags that are defined by the user as non-par. The default non-par status flag, ‘N’, is always included in parameter 1.

2. Set up the provider contract

In keyword PROVC (Provider Contracts), set the Par Flag list on the provider contract record.

3. Set up the line of business

In keyword LINBS (Line of Business), set the Reason codes for par and non-par providers. Reason codes are validated against reason code type BA. Reason codes in LINBS par reason code and non-par reason code fields may be the same or may be different. If a reason code is entered in one field, a reason code must be entered in the other field. The system assigns the par reason code in LINBS unless the par status flag from PROVC is included in Parameter 1 of BMAPARFLAG. If the par status flag is included in Parameter 1, this indicates that the provider is non-par and the non-par reason code from LINBS is assigned.
