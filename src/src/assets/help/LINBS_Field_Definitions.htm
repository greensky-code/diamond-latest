Allowed > Billed

O

If selected, allows the allowed amount on a professional services claim detail line to be greater than the billed amount.

Selected from predefined list

Overrides the logic which otherwise reduces the allowed amount to the billed amount when the calculated allowed is greater.

Not required for institutional claims, where allowed amounts can be greater than billed amounts (to account for per diem and other hospital pricing).

Apply Unclean Ind.

O

Select if you want DIAMOND to evaluate claim detail lines (for application of the Unclean Indicator) separately, or as a group.

Selected from predefined list

Valid values are:

  • 1 Line Unclean = All Lines Unclean

  • Indiv. Lines Eval. Separately

For more information, see Claims Discount and Interest Calculations, How DIAMOND Thinks.

Auth Level

O

Value entered in this field is used when an authorization waiver applies but no authorization is linked to the claim.

Selected from predefined list

When this field is not used, the authorization level is carried over to the claim from the linked authorization.

Auth/Claim Match

O

Invokes automatic authorization/claim matching for members in this line of business.

Selected from check box

If selected, then automatic authorization/claim matching will occur in keywords DNCLM, INCLM, and PSCLM.

If cleared, then claims processors can manually select the available authorizations for a member in keywords DNCLM, INCLM, and PSCLM. After entering the member ID, the Lookup dialog box will display a list of available authorizations.

Calc. Disc./
Intr.&Pnlty

O

Indicates if claims will be evaluated for discount and/or interest payments.

Selected from predefined list

Valid values are:

  • discount only

  • interest & penalty only

  • no (default)

  • disc./interest & penalty

DIAMOND cannot apply a discount calculation and an interest calculation to the same claim line. If value B is selected, DIAMOND evaluates discount calculation rules first and if a discount calculation is applied, DIAMOND stops searching. However, if no discount calculation is applied, or if the discount amount is 0, then DIAMOND searches the interest calculation.

Selected with check box

Unavailable if MEMSITEVAL Parameter 1 = No.

For more information, see Site Validation, How DIAMOND Thinks.

Covering Prov

O

Invokes authorization/claim matching for "covering" provider groups.

Selected from check box

You must select the Auth/Claim Match check box and enter values in the Days Before and Days After boxes before you select this check box. When selected, DIAMOND performs a predefined auth/claim match for covering providers (professional services only).

Note: The VENDRFIRST parameter logic cannot be used if you select this check box.

CovProv Days After

C

"Added" to the Authorization request date to define an acceptable range for a claim to be linked to an authorization when compared to the primary or admit date on that claim.

Manually entered

Entry in this field is also used by keyword ARULE (Authorization Claim Link Rule).

CovProv Days Before

C

"Subtracted" from the Authorization request date to define an acceptable range for a claim to be linked to an authorization when compared to the primary or admit date on that claim.

Manually entered

Entry in this field is also used by keyword ARULE.

Default PCP

O

PCP ID you can use to satisfy a line of business PCP requirement when adding a member eligibility record, if the PCP is unknown or has not yet been assigned.

Selected from predefined list

Designed to enable you to add a member eligibility record to DIAMOND when no PCP assignment exists and the memberâs line of business requires a PCP. You must manually add this Default PCP to the eligibility record prior to submission. Typically followed up by PCP Auto Assignment (or manual PCP assignment) to replace the Default PCP. For more information, see PCP Auto Assignment, How DIAMOND Thinks.

F5 lookup available.

You must set up valid provider contract for this line of business for the Default PCP ID selected.

When selected, forces dependent eligibility to be consistent with that of the subscriber with regard to dates, group, and plan.

ID Card Reinst Days

O

Minimum number of days before reinstatement of eligibility that must elapse before an ID card is generated.

Manually entered

Applies only if ID Card order for reinstatements is enabled in system parameter IDCARDORDER.

DIAMOND calculates elapsed days as number of days between the termination date of the most recent prior eligibility record and the effective date of the current record.

INCLM Order

R

The order in which windows are displayed during INCLM processing.

Selected from predefined list

Select from the following:

  • D - Claim detail window is displayed after data is entered in the header; UB92 window must be specifically requested by the operator

  • U - After data is entered on the header, the UB92 window is displayed; after UB92 entry, the detail window is displayed.

Ind PCP/LINBS Threshold

O

Percentage of applicable enrollment limits that can be assigned via PCP Auto Assignment, unless a PCPAA override applies

Manually entered

You enter a decimal number that DIAMOND converts to a percentage.

For example:

1.00 = 100%

.85 = 85%

Applies to PCP Auto Assignment only.

You define the enrollment limits included in system parameter ENROLLLIMITS. See Parameters ENROLLLIMITS user documentation for more information.

Use Reset Date

O

Enables you to use the Reset Date (if it has been entered on the claim detail line) as the Calculation Date for interest payments.

Manually entered

Valid values are:

  • Checked = yes

  • Blank = no (default)

The default for interest payments is to use the claimâs Received Date as the calculation date. Checking this field enables you to override that default and use a date that you manually enter into the Rest Date field on the claim detail line.
