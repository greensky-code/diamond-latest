Authorization/Claim Detail Linking and Pricing - Overview

You can use authorization/claim detail linking and pricing to perform the following actions, based on rules you define by line of business/authorization type/effective date combination:

  • Match submitted claim detail procedure/revenue codes to ranges of authorized codes on a matching authorization’s procedure detail records

  • Price matching claim detail lines using:

  • The Auth Price from the matching authorization procedure detail record

  • The Auth Procedure from the matching authorization procedure detail record

  • Standard DIAMOND provider pricing logic

  • Hold or deny matching claim detail lines based on the application of an authorization/claim detail linking and pricing rule

  • Assign specific allowed, hold, and deny reason codes to matching claim detail lines when applying an authorization/claim detail linking and pricing action

DIAMOND performs Authorization/Claim Detail Linking and Pricing for online and DIAMOND Data Service (DDS) API transactions.

Setup

You set up rules to perform this logic using the following DIAMOND keywords:

  • PRCRA (Authorization Procedure Ranges)

Use keyword PRCRA to define sets of procedure code ranges to be used during authorization/claim detail linking. After you define a PRCRA range set, you can use it as a procedure code match criterion when you build CDAPL (Claim Detail Authorization Procedure Link) link rules.

The PRCRA range set is used to determine if submitted claim procedure codes that do not exactly match the authorized procedure code should be considered authorized. For more information, see PRCRA, Authorization Procedure Ranges.

  • CDAPR (Claims Detail Authorization Procedure Rules)

Use keyword CDAPR to define authorization procedure detail match criteria rules based on the authorized procedure amount, authorized procedure status, and whether the submitted procedure code and authorized procedure code are an exact match, in range, or out of range.

After you define CDAPR rules, you can use these rules as authorization detail match criteria when building CDAPL (Claim Detail Authorization Procedure Link) rules. For more information, see CDAPR, Claims Detail Authorization Procedure Rules.

  • CDAPL (Claims Detail Authorization Procedure Link)

Use keyword CDAPL to define rules for performing authorization/claim detail linking and pricing. You define CDAPL rules for a specific line of business, authorization type and effective date combination.

When you enter a claim that has a matching authorization and also matches a CDAPL rule line of business/authorization type/effective date combination, DIAMOND searches for the matching authorization procedure details. DIAMOND then applies the CDAPL rules to the claim detail line and matching authorization to determine what action to take on each claim detail line. For more information, see CDAPL, Claims Detail Authorization Procedure Link.

Enabling Authorization/Claim Matching

A claim must match an authorization at the header level in order for authorization/claim detail linking and pricing logic to be performed. Thus you must enable authorization/claim matching in order to use this logic. To learn how to enable and use DIAMOND authorization/claim matching logic, see Authorization Claim Links, How DIAMOND Thinks.

Additional Setup Information

For more information on setting up this logic, see Setting Up Authorization/Claim Detail Linking and Pricing.

Pay Actions

CDAPL Pay Actions specify how to price a claim detail line that satisfies the CDAPL criteria and also whether the line is held or denied. Available Pay Actions include:

  • 1 - Price with Standard Provider Pricing.

  • 2 - Price with Authorized Price

  • 3 - Price with Authorized Procedure

  • 4 - Price and Deny with Standard Provider

  • 5 - Price and Deny with Authorized Price

  • 6 - Price and Deny with Authorized Procedure

  • 7 - Price and Hold with Standard Provider Pricing

  • 8 - Price and Hold with Authorized Price

  • 9 - Price and Hold with Authorized Procedure

For more information on Authorization/Claim Detail Linking and Pricing processing logic, see Authorization/Claim Detail Linking and Pricing, How DIAMOND Thinks.

APC Pricing and Authorization/Claim Detail Linking and Pricing

Warning!   Ambulatory Payment Classification (APC) pricing can override other pricing on claim detail lines. Do not use both APC pricing and authorization/claim detail linking and pricing.

Third Party Claim Audit Tools and Authorization/Claim Detail Linking and Pricing

Warning!   Third party claim audit tools override authorization/claim detail linking and pricing. Do not use both third party claim audit tools and authorization/claim detail linking and pricing. For more information on third party claim audit tools that interface with DIAMOND, see Claim Auditing - How DIAMOND Thinks.
