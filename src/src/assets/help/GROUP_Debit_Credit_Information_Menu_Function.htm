Debit/Credit Information Menu Command

Field Name

Req

Description

Source

Comments

Account No

R

Account number for this group.

Manually entered

 

Type

R

Account information type.

Selected from predefined list

Types are maintained in keyword CODES (System Codes Maintenance) with type ACCTINFTYPE.

Credit card accounts require a type code, which begins with CR.

Use

R

Indicates use of account.

Selected from predefined list

Valid values are

  • Debit

  • Credit (future functionality)

  • Both

A customer (group or individual) may have one Debit and one Credit account with the same effective date, or one Both account.

If customer has a Debit or Both account, then the new balance is written to the pmb_eft_interface table.

Eff Date

R

Date the account is effective.

Manually entered

 

Exp Date

C

Date a credit account expires.

Manually entered

This is a required field if the Type begins CR; it is not used if Type does not begin with CR. Format is MM/YY.

Status

R

Status of account.

Selected from predefined list

Valid values are:

  • Active

  • Inactive

  • Prenote: Used for testing EFT interfaces with banks.

Routing Number

C

 

Manually entered

Optional field if Type does not begin with CR; not used if Type begins with CR.
