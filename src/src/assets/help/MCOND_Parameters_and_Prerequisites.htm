MCOND Parameters and Prerequisites

Parameter

Description

Comments

DATEFORMAT

Determines the format for dates for all processing on the DIAMOND server. Client processing date format is controlled by the Windows Control Panel.

Relative positions of month, day and year must be the same for both client and server formats.

You choose one of the following formats:

MM/DD/YYYY

DD/MM/YYYY

YYYY/MM/DD

DIDSEARCH

Use to enable dual coverage DIAMOND ID functionality.  

Once you enable DIAMOND ID functionality in DIDSEARCH, a DIAMOND ID must be entered in the MEMBR screen for each new member.

For more information, see Dual Coverage - Overview.

PREEXISTCOND

When a claim or claim line is held by pre-existing condition logic, the DIAMOND application displays the warning message “Possible pre-existing condition, line pended” when this parameter is activated.
