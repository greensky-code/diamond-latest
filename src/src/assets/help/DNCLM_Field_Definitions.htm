Auth No

O

The authorization number linked to this claim.

Manually entered or system generated

You can type an authorization number or after you enter the member ID, the Lookup dialog box will display a list of authorizations for the member you can select.

If the member’s line of business is set up for automatic authorization/claim matching, then one of the following occurs when you first display the detail window:

  • If there is one matched authorization for the member, then the authorization number automatically displays.

- or -

  • If there are multiple matched authorizations for the member, then the Select Matched Authorizations dialog box displays. It displays a list of authorizations for the member that you can select.

A line of business is set up for automatic authorization claim matching when the Auth/Claim Match check box is selected in keyword LINBS (Line of Business).

COB Pat Liab

D

The reduction in patient liability resulting from coordination of benefits (COB), if any.

System generated

DIAMOND may truncate the display of the currency amount.

Calculated as the Total Patient Liabilities amount before COB minus the Total Patient Liabilities amount after COB.

Only calculated if all the following apply:

  • COB Calculation method Non-Duplication of Benefits or Integration of Benefits was applied during automatic adjudication

  • Total Pt Liabilities after COB is less than Total Pt Liabilities before COB

  • BENEF detail stipulates that benefit accumulators are not adjusted for other carrier paid amounts

Date

R

First date of service pertaining to this detail line.

Manually entered

During adjudication, DIAMOND validates that this (from) Date of service falls within the date range of one of the member’s eligibility periods. If it does not, DIAMOND denies the line, sets the Claim Status list to D – Denied, sets the Allowed Amt box and Total Not Cov box to 0.00, and assigns the Reason code associated with Reason code type ME as the Not Cov Rsn code. Reason code ME (only one is allowed) must exist in keyword REASN.

DIAMOND ID

Dx 4

O

Member’s diagnosis codes associated with this claim.

Manually entered

You can type up to 30 alphanumeric characters. Unless the insertion point is in this box, then only 7 characters display.

The primary diagnosis code must go in the Dx 1 box. Entered codes are validated against the member’s gender/age and a warning displays if the code is flagged for case management. DIAMOND displays an error message if codes do not pass validation.

Additional diagnosis codes can be entered by clicking Other Claim Info on the Topic menu.

Diagnosis codes are maintained in keyword DIAGN (Diagnosis Code).

Dx Ptr

O

Indicates which Dx box on the claim header window or Other Claim Info window contains the diagnosis code that most pertains to this claim line.

Manually entered

Valid values of 1, 2, 3, 4, 5, 6, 7, and 8 can be typed in these eight boxes. For example, type 1 to point to the Dx 1 box on the Claim header window.

During claims adjudication, DIAMOND uses the pointer in the first Dx Ptr box.

If only one diagnosis code was entered in the Dx 1 box on the claim header window, then 1 displays in the first Dx Ptr box.
