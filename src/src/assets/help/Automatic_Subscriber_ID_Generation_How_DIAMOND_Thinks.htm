Automatic Subscriber ID Generation, How DIAMOND Thinks

Characteristics of Automatically Generated Subscriber IDs

Online Automatic Subscriber ID Generation

API Automatic Subscriber ID Generation

You can have DIAMOND automatically generate a Subscriber ID when you add a new subscriber. You can specify that this occur:

  • When you are adding a subscriber online (at your option)

  • When you are adding a subscriber via API using the ANSI 834 Benefits and Member Maintenance Transaction

  • Both online and API

You enable Automatic Subscriber ID Generation using the system parameter MEMBSUBAUTGEN. See Enabling Automatic Subscriber ID Generation for more information.

Characteristics of Automatically Generated Subscriber IDs

Subscriber IDs automatically generated by DIAMOND have the following characteristics:

  • Sequentially generated, incrementing by one

  • Not a duplicate of an existing Subscriber ID

  • Numeric

  • Left justified with no leading zeroes

  • Maximum length of 12

Online Automatic Subscriber ID Generation

If you have enabled online Automatic Subscriber ID Generation in MEMBSUBAUTGEN, DIAMOND performs as follows.

When you tab out of the Subscriber ID field in the keyword MEMBR without entering a Subscriber ID, DIAMOND prompts you to automatically generate the Subscriber ID. If you click Yes, DIAMOND generates the next ID in the sequential series. If you click No, you remain in the Subscriber ID field.

API Automatic Subscriber ID Generation

If you have enabled API Automatic Subscriber ID Generation in MEMBSUBAUTGEN, DIAMOND will perform Automatic Subscriber ID Generation during API processing of ANSI 834 Benefits and Member Maintenance transactions. See the ANSI 834 Benefits and Member Maintenance Transaction API Guide for more information.
