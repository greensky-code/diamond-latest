What is Covering Provider?

You can use the Covering Provider functionality for a physician that arranges to have another physician cover his or her patients. Typically, specialists cover for other specialists and primary care providers (PCPs) cover for other PCPs.

Setting Up Covering Provider

To set up Covering Provider, you must do the following:

  1. Check the VENDORFIRST parameter in keyword PARAM (Parameters). The Parameter 1 box must be set N in order to use Covering Provider. For more information on the VENDORFIRST parameter, see PARAM Record Definitions.

  2. In keyword LINBS (Line of Business), the following fields must be set up for the line of business:

  • The PCP Required list must be set to R – Required.

  • The Auth/Claim Match check box must be selected.

  • The Days Before and Days After boxes must have dates that are applicable for your business needs. These dates are evaluated against the request date on the authorization.

  • The Covering Prov check box must be selected. If you cannot select this check box, then the VENDORFIRST parameter in keyword PARAM has the Parameter 1 box set to Y, and it has to be set to N in order for you to use Covering Provider.

  1. In keyword CVPVC (Covering Provider Group Codes), add a name and ID for a covering provider group.

  2. In keyword CVPVG (Covering Provider Groups), add providers to your provider groups. Also select a reimbursement method.

  3. In keyword PROVC (Provider Contracts), display the provider and in the Cov Group box, type ID of the covering provider group. F5 lookup is available.

Using Covering Provider

You can change the original provider to a covering provider on the detail window in keywords PSCLM (Professional Services Claims) and DNCLM (Dental Services Claims). Display the claim line on the detail window. On the Special menu, click Covering Provider Info. Click Change to change to the provider to another provider. Then, price or adjudicate the claim again. The claim will re-price according to the covering provider method that was selected in keyword CVPVG.

Covering Provider Logic

Covering Provider uses the setting in the Parameter1 box of the parameter record DUPAUTHRUL2 to determine whether to use the Admit Date or the Req Date to calculate acceptable dates.

Considerations

  • Vendor First logic cannot be used with Covering Provider logic

  • All match rules must include Provider ID as a criteria

  • Days before and days after is reading the Authorization request date

  • A provider can only be a part of one covering provider group, per contract

  • Providers covering for one another can have different vendors if the Reimb Method list is set to Use Covering Provider pricing rules in keyword CVPVG

  • Days/Visits used will be correctly applied to the Authorization when Covering Provider is automatically applied to the claim
