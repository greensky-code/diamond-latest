ACPAY Field Definitions

< <

Field Name

Req

Description

Source

Comments

Additional Info Exists

O

Indicates whether or not additional information exists.

Manually entered

If this box is checked, you can view the additional information by selecting View AP Additional Info from the Special menu.

Address Line 1

D

First line of vendor’s address.

System generated

Some vendors have multiple addresses. The vendor address selected during the claims process is displayed in this field.

AP Transaction ID

D

Unique identifier of the account s payable transaction.

System generated

 

AP Type

D

Transaction category.

System generated

Valid values are:

  • Capitation

  • Medical

Bank Account Code

D

Identifier of the bank account from which the payment was made.

System generated

Value is from keyword BANKA (Bank Account).

Cap Fund Model ID

D

Displays the Model ID from the provider contract used to price the claim.

System generated

Value is from keyword CPMOD (Capitation Model).

Cap Fund Sub Model ID

D

Displays the Submodel ID from the provider contract used to price the claim.

System generated

 

Check Number

D

Number of the check that includes this transaction, once the transaction has been included in a posted check run.

System generated

The check number is generated during CKPST, Run Job.

Check/EFT Date

D

Date printed on the check, or the EFT transaction date.

System generated

This field is blank if the payment run has not been executed.

Claim Type

D

Source of the transaction.

System generated

The following claim sources may display:

  • Capitation

  • Dental

  • Institutional

  • Professional Services

Company Code

D

General ledger company with which the transaction is associated.

System generated

Value is from keyword COMPF (Company).

Credit G/L No. 1

D

Primary account that is credited by this transaction, as indicated by the G/L reference code in keyword GLREF (G/L Reference).

System generated

For claim detail payables, the account is credited when APUPD posts the transaction.

For capitation payables, the account is credited when the capitation run is posted in keyword CCALC.

Credit G/L No. 2

D

Secondary account that is credited by this transaction, as indicated by the G/L reference code in keyword GLREF.

System generated

For claim detail payables, the account is credited when APUPD posts the transaction.

For capitation payables, the account is credited when the capitation run is posted in keyword CCALC.

Debit G/L No. 1

D

Primary account that is debited by this transaction, as indicated by the G/L reference ode in keyword GLREF.

System generated

For claim detail payables, the account is credited when APUPD posts the transaction.

For capitation payables, the account is credited when the capitation run is posted in keyword CCALC.

Debit G/L No. 2

D

Secondary account that is debited by this transaction, as indicated by the G/L Reference Code in keyword GLREF.

System generated

For claim detail payables, the account is credited when APUPD posts the transaction.

For capitation payables, the account is credited when the capitation run is posted in keyword CCALC.

Discount Withhold

D

Claim detail line discount or withhold amount.

Displayed

DIAMOND may truncate the display of the currency amount.

Dscnt. Amt

D

Displays the discount amount calculated for this claim detail line.

Displayed

This field is filled in after you run check post.

Dscnt. Credit G/L No.

D

Displays the discount credit account for the claim detail line.

Displayed

This field is filled in after you run check post.

Dscnt. Debit G/L No.

D

Displays the discount debit account for the claim detail line.

Displayed

This field is filled in after you run check post.

Due Date

D

Calculated due date for the transaction based on the Payment Delay Days on the vendor’s record.

System generated; can be manually overridden.

 

EFT Trans Number

D

Number of the EFT that includes this transaction, once the transaction has been included in a posted electronic funds transfer run.

System generated

The EFT Trans Number is generated during CKPST, Run Job.

G/L Month

D

Month and year the transaction is posted to the general ledger.

System generated

For claim detail payables, the month and year are entered when the transaction is posted in keyword APUPD.

For capitation payables, the month and year are entered when the transaction is posted in keyword CCALC.

Intr. Amt

D

Interest amount calculated for the claim detail line.

Displayed

This field is filled in after you run check post.

Intr. Credit G/L No.

D

Displays the interest credit account for the claim detail line.

Displayed

This field is filled in after you run check post.

Intr. Debit G/L No.

D

Displays the interest debit account for the claim detail line.

Displayed

This field is filled in after you run check post.

Net Amount

D

Transaction net dollar amount of the claim(s).

Displayed

DIAMOND may truncate the display of the currency amount.

Paid Net Amt

D

Displays the amount of the payment.

Displayed

Net Amount less any discount applied or plus any interest applied.

Penalty Amt

D

Amount of total penalties in paid claim.

Displayed.

 

Posted Date

D

Date the transaction was posted to Accounts Payable.

System generated

For claim records, the date the transaction was posted in keyword APUPD.

For capitation records, the date the transaction was posted in keyword CCALC.

Pre Price Only

D

Indicator that the claim line that is the source of the transaction was pre-priced, rather than fully adjudicated.

System generated

Values are

  • Y Pre priced only

  • N Fully adjudicated line or capitation payment

Processing Status

D

Indicates the stage in the processing cycle.

Displayed

Valid values are:

  • F Final (posted)

  • H Held

  • M Manual Check

  • N Paid/No check

  • P Paid

RA/EOB Flag

D

Combination of instructions for generating an EOB and/or RA

System generated

DIAMOND generates the print flag from selections on LINBS and PROVC, rules in SEBRA and the Pay Subscriber flag.

Hard coded values are:

  • 0 No EOB, no RA

  • 1 Paper EOB, no RA

  • 2 No EOB, paper RA

  • 3 Paper EOB, paper RA

  • 4 No EOB, electronic RA

  • 5 Paper EOB, electronic RA

Use the Print Flag Info command to override the print flag. The print flag cannot be changed once the claim has been finalized.

Select for Payment

D

Indicator that the transaction has been selected for payment.

System generated

Values are:

  • M Manual

  • S Selected

  • U Unselected

  • C Check run in progress (Paid)

Vendor 1099 Flag

D

Indicator that the transaction will be included in 1099 reports for the vendor.

System generated

If the box is selected, the transaction will be included in 1099 reports. If cleared, it will not be included.

Vendor ID

D

Unique identifier of the vendor (payee) and the vendor’s name.

System generated

Value is from keyword VENDR (Vendor Master).

Withhold Amt

D

Displays the difference between the original Net Amount, prior to Cap Fund Distribution, and the new Net Amount resulting from a (posted) Cap Fund Distribution process.

System generated

DIAMOND may truncate the display of the currency amount.
