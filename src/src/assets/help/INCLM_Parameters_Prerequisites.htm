INCLM Parameters & Prerequisites

The following is only a brief listing and description of the parameters that affect this keyword. See PARAM Record Map for links to more information about each parameter. How DIAMOND Thinks documents may also give further details on how to configure these parameters for your particular needs.

Parameter

Description

Comments

APCBILLTYPES

Defines the bill types you want to be available for Ambulatory Payment Classification (APC) processing. Bill types are entered in the Bill Type box on the claims header window. Only the bill types you define will be available for APC processing.

Also defines the outpatient bill types you want excluded from Diagnosis-Related Group (DRG) processing.

Usually, bill types 130 through 139 are outpatient bill types.

BILLEDCHECK

Controls enforcement of an edit between the billed amount on the header and the sum of the billed amounts on the detail lines.

If the entry is EQUAL, DIAMOND will issue a warning on the claim whenever the detail line billed amounts do not add up to the Total Billed. If the entry is GREATER, DIAMOND will issue a warning only if the sum of the detail lines is greater than the Total Billed. If this parameter is left blank, DIAMOND does not edit the total detail against the Total Billed.

CLAIMAGE

Specifies the maximum number of days allowed between the current (system) date and the claim date of service.

 

CLAIMDUEDATE

When the vendor record does not specify Payment Delay Days, the entry in this record is used instead.

The due date is calculated as the claim receipt date + the number of days specified in this parameter.

COBCLAIMACTN

Specifies the action taken automatically when the member has other primary coverage.

Actions are Warn (default), Hold or Deny. Action is taken based on whether the claim’s primary date of service falls within primary other coverage as recorded in the member’s COBHS record.

DFLTPRINTSTAT, Param 2

Determines whether Explanations of Benefits (EOBs) and Remittance Advices (RAs) are printed for claims processed online when the member is ineligible.

Values are:

  • 0 No EOB, no RA

  • 1 Paper EOB, no RA

  • 2 No EOB, paper RA

  • 3 Paper EOB, paper RA

  • 4 No EOB, electronic RA

  • 5 Paper EOB, electronic RA

DFTCNTRCTPRV

This parameter record stores the Provider ID and the Oracle Sequence ID used to identify the Default Provider.

The Default Provider is used to create provider contracts in PROVC that apply to more than one provider. The contract search methods, which may include looking for a default provider contract are defined in the LINBS record corresponding to the member’s eligibility.

ELIGSTAT

Identifies valid eligibility status codes.

If the entry in the member’s Elig Sts field matches an entry in ELIGSTAT, the member is eligible. If the entry is not found in ELIGSTAT, the member is ineligible.

GLASSIGN1-6

Specifies which data fields on the claim are used to automatically assign the general ledger reference and company codes to claim detail lines.

INVALIDHOLD

Indicates if DIAMOND should hold a claim submitted through the DIAMOND Data Service API if invalid data was replaced with data from one of the following parameters:

  • INVALIDADMHR

  • INVALIDADMPV

  • INVALIDADMSO

  • INVALIDADMTY

  • INVALIDATTPV

  • INVALIDBILTY

  • INVALIDDIAG

  • INVALIDDISHR

  • INVALIDDRGCD

  • INVALIDFACID

  • INVALIDMBRID

  • INVALIDMODIF

  • INVALIDOCCCD

  • INVALIDPATST

  • INVALIDPOS

  • INVALIDPROC

  • INVALIDPRVID

  • INVALIDREFPV

  • INVALIDVENDR

This parameter is also used by the DDS Claims API.

Parameter 1:

Y Claim is allowed to process with replacement value; DIAMOND puts the Claim Header on Hold with the Hold Reason Code from Parameter 2.

N Claim is allowed to process with replacement value.

Parameter 2:

Valid Hold Reason Code to be applied to Claim Header. If this field is left blank, DIAMOND will process the claim as if Parameter 1 is set to N.

This process occurs automatically for claims submitted via the DDS API. It may also be manually selected for online claims if the user selects the Invalid Data check box.

TESTSVCAREA

Determines whether the provider’s postal code on a claim or authorization will be validated against the member’s IPA service area. The Service Area Indicator will display yes, no or blank.

Param 1 = Institutional Claims

Y Validation activated

N No validation

TPLCODEACT

When an authorization is matched to a claim during claims processing, determines whether the claim will be held or denied based on the AUTHS> Member ID/Eligibility>TPL (Third Party Liability) Code check box.

Parameters 1, 2, and 3:

Hold or Deny;<ReasonCode>

or Blank

Example 1 = H;HRESN

Example 2 = D;DRESN

USEAUTHDATA

Specifies the action taken when DIAMOND finds a match between a claim and an authorization.

If Parameter 1 is Yes, DIAMOND will automatically populate various institutional claim (INCLM) and professional service claim (PSCLM) header fields with associated authorization data. If No, it will not.

VENDORFIRST

Determines the order in which the cursor moves between the vendor and provider fields for newly entered claims.

Parameter 1:

  • Y - Cursor goes to Vendor ID/Address first.

  • Blank - Cursor goes to Provider ID/Address first

Parameter 2: (Available if Parameter 1 = Y)

  • Y - Display a list of providers that match the vendor ID and Vendor Address criteria before performing the contract search.

  • Blank - The system performs the contract search using the information provided in Vendor ID and Address, then displays valid provider contracts for selection.

WHLCLMPRICE

Controls whether whole claim pricing is available.
