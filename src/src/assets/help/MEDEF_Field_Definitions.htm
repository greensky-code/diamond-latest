Claim Type

R

Type of claim to which this medical definition applies.

Selected from predefined list

Valid values are:

  • P Professional services

  • I Institutional services

  • D Dental services

Description

O

Text name or description of the determinant in this record.

Manually entered

The DIAMOND table name list can be accessed by pressing F5.

Determinant

R

Element from the Determinant Table to use as a the search factor.

Selected from predefined list

The DIAMOND table name list can be accessed pressing F5.

Det Table

R

Name of the table in which the determinant described in this record resides.

Selected from predefined list

 

From

O

First value in the range of determinant values to which this record applies.

Manually entered

The range described by From and Thru is inclusive of the values entered in those fields.

Important: If you are entering a date(s) in the From/Through determinant fields, you must enter the date in MM/DD/YYYY format (including slashes). For example, 12/25/1999, 02/01/2000.

If you have selected a field in the Determinant box that uses system codes for its values, then you must type the from value exactly as it appears in the System Code box in keyword CODES for the system code type.

For example, if the System Code box is set to 01, to match that value, you must also type 01 in the From box. The value 1 is different than 01.

Include in Global Search

O

If selected, this rule applies across the board, and may or may not be tied to an additional data requirement defined in MEDCO for an Alternate Search.

Selected from check box

Regardless of whether this box is selected or not, this rule may be applied to an Alternate Search. However, if not selected, it will not also be applied across the board.

Med Def Code

R

Medical definition to which the attributes of this record apply.

Selected from predefined list

Medical definitions are defined in keyword MEDLU.

Operator

R

Indicator of whether the determinant values in this record represent an ‘include’ logic or ‘exclude’ logic.

Selected from predefined list

Valid values are:

  • EQ Apply this medical definition if the values listed in this record match those on the claim.

  • NOT Apply this medical definition if the values listed in this record do not match those on the claim.

Order

R

Order in which DIAMOND evaluates the applicability of the determinant values for this medical definition.

Manually entered

The lowest order number record is evaluated first.

A maximum of four digits can be used for order; leave gaps between order numbers so that records can be inserted later on, e.g. 1005, 1010, 1015, and so forth.

Search Sequence

R

Indicator that more than one determinant value must be satisfied to assign this medical definition; search sequence is applied within a single order number.

Manually entered

Sequence numbers indicate an "And" relationship among records.

Thru

O

Last value in the range of determinant values to which this record applies.

Manually entered

The range described by From and Thru is inclusive of the values entered in those fields.

Important: If you are entering a date(s) in the From/Through determinant fields, you must enter the date in MM/DD/YYYY format (including slashes). For example, 12/25/1999, 02/01/2000.

If you have selected a field in the Determinant box that uses system codes for its values, then you must type the thru value exactly as it appears in the System Code box in keyword CODES for the system code type.

For example, if the System Code box is set to 01, to match that value, you must also type 01 in the Thru box. The value 1 is different than 01.
