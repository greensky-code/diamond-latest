Entering a ZIP/Postal Code

The DIAMOND program uses pre-loaded U.S. Standard Attributes and Edits that control the following:

  • How you enter numbers in a ZIP/Postal Code field on keyword windows.

  • How ZIP/Postal Codes are displayed on keyword windows.

Enter a ZIP Code as digits and without punctuation; i.e., no dash. A dash is automatically added in the ZIP Code field after you type the number. If you type 94112, then 94112- displays. If you type 941124545, then 94112-4545 displays.

You can override these internal pre-loaded U.S. Standard Attributes and Edits and create customized ZIP Code attributes. You can then enter postal codes with formats that differ from the U.S. Standard. For more information, see UDATT, User Defined Attributes.

Automating city and state entries

You can automate the entry of cities and states on many keyword windows by typing a ZIP/Postal Code. For example, if you type 94112 in the ZIP Code field, then San Francisco automatically displays in the City field and CA automatically displays in the State field.

To automate city and state entries

  1. In keyword ZIPRF, enter the ZIP/Postal Code, city, state, county, and telephone area code. For more information, see ZIPRF, ZIP Code Lookup.

  2. In keyword PARAM, display the DFLTFZIP parameter and in the Parameter 1 box, type Y. For more information, see PARAM, Parameter.
