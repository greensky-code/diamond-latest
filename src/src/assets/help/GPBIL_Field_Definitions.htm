GPBIL Field Definitions

Field Name

Req

Description

Source

Comments

Action

R

Reason or result desired from this billing run.

Selected from predefined list

Hard coded values are:

Calculate: DIAMOND ensures there are no other calculate runs for the same Billing Cycle, Bill Job Type, and Bill Through Request Date, and that there are no outstanding billing runs for the same Billing Cycle.

Post: posts revenue to the GL

Cancel: cancels the billing run; it must be resubmitted.

Bill Job Type

R

Indicator of whether this is a routine monthly bill, or a bill showing changes in eligibility only (for groups billed less frequently than every month)

Selected from predefined list

Valid values are:

Regular

Supplemental

Groups that are billed quarterly, semi-annually, or annually require only supplemental bills.

Bill Thru Req. Dt

R

Premium bill records are created for each customer in the cycle for the number of frequency periods that are completed by this date

Manually entered

When the (GPBIL) Billing Type = Group and the group is being billed at the Super Group or Administrator level (GRUPB Bill To), only one billing period is calculated, for each GPBIL run, regardless of the Bill Thru Req Dt.

When the (GPBIL) Billing Type = Group and the group is being billed at the Group level (GRUPB Bill To), multiple billing periods are calculated for each GPBIL run, based on the Bill Thru Req Dt.

Billing Cycle

R

Set of groups with the same Type and Frequency; the cycle is defined in GRUPB.

Manually entered

 

Billing Type

R

Variable that defines whether a set of individuals is being billed to a group, such as an employer, or are being billed individually.

Selected from predefined list

Valid values are:

Annual

Group

Individual

Combined

Comments

 

Narrative information about the run.

Manually entered

 

Date

D

Date of original submission of billing run.

System generated

 

Job ID

R

Unique identifier of billing run.

System generated

Previously submitted Job IDs are displayed in addition to current ID.

The entry in this field may be manually overridden by entering an ID not previously used.

Post Month

R

Month and year to which revenue will be posted from this billing run.

Manually entered

 

Request Type

 

Future use.

Selected from predefined list

Valid values:

Immediate: Default

Deferred: Future use

Request User

D

ID of the user who originally entered the billing run.

System generated

 

Status

D

Status of the billing run.

System generated.

Values are:

New: Run has been setup but not calculated

Calculated: Run has been calculated but not posted.

Posted

Cancelled

Error: An error was detected during either calculation or posting.
