Benefit Pkg

Displayed

 

Compare Dates

C

Determines if the system compares or ignores ACCUM record date when calculating accumulated benefits

Manually entered

This field is only available when the Rule Type is Limit and the Limit Type is Days or when the Rule Type is Full Coverage and the Full Coverage Type is Days.

Valid values are:

C - Compare dates: the system compares the date of the manually entered accum record against the date of service on claims. If the dates are the same, the system does not include the accum record in the calculation.

I - Ignore dates: the system counts each accum record as one day regardless of whether or not the date on the accum record matches the date of service on a claim.

DIAMOND ID

Unique Identifying Number

There is one identifier for every instance of a person in the Member Master records

Manually entered

A DIAMOND ID enables dual coverage in DIAMOND.

If this person has more than one insurance coverage, the other instances of the person appear in a window and the appropriate one can be selected from the list.

For more information, see Dual Coverage, How DIAMOND Thinks.
