CLHLD Field Definitions

Field Name

Req

Description

Source

Comments

Claim Type

R

The calibre126 of claim.

Selected from a predefined list.

Valid values are:

Dental

Institutional

Professional

Description

O

Meaningful description of the hold Rule.

Manually entered.

 

Determinant

R

Column name from the claim header DIAMOND uses to match a rule to a claim.

Manually entered.

 

From Value

R

From Value of a range for the claim header determinant used to determine whether this claim header rule applies to a claim.

Manually entered.

Must be a valid value for the current claim header determinant.

For certain columns, the value needed is the Sequence ID, not the actual ID. For example, the Provider Sequence ID is needed, not the Provider ID.

One or multiple From Value – Thru Value ranges may be entered for a single Determinant. Press CTRL+N to add a new range.

Hold Rule

R

Hold Rule name.

Manually entered.

 

Operator

R

Operator (equal to or not equal to) used in application of this header hold rule.

Selected from a predefined list.

Valid values are:

EQ (Equal to)

NOT (Not equal to)

Reason Code

R

The Reason Code assigned when DIAMOND determines this hold Rule applies to a claim.

Selected from an implementation specific list.

Hold reason codes are entered in Reason (REASN) with reason code type HD.

Thru Value

O

Thru Value of a range for the claim header determinant used to determine whether this claim header rule applies to a claim.

Manually entered.

Must be a valid value for the current claim header determinant.

If no value is entered in this optional field, DIAMOND uses the single value specified in the From Value field. This is how you may specify a single determinant value.
