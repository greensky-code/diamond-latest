Enabling Notes Functionality

 

 

 

Overview

The Notes functionality enables you to save text related to a particular record such as a claim, a member, or an authorization. For example, you may make a note of a call from a member or you may enter comments about an authorization. Once a note has been saved, it can be edited but it cannot be deleted.

DIAMOND includes many notes-capable windows. You can set up any of these windows to use notes functionality.

The Components of Notes Functionality

The components of notes functionality:

  • Note Types

  • Window and Note Type Associations

  • Window and Note Type Links

  • Note Notification

Note Types

Note types are categories of notes such as ELG for member eligibility notes or ATH for authorizations notes.

Window and Note Type Associations

Window and Note Type Associations determine which note types are available from which windows. You can make a note type accessible from just one window or you can make it accessible from two or more windows. For example, you might create the Note Type ELG and create one association to make it available from MELIG. Or you might create two associations to make ELG available from MELIG and MEMBR.

Window and Note Type Associations allow you to view and edit notes in the screen from which the note was created. For example, If you set up two Window and Note Type Associations, one to make ELG accessible from MELIG and another to make ELG accessible from MEMBR, you can create an ELG note from MELIG but you can only view and edit that same note in MELIG. If you want to create an ELG note in MELIG and have the ability to view and edit that same note in MEMBR, you must create a Window and Note Type Link.

Window and Note Type Links

Window and Note Type Links enable you to create a note in one window and edit the same note in another window. See the Linking Note Types to Multiple Keywords section of this document for more information.

Note Notification

Note Notification, if used, displays an indicator or a dialog box when you are accessing a database record that has an associated note. For example, you might be viewing a claim that has an urgent note associated with it. While you are viewing or editing the claim, depending on your organization’s notes setup, you may see an indicator in the title bar or a dialog box may display with information about notes that exist for a claim.

About Note Types

You add note types in keyword NOTET. The information you provide affects the Note Detail window. The Note Detail window is where you enter new notes and edit existing notes.

User defined fields

Each note type record can have up to 5 user-defined text fields and up to 5 user-defined date fields. The user-defined fields only show up in the Note Detail screen if you set them up in the note type record. Keyword NOTET includes a check box that allows you to make any of the user-defined fields required.

Note Types and Note Priority

Each note type has a default note priority. The priority level you assign to a particular note type in NOTET is automatically assigned to any new note of that type that you create. See the Notes Notification section of this document for more information.

Enabling Notes Functionality

There are two requirements for enabling notes functionality for a window. First, the window must be one of the 35 notes-capable windows. See the Notes-Capable Windows and Link Context section of this document for a list of notes-capable windows. Second, you must associate at least one Note Type with the window by creating a Window and Note Type Association. Window and Note Type associations are set up in NOTEW (Note Window).

Associating Note Types with Windows

For a note type to be usable, it must be associated with at least one window. If you create a Note Type but neglect to associate it with a window, you will not be able to use that note type. Once you associate a note type with a window, you can create notes of that type from that window. For example, if you create note type ELG and make a Window and Note Type Association between MELIG and ELG, you can then create ELG notes from the MELIG window. You can associate a note type with more than one window. In the example above, if you associate ELG only with the MELIG window, then you can only create ELG notes from MELIG.

Linking Note Types to Multiple Keywords

You can setup links from each Window and Note Type Association to other windows, this is called a Window and Note Type Link. Links allow you to add a note in one window then view or edit the same note from another window. Each link requires two Window and Note Type Link records. For example, if you want to create notes with Note Type ELG in MELIG and you want to view or edit these ELG notes in INCLM, you must create the following associations and links in NOTEW:

Notes-Capable Windows and Link Context

Link context is part of the Window and Note Type Link. Link context is also used to filter a list of notes while in the note view window. There are three link-contexts available: M (member), G (group), and P (provider). Each of the 35 notes-capable windows has a list of possible link contexts. Some windows can use all three link contexts while other windows cannot use any. Windows that don’t have any available link context cannot be linked to other windows. You can only link windows if they share at least one link context in common.

Notes Notification

Each Note Type has a default priority level that you define in NOTET. The priority level determines the type of notification that the user receives when viewing a record that has an associated note. The notification may be a message that appears in the title bar while the user is viewing or editing a record. Or the notification may be a pop-up window that forces the user to view the associated note(s).

Only the originating author can change the priority of a note and only while creating a new note. Once the note has been saved, the priority cannot be changed.

Only active notes trigger notes notification actions. A note is considered active as long as the note’s expiration date is later than the current date.

Three levels of Notes Priority

DIAMOND offers three levels of notes priority. Each priority level has a different notification action.

The user must read each of these notes before the system will allow them to return to the record.

Disabling Note Notification Actions

If you do not want to use the notification options associated with the priority levels, but you want to assign the priority levels to notes, set NOTEALERT parameter 1 to N (no). See Param user documentation for more information.

Note Notification Message Text

The text for each note notification action is located in MSGMT (Message Maintenance). The note notification message text is hard-coded and cannot be changed.

Viewing Notes

Two screens are available for viewing notes. The first is the Notes List and the second is the Note Detail. The Notes List can be accessed from the Notes menu or from the F4 Function key in any of the notes-enabled windows. The Note Detail screen is accessed from the Notes List or from the Go To Notes Window popup that appears for priority 2 notes. The Note Detail screen displays the detail of a single note.

Viewing the Note List Window

When you access the Note List by using the Note Menu or F4 function key from any notes-enabled window, it displays the default note list. The default note list is a list of all notes that are directly associated with the record you were viewing and that were originated in the window from which you accessed the Notes List. The system determines which notes are associated to the current record based on the screen from which the note was originated. For instance, if you access the Notes List from the AUTHS window, the system lists all notes that have the same authorization number as the authorization record you were viewing, or if you access the Notes List from INCLM, the system lists all notes that have the same claim number as the claim record you were viewing. See the Search Criteria for Notes List Window Default Display column of the Notes-capable windows and their link contexts table in the Notes, How DIAMOND Thinks section of this document.

The list of notes is sorted by priority and then by date in descending order. You can resort the list by clicking on any of the column headings and you can use the options at the top of the screen to filter the note list or resort it.

Consolidating the Note List

Consolidating the note list enables you to view notes that were originated in different windows and that are related to the record you are viewing at the time you access Notes. Consolidation only works if the source window is linked to other windows by a Window and Note Type Link in NOTEW. Also, the context of the Window and Note Type Link must be the same as the consolidation option you select in the Note List screen.

When you select a consolidation option, DIAMOND checks to see if any Window and Note Type Links exist for the source window. The source window is the window from which you access the note list. If links do exist, DIAMOND looks at notes that were created in the linked windows. If DIAMOND finds notes that were created in other windows and that are related, based on the consolidation option you chose, these notes are combined with the notes from the default list to create a new note list display.

As an example, you are in the AUTHS window viewing authorization record 001 for member Betty Smith. The default note list only shows notes that were originated in AUTHS and that are directly associated to authorization record 001 for Betty Smith. You want to see if there are other notes for Betty Smith, whether or not they are related to authorization 001. You can use the Member Consolidation Option to accomplish this. Once you select to Consolidate Notes by Member, DIAMOND checks NOTEW to see what links exist for AUTHS. If links do exist, DIAMOND looks for notes for member Betty Smith that were created in the windows to which AUTHS is linked. DIAMOND keeps the default list of notes on display and adds to the note list the other notes it has located for Betty Smith.

You cannot use every consolidation option with every screen. Refer to the Notes-Capable Windows and Their Link Contexts list for information on notes-capable windows and what context each can use.

Viewing the Note Detail Window

If you access Note Detail from the Notes List, you can return to the Note List at any time by clicking on the List button.

When you access Note Detail from the Go To Notes Window popup box, you are required to view all priority 2 notes associated with the record you were viewing. As long as there are more unread priority 2 notes, the Next button is active. After you have read all of the priority 2 notes, the List button becomes available. The list button enables you to go to the Note List screen to view a list of all of the priority 2 notes. When you access the Note List from the Go To Notes Window popup, the list only shows active, priority 2 notes, it does not show priority 0 or 1 notes, or expired priority 2 notes that exist for this record. If you want to view the Note List in default mode, (see all records associated with the record you were viewing and that were originated in the window where you received the Go To Notes Window popup box), select All Types in the Filter Note By box and select None in the Consolidate Notes By box.

Notes Expiration Dates

The termination date on the Note Detail window only affects notes notification actions. If a note has a termination date that is equal to or earlier than the current date, it will not trigger a notification action.

Hard-Coded Note Types

DIAMOND includes three hard-coded note types: API, CHK, and PRI.

API Note Type

Notes are automatically created for claims entered through an API. These notes are assigned note type API. Any remarks associated with institutional claims that come into DIAMOND through the API are included in the API note.

CHK Note Type

DIAMOND automatically creates notes for claim detail lines that are run through CVSUP (Clear, Void, Stop Update). These notes are automatically assigned note type CHK and store the Check Number, Check Date, Check Status, Reason Code, Claim Line Number, and Status Date.

PRI Note Type

A PRI note is automatically created when you set or change a member’s privacy indicator in MEMBR. PRI notes are assigned a priority of 0 (low). The user-defined fields store the Termination Reason, Information Date, Information Received By, and Information Received Through.

No Notes Archive or Purge Function

DIAMOND does not currently include notes archive or notes purge functionality.

Comment
