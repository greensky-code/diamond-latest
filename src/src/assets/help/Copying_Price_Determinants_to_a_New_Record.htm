Copying Price Determinants to a New Record

Restrictions

Instructions

Because price determinant validation data elements can be lengthy, it is handy at times to copy the entries from one record into a new record, and making any changes to that new record if necessary.

Restrictions

None.

Instructions

Step

Comment

1. In PDTER, press F5 to select an existing price rule determinant code or create a new price rule determinant to be the receiver of the data to be copied.

 

2. Select Copy Price Determinant from the Special menu.

 

3. Highlight the price rule determinant code to be copied.

In the bottom focus box DIAMOND displays data from the selected price rule determinant code.

4. Click the validation data elements you want to copy to the receiving record.

To copy all data elements, click Select All.

To copy none of the validated data elements click Deselect All.

To select a series of consecutive validation data elements, click the first data element, hold down SHIFT and click the last data element. All validation data elements between (and including) the selected first and last will be highlighted.

To select specific non-consecutive validation data elements (or to select data elements one-at-a-time), click on (highlight) the first data element, hold down CTRL and click once on each of the additional data elements you want to copy.

5. Click OK to copy the validation data elements to the receiving record; click OK again to complete the copy operation or Cancel to cancel the operation.

 

6. Save the new record with CTRL+S, or click Yes to "Save changes?" when exiting the receiving PDTER window.

To cancel the copy, click NO to the "Save changes?" prompt.
