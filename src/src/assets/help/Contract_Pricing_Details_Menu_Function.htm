Field Name

Req

Description

Source

Comments

Action

O

The action taken on the claim line when the Target Rev Code Edit fails.

Selected from predefined list

Valid values are

Deny

Hold

Claim Type

R

The type of claim to which pricing details apply.

Selected from predefined list

Values are

Dental

Institutional

Professional

Contract Type

R

The contract type that is applied when the claim meets the conditions defined by Det Table and Determinant.

Selected from predefined list

Valid values are

P Payable (fee-for-service)

F Cap/Special Fund

C Capitated

S Staff

Det Table

C

The tables associated with the entry in Claim Type—if Institutional, only institutional claim tables are shown, likewise for professional or dental.

Selected from predefined list

Required field except for Order 9999, the default record.

Determinant

C

The values to which this contract detail apply, from the table specified in Det Table.

Selected from predefined list

Required field except for Order 9999, the default record.

From/Through

R

The determinant values used to evaluate whether there is a match to the claim being processed, used in combination with the Operator.

Manually entered

One set is required. Values must be consistent with the values contained in the table and column above.

Multiple values are evaluated as "or."

Important: If you are entering a date(s) in the From/Through determinant fields, you must enter the date in MM/DD/YYYY format (including slashes). For example, 12/25/1999, 02/01/2000.

Geo Region

O

The geographic region used for the matched claim.

Selected from predefined list

 

Operator

R

The operator for the determinant, applied to the values entered below.

Selected from predefined list

Valid options are

EQ Equal

NOT Not equal

Order (Identifier section box)

R

The order in which DIAMOND will search detail records for a rule that matches the claim being priced.

Manually entered

Default contract pricing records are assigned Order 9999.

Detail rules should be arranged so that the most specific rules have the lowest Order numbers.

Order (Price Schedules section box)

R

The order in which the price schedule/price schedule percent allowed will be evaluated during pricing.

Manually entered

Evaluation starts with the lowest order number.

Pct Allowed

O

The allowed percentage used to price the matched claim detail

Manually entered

 

Pct of Billed

The percentage of the billed amount used to price the matched claim detail.

Manually entered

 

Pct Withhold

O

The percentage used to calculate the withhold amount on the matched claim detail.

Manually entered

 

Price Rule 1

O

The price rule used for the matched claim.

Selected from predefined list

 

Price Sched

O

The price schedule used for the matched claim.

Selected from predefined list

F5 lookup is available.

Price Sched Pct Allowed

O

The allowed percentage to be applied to the allowed amount for a price schedule.

Manually entered

 

Pricing Region

O

The pricing region used for the matched claim.

Selected from predefined list

 

Reason

O

The reason code applied to the detail claim line when the Target Rev Code Edit fails.

Selected from predefined list

Applies to keyword INCLM only.

Reason codes are maintained in keyword REASN; reason type must match action taken (see Action).

Search Sequence

R

An indicator that more than one set of pricing details apply to a single rank within Order.

Manually entered

In the search for a pricing detail matching a claim, multiple sequence numbers within Order are evaluated together; a claim must match all of the sequenced records within an order for a ‘hit.’

Service Region

O

The service region used for the matched claim.

Selected from predefined list

 

Target Rev Code Edit

O

If selected, DIAMOND validates that the alternate procedure code entered is associated with the revenue code entered. If not, the Action and Reason in PROVC details are assigned.

Selected from check box

Applies to keyword INCLM only.
