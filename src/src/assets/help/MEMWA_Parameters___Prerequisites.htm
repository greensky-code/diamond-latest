MEMWA Parameters & Prerequisites

The following is a brief listing and description of the parameters that affect this keyword. See PARAM Record Map for links to more information about each parameter. How DIAMOND Thinks documents may also give further details on how to configure these parameters for your particular needs.

Parameter

Description

Comments

DATEFORMAT

Determines the format for dates for all processing on the DIAMOND server.

Client processing date format is controlled by the Windows Control Panel.

Relative positions of month, day and year must be the same for both client and server formats.

Parameter 1:

Enter one of the following formats:

MM/DD/YYYY

DD/MM/YYYY

YYYY/MM/DD

This parameter is changeable at any time.

DIDSEARCH

Parameter used to set dual coverage DIAMOND ID functionality.

Once this parameter is set, a DIAMOND ID must be entered in the MEMBR screen for each new member.

‘Y’ enables dual coverage

‘N’ is the default parameter. If ‘N’ is set, then dual coverage is turned off.

For more information, see Dual Coverage - Overview.
