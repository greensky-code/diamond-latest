About modifier codes

A modifier code is a 2-digit suffix entered along with a procedure code that further defines the nature of the service provided. While there can be implementation-specific modifier codes, most are defined in the CPT schedule and therefore come with the claim.

Some modifier codes require manually pricing; the calculations required are situation-specific. You can also use keyword PRULD (Price Rule Detail) to set up a pricing logic so the modifier code will use table-driven pricing.

Learn how to add a modifier code.

Where you can use modifier codes

You can use modifier codes when entering claim detail line items in keyword PSCLM (Professional Services Claims) and keyword INCLM (Institutional claims).

You can enter up to four modifier codes in keyword PSCLM, but only the modifier codes entered in the first or second Mods boxes are using in pricing and adjudication. The modifier codes entered in the third or fourth Mods boxes can be used for reporting purposes.

You can enter one modifier code in the Modifier box in keyword INCLM.

About wildcard modifier codes

A wildcard modifier code can be used by added a default procedure price record in keyword PROCP that will be used when DIAMOND does not find a match between the claim detail line modifier code and the procedure price record.

Sample calculation

The following sample calculations refer only to a modifier code being entered in the first Mods box.

A. Price rule record 12345 is added in keyword PRULE with a wildcard modifier of XX

  • Price rule 12345; wildcard modifier XX

B. The following price rule detail records are added in keyword PRULD for price rule 12345:

  • Price rule 12345; modifier code 90

  • Price rule 12345; modifier code 30

  • Price rule 12345; no modifier code

Price rule detail records must be added in keyword PRULD for all modifier codes that will be entered on claim detail lines.

C. Procedure price records are added in keyword PROCP for the following:

  • Procedure code 98765; modifier code 90; $44 (procedure/modifier specifically priced)

  • Procedure code 98765; no modifier code; $27 (price for procedure if no modifier code is entered on the detail claim line)

  • Procedure code 98765; wildcard modifier XX; $26 (default modifier price)

D. Claim detail lines entered for the following:

  • Procedure code 98765; modifier code 90; pay at $44

  • Procedure code 98765; no modifier code; pay at $27

  • Procedure code 98765; modifier code 30; pay at $26 (default when modifier entered, modifier is not specifically priced and wildcard modifier is defined/priced)

About Professional Modifier Pricing

In keyword PRULD, you can add a price rule detail record to use Professional Modifier Pricing. It allows you to use the second Mods box in keyword PSCLM to price a claim. In keyword PRULD, the Price Method list allows you to select four different methods of pricing.

The following table describes the methods available in the Price Method list in keyword PRULD:

Method

Description

Standard Modifier 1

In keyword PSCLM, use the modifier code in the first Mods box to calculate the allowed amount for the claim line item using the price rule detail record. This is standard modifier pricing.

If modifier codes are entered in the second, third, and fourth Mods boxes, then pricing and adjudication ignores them. Instead, they are only available for reporting purposes.

Pay Modifier 2

In keyword PSCLM, use the modifier code in the second Mods box to calculate the allowed amount for the claim line item using the price rule detail record.

If modifier codes are entered in the first, third, and fourth Mods boxes, then pricing and adjudication ignores them. Instead, they are only available for reporting purposes.

Modifier 1 + Modifier 2

In keyword PSCLM, do the following:

  • Use the modifier code in the first Mods box to calculate an allowed amount for the claim line item using the price rule detail record

- Then -

  • Use the modifier code in the second Mods box to calculate an allowed amount for the claim line item using the price rule detail record

- Then –

  • Take the results of each both calculations, add them together, and calculate the final allowed amount

Modifier 1 + Modifier 2%

In keyword PSCLM, do the following:

  • Use the modifier code in the first Mods box to calculate an allowed amount for the claim line item using the price rule detail record

- Then -

  • Take the resulting calculation and multiply it by the amount in the Allowed Factor Ovr box under Multiple Modifier Pricing Information on the price rule detail record

You can also set up the PRICEMODS12 system parameter in keyword PARAM (Parameter) to price or deny a claim line item when price rule detail can be found for the Modifier code in the first Mods box in keyword PSCLM, but price rule detail cannot be found for the modifier code in the second Mods box. For more information about system parameters, see PARAM, Parameters.
