EOBs and RAs - Overview

The following flowchart illustrates the EOB and RA process.

Process Flow

During claims processing, DIAMOND processing evaluates if an EOB and/or an RA should be sent for each claim.

  1. DIAMOND processing checks the member’s eligibility. If the member is eligible, then DIAMOND continues to the next step. If the member is not eligible, then the DFLTPRINTSTAT parameter setup determines the print flag and skip to step 4.

  2. DIAMOND processing reads the provider’s RA information from the provider contract in keyword PROVC (Provider Contract); then reads the member’s EOB information from keyword LINBS (Line of Business).

  3. DIAMOND processing then uses those two pieces of information to calculate the print flag.

  4. DIAMOND processing searches through the EOB/RA suppression rules in keyword SEBRA. If any rules apply, the information from keywords PROVC or LINBS is ignored and the EOB/RA is suppressed as defined by the rules.

  5. DIAMOND processing then examines the Pay Subscriber and Pay Dependent fields in keywords DNCLM, INCLM, or PSCLM. If either the Pay Subscriber or Pay Dependent field is set to Y, then the RA is suppressed.

  6. DIAMOND processing writes the final print flag to the claim.

  7. You can manually overwrite the print flag on the claim, at the detail level only, before finalizing the claim.

If the print flag has been manually overwritten, the Print Flag Status indicator will display "M" for manual. This status will not change and the print flag will remain as manually selected, even if the claim header or detail is altered. To change the Print Flag Status indicator back to "A" for Automatic, you must use the "Auto" button on the Print Flag Info Special menu window to reinvoke the automatic process as described in steps 1 - 7.
