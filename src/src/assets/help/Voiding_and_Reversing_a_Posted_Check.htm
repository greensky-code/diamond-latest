Void and Reverse a Posted Check

Use the following procedure to void and reverse a posted check. This procedure assumes that the claims paid by the check were processed in error and new claims will be entered and paid.

Important Note  In keyword CHECK (Check Register), the check status must be "Posted."

To void and reverse a posted check

  1. Press F3, type CHECK, and then press ENTER.

  2. Select the check or EFT transaction.

  3. On the Special menu, click Stop/Void.

  4. On the Action list, click Void//Reverse.

  5. In the Reason list, click a reason.

  6. Click OK. The check's status changes to "In Process."

  7. On the File menu, click Save.

  8. In keyword CVSUP, run the clear/void/stop job to verify its accuracy, and then post it.
    When you post a clear/void/stop job, the following occurs:

  9. In keyword CHECK, the Check Status field is set to Reversed.

  10. In the claim detail window for each affected claim detail line, the Proc Stat list is set to F – Final (posted).

  11. In keyword ACPAY, the original claim’s A/P transaction record has the Processing Status box set to F – Final (posted), the Select For Payment list set to Selected, and the Check Number and Check/EFT Dt fields are cleared.

  12. In the claim detail window for each affected claim detail line, a new claim line is added with negative dollar amounts equal to the original claim detail line. The Claim Status list is set to A – Adjustment and the Proc Stat list is set to U – Unposted (open). If a claim line has a prior reversal, then any applicable patient liabilities or accumulators are not reversed again. When you run an A/P update job in keyword APUPD, an A/P transaction record is created for the offset amount in keyword ACPAY.
