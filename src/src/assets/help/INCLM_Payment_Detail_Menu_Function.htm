Offset

 

Indicates if the payment is an offset to an advance payment.

 

Displayed

Payment Status

D

Check or EFT payment status

Displayed

Values are:

  • Cleared

  • Posted

  • Reissued

  • Reversed

  • Stop Confirmed

  • Stop Requested

  • Voided

See Check Register (CHECK) for complete status descriptions.

Print Flag

D

Combination of instructions for generating an EOB and/or RA

Displayed

DIAMOND generates the print flag from selections on LINBS and PROVC, rules in SEBRA and the Pay Subscriber flag.

Hard coded values are:

  • 0: No EOB, no RA

  • 1: Paper EOB, no RA

  • 2: No EOB, paper RA

  • 3: Paper EOB, paper RA

  • 4: No EOB, electronic RA

  • 5: Paper EOB, electronic RA

Use the Print Flag Info Special menu command to override the print flag. The print flag cannot be changed once the claim has been finalized.

Print Flag Status

D

Indicates how the print flag was calculated

Displayed

Hard coded values are:

  • A: Automatic

  • M: Manual

RA Flag

D

RA Flag as selected on PROVC

Displayed

This is the original selection from PROVC, prior to evaluating rules or invoking processing logic.

Values are:

  • 1: Paper RA for Inst & Prof

  • 2: No RA for either Inst or Prof

  • 3: Electronic RA for Inst & Prof

  • 4: Electronic RA for Inst only

  • 5: Electronic RA for Prof only

  • 6: Paper RA for Inst only

  • 7: Paper RA for Prof only

  • 8: Paper for Inst & electronic for Prof

  • 9: Paper for Prof & electronic for Inst

Reason

D

Reason code (if any) associated with the payment status.

Displayed

Payment status reasons are maintained in keyword REASN with a type of CK.
