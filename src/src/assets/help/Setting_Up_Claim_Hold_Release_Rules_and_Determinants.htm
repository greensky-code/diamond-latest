Setting Up Claim Hold Release Rules and Determinants

Restrictions

Instructions

Frequently Asked Questions

In order to release and retest multiple hold conditions in a batch process, called Batch Claim Hold Release, you must set up claim hold release rules with specific determinants. Key particulars used by claim hold release rules are claim type, whether to apply the rule to header and detail holds or just header holds, the reason codes included in the processing, and the determinants and determinant value ranges themselves. You must specify a value for all of these key elements on each rule.

For more information about how to determine the key values for your models, see Batch Claim Hold Release, How DIAMOND Thinks.

Restrictions

None.

Instructions

Step

Comment

1. In keyword RHRUL, create batch claim hold Release Rules. Specify important criteria including:

  • What specific hold reason codes the rule includes

  • Whether each rule applies to Institutional or Professional claims

  • Whether the rule applies to both claim header and detail holds or header holds only

A claim hold release rule reflects the business needs of the health plan.

Rule names and claim types may not be changed once the rule is saved. Reason codes may be changed.

2. In keyword RHDET, select the determinants and define the determinant values for the claim hold Release Rules.

Determinants are available from the following tables:

  • Group Master

  • Professional and Institutional Claim Headers (except dates)

  • Professional and Institutional Claim Detail (except dates)

Determinant values entered in RHDET are not validated so exercise caution.

Determinants and Determinant values may be changed.

Sequence IDs for vendors, groups and providers are available as determinants in lieu of Vendor IDs, Group IDs and Provider IDs. Value ranges should not be used with these determinants since a range of Sequence IDs is not normally meaningful.

Frequently Asked Questions

  • How do I apply a claim hold release rule to both Institutional and Professional claims?

Claim hold Release Rules may only apply to either Institutional or Professional claims, not both. Instead, create two Release Rules, one for claim type Institutional and one for claim type Professional.
