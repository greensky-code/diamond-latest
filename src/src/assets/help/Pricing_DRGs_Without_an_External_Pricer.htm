Pricing DRGs Without an External Pricer

Restrictions

Instructions

Frequently Asked Questions

You can have DIAMOND price institutional claims at the whole claim level using Diagnosis-Related Groups (DRGs). To do so, accept the submitted DRG on the claim or enter a valid DRG, and then invoke DIAMOND whole claim pricing.

Alternatively, you can use an external grouper without an external pricer and still have DIAMOND whole claim price the claim. For more information on using an external grouper, see Grouping DRGs with an External Grouper.

Depending on your business requirements, you can instead use an external grouper and external pricer to both group and whole claim price institutional claims. See Pricing DRGs With an External Pricer for more information.

Restrictions

None.

Instructions

Step

Comment

1. DRG pricing relies on values in the following tables:

  • DIAGN (Diagnosis Code)

  • PROCD (Procedure Code)

  • PSCHD (Price Schedule)

  • REGIN (Region)

  • PROCP (Procedure Price)

  • REASN (Reason)

  • PROVC (Provider Contracts)

  • GRUPD (Group Detail)

  • LINBS (Line of Business)

Prior to attempting to set up DRG contracts you must analyze precisely how the contracts work and what they mean.

2. In keyword PDTER (Price Determinant Maintenance), add price determinants that define the characteristics of a claim to be priced using DRGs.

Keyword PRULW (Whole Claim Price Rule) records are selected using price determinants defined in Keyword PDTER.

3. In keyword PRULE (Price Rule), add a price rule.

In most cases, there will be a separate price rule record for each institutional provider; in other words, it is rare that more than one hospital will share the same contractual requirements for claims pricing.

4. If there are institutional services to be priced at the procedure or revenue code level and not included in the DRG rate, add records in keyword PRULD to define those ‘carved out’ procedures or revenue codes.

 

5. To define the specific procedure codes and/or revenue codes to which records in keyword PRULD apply, add keyword PRULS (Price Rule Detail Selection) records as needed.

DIAMOND reads PRULD records in level+sequence order. If there is more than one keyword PRULD record, there may be no need to set up a keyword PRULS record for the last keyword PRULD record read, since by the point that last record is evaluated, all specific exceptions will have been defined as part of the ‘earlier’ keyword PRULD records. In other words, the last rule read is the fall through, or all else, rule.

6. Add a keyword PRULW record with a Calc Method of 04 – DRG. This calculates the base allowed amount.

No Base Rule Stop Logic applies to DRG rules.

7. If outlier payments can apply, define a keyword PRULW record with outlier rules and Calc Method of 24 – Multiply DRG.

DRG outliers are defined according to dollar amounts. The outlier defines an additional payment; typically when the billed amount exceeds a calculated amount

8. Add a record in keyword DRGGP (DRG Grouper/Pricer Maintenance) with a Code of P - Pricer.

 

9. Add a record in keyword DRGPR (DRG Pricer Revision Maintenance) to identify valid revisions of the DRG pricer versions being used.

A revision record is required; set up the first version in use as revision 01 even though it is not truly a revision.

While a single provider has a single revision/version per contract, over time or over different contract relationships different revisions may apply. Also, different providers may require different revisions.

10. In keyword DRGCD, add the individual DRG codes.

The DRG code on the claim header is validated against DRGCD.

11. Enter the weights for each DRG code in keyword DRGWT.

Tip:

The field USERDEF1 can be used as a price determinant (PDTER) to group "weightless" DRGs to point to a separate PRULW with its own calculation method, e.g. a percentage of billed, since the DRG calculation requires a weight.

12. In keyword PROVC, add contract records as necessary for institutional providers. Be sure to enter all required DRG data, including:

  • DRG Base Multiplier (dollar amount multiplied by the individual DRG weight)

  • DRG Pricer ID

  • Pricer Version

  • Revision Level

  • DRG Outlier Multiplier (if applicable)

  • DRG Outlier % of Billed (if applicable)

If the pricer information does not vary with contract pricing details, you can set up the pricer information once at the contract level. If the pricer information does vary with contract pricing details, you must enter the pricer information for each needed contract pricing details record. Specifically:

  • If the pricer information does not vary with contract pricing details, from the Special menu, click DRG/APC Information and enter the required DRG data.

  • If the pricer information varies with contract pricing details, on the Special menu, click Contract Pricing Details. The Contract Pricing Details window displays. Then on the Special menu, click DRG Information and enter the required DRG data.
