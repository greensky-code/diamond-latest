Running Cap/Special Fund Distribution

Restrictions

Instructions

Frequently Asked Questions

As part of the process of paying providers from a specified fund of dollars, called a Cap/Special Fund in DIAMOND, it is necessary to run a batch job that calculates the amount owed to each provider in the fund for a given period of time. This allocation amount is based upon 1) the payment arrangement with the provider as defined in a CFD model and 2) how many claim lines have been entered during the period for the provider. During the batch job, DIAMOND actually modifies the Net Amount on the relevant claim detail lines, inserts any Cap Fund Withhold Amount, and flags them with a Cap Fund Status for Accounts Payable.

Restrictions

Do not run the Cap/Special Fund Distribution batch job while claims are being entered or processed for the provider or group of providers associated with the CFD fund model selected for the job.

Instructions

Step

Comment

1. Check that everything is ready to run the job:

  • Partially entered claims for the providers have been completed.

  • Holds on claims for the providers have been resolved.

  • Claims for the providers have been split when needed so that the claim detail lines all have From Dates of service in a single month.

  • The allocation amount for the period is available.

Each Cap/Special Fund batch job can affect unfinalized claim detail lines for any provider included in the Cap/Special Fund arrangement.

2. Determine the earliest date of service allowed on a claim line that should be included in the calculations.

 

3. Initiate the job for a given Cap/Special Fund model using CFDST in Run (edit) mode. Key items to specify are the monthly allocation amount and the service dates for this distribution period.

If you receive an error message or cannot initiate the job, see "Frequently Asked Questions."

During the edit run, you can cancel the job and run a new job for the same model and month/year. Once a job has posted, you cannot re-run the job or run another job for the same model and month/year.

4. Display the results of the job by using a CFDST function to generate the Audit Report Summary.

Each time a run/edit mode or post mode Cap/Special Fund Distribution run is initiated, a summary and a detailed CFD Audit report will be generated. These reports can be viewed on-line and printed if desired.

5. Review the results on the Audit Report Summary/Detail to decide if the calculated distribution amounts are reasonable according to the guidelines established by your health plan.

See CFDST reference documentation for Audit Report Summary and Detail field information.

Examples of questions to ask:

  • Are all providers included on the report that you expected to receive a payment?

  • Do the calculated amounts seem to correspond to claims utilization for the period?

6. If the calculated distribution amounts seem incorrect, cancel the job using CFDST.

  1. Research the problems and correct them.

  2. You may need to print the Audit Report Detail using a CFDST function before researching the cause. See "Frequently Asked Questions" for help in correcting any problems.

  3. Initiate another Cap/Special Fund Distribution job in run (edit) mode, changing the job specifications if necessary. (Return to Step 3.)

See CFDST reference documentation for Audit Report Detail field information.

If you cancel a CFDST job in the run/edit mode, DIAMOND changes the Cap Fund Status of all the qualifying claim detail lines from 2 - Successful run back to 1 - Cap Fund.

7. If the calculated distribution amounts seem correct, run the job in Post mode.

You cannot cancel a CFDST job once it is started in Post mode.

You cannot run another Cap/Special Fund batch job for the same model and period once the job completes successfully in Post mode.

8. Repeat these steps for any other CFD models that need to be processed for the period.

 

Frequently Asked Questions

  • I reviewed the CFDST Audit Report Detail and did not see any claim detail lines for a certain provider as I expected. What should I do?

Check the following.

First, review your provider contracts and verify that you entered the required data. Verify that you entered a PROVC > Def Contract Type = F - Cap Fund, a PROVC > Special menu command Contract Pricing Details > Contract Type = F - Cap Fund and the right Cap/Special Fund Model ID. Make corrections as needed.

Next, verify that the provider has claim detail lines with a date of service that is between the Cap Fund From Date on the CFDST job and the last day of the Cap Month for the job being initiated (or that the INCLM/PSCLM Override Cap Fund Date Range flag is selected).

  • A provider is inquiring on the status of a specific CFD claim line. How do I research this?

Verify that the claim line is a payable Cap/Special Fund Distribution claim line. If yes, check the Cap Fund Status for the line.

If Cap Fund Status = 1 - Cap Fund,

the line will be selected when a CFD job is run which includes the lines.

If Cap Fund Status = 2 - Successful Run,

the payment will be generated upon posting the corresponding CFD job.

If the Cap Fund Status = 3 - Posted,

the claim is ready for Accounts Payable. Check the payment detail to see if this claim has been posted to Accounts Payable.

  • How do I re-run a cancelled Run/Edit job?

You must enter and run a new CFDST job with the same specifications.

  • What if I can’t run a Run (edit) mode CFDST job?

If you already ran the CFDST job in edit mode, and you want to run CFDST again for the same Model and month/year (for example, if you entered more claims in the interim), you must first cancel the existing run. Then run the CFDST job in Run (edit) mode as usual.

If you already ran a job in post mode for this fund model and month/year and had it complete successfully, you cannot run another job for the same fund model and month/year.

  • What happens if I enter claims for Cap/Special Fund Distribution with dates of service included in a previously posted CFDST job?

If you have already run CFDST in Post mode for the month including these dates of service, you have a few options. First, you must manually identify and review these lines. Then, you may choose to:

  • Deny the lines (for example, for lack of timely submission)

  • Enter a CFDST From Date that will include the desired dates of service when you run the CFDST job for the next month’s cycle. Caution: If you never run a CFDST job including these dates of service, these claim detail lines will never be selected by a CFDST run.

  • Manually flag the lines to be available for CFD selection regardless of the CFDST date requirement (using the PSCLM/INCLM Special Function – Override Cap Fund Dates)

Example 1:

Part 1: You are running CFDST monthly for Model 1. You run CFDST for Model 1 for the month of January on February 15 with a From Date of 1/1/99. DIAMOND selects all CFD claim detail lines with a From date of service on or after 1/1/99 and through 1/31/99. On 2/20/99, you enter a CFD claim detail line with a From date of service of 1/22/99. You do not manually flag this line for inclusion in the next CFDST run for this model.

Part 2: You run CFDST for Model 1 for the month of February on March 15 with a From Date of 2/1/99. DIAMOND does not select the CFD claim detail line with the From date of service of 1/22/99 since it is before the CFDST From Date. As you continue with monthly CFDST runs, this detail line will not be selected. You must manually address this detail line as noted above or specify a From Date early enough to include this line.

Example 2:

You are entering in a new claim line for 2/15/98, but you have already run the CFDST job for 2/98 for the specified CFD model. When you run the 3/98 CFDST job you may specify the From Date to be 2/15/99 to include this claim line in the CFDST run.
