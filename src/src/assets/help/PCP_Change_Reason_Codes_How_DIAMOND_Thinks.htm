>PCP Change Reason Codes, How DIAMOND Thinks

You can add a reason code to store the reason why a member changed his or her primary care physician (PCP). Then, when you change a member’s PCP in keyword MEMBR (Member Master), you can select a PCP change reason code. Later, the PCP change reason code displays in keyword MELIG (Member Eligibility History). If you use Auto Letter Correspondence, then you can automatically generate correspondence based on the PCP change reason.

Setting up a PCP change reason code

To set up a PCP change reason code, do the following:

1. Add a reason code for PCP changes

You can add a reason code for each type of PCP change you want. If you submit API/DDS transactions, then it may be convenience to add a PCP change reason code for just those transactions, too. For more information, see Adding a Reason Code for PCP Changes.

2. Require users to select a PCP change reason code

You can require users to select a PCP change reason code when changing a member’s PCP in keyword MEMBR. That is, you can make the PCP Chng Rsn box required in the Provider Change dialog box. The Provider Change dialog box displays when you select Provider Change from the Special menu in keyword MEMBR. Make the PCP Chng Rsn box required using the PCPCHGREASN parameter in keyword PARAM (Parameter)

3. Set up default PCP change reason code

You can select a PCP change reason code to be the default in the PCP Chng Rsn box for both standard and API/DDS transactions. Set up a default PCP change reason code by using the PCPCHGREASN parameter in keyword PARAM.
