GROUP Field Definitions

Field Name

Req

Description

Source

Comments

Address 1

O

The first line of the group’s address.

Manually entered

 

Address 2

O

The second line of the group’s address, if any.

Manually entered

 

AL Dist. Meth.

 

Displays the preferred method of auto letter distribution.

Selected from predefined list

Select the contact’s preferred method of receiving auto letter correspondence: printed, faxed, or sent in e-mail.

For more information on auto letters, see Auto Letter, How DIAMOND Thinks.

Calc Days

O

A positive or negative integer that DIAMOND will add to the date resulting from the Due Date Rule.

Manually entered

You must first select a value in the Due Date Rule list before this box is available.

If you are using Super Group/Administrator level groups, then DIAMOND calculates the premium payment due date (using this field and the Due Date Rule list) from the Super Group or Administrator group record.

City

O

The city of the group’s address.

Manually entered

Depending on the values in keywords ZIPRF (ZIP Code), UDATT (User Defined Attributes), and the DLFLTZIP parameter in keyword PARAM (Parameter), this field automatically is filled based on the ZIP Code you entered.

Claim Hold Reason Code

O

The reason code which DIAMOND enters on detail lines for claims for members of this group to hold them from payment in the event the group has not paid sufficient premiums due.

Selected from predefined list

This Claim Hold Reason Code is invoked for Group Type 1 for claims with service dates after the Paid Through date found on GRUPB.

If the Group Type is either 2 or 3, the Claim Hold Reason Code here is carried over to the Claim Hold Reason Code found on each member’s MEMBR record.

This field is not used for Group Types 4 and 5.

Contact Name

R

Name of a contact person for this group.

Manually entered

Multiple contacts may be entered sequentially.

Country

O

The country in which the group is located.

Selected from predefined list

This field may be hidden, depending on how keywords UDATT and CNTRY (Country) are set up.

Date

C

The as of date for holding claims, if a Hold Reason has been entered.

Manually entered

Claims for members in this group will be held if the primary date of service on the claim is on or after the date entered here.

Due Date Rule

O

Used with the Calc Days field to calculate a due date that appears on premium bills.

Selected from predefined list

Select one of the following values:

  • <blank>: Results in a due date of the current system date.

  • Bill From: First date of the group’s billing period.

  • Bill Through: Last date of the group’s billing period.

  • Run Date: Due date equals the date the billing run is executed.

While a due date is calculated using this field + Calc Days, the resulting due date does not affect cash receipt delinquency processing.

If you are using Super Group/Administrator level groups, then DIAMOND calculates the premium payment due date (using this field and the Calc Days field) from the Supergroup or Administrator group record.

Email ID

 

The contact’s e-mail address.

Manually entered

If in the AL Dist. Meth. field you selected E-mail, then this e-mail address is used to send auto-letter correspondence. If E-mail is not selected in the AL Dist. Meth. field, then this field is for informational purposes.

For more information on auto letters, see Auto Letter, How DIAMOND Thinks.

Fax Number

O

Fax telephone number for the contact person.

Manually entered

If in the AL Dist. Meth. field you selected Fax, then this telephone number is used to fax auto-letter correspondence. If Fax is not selected in the AL Dist. Meth. field, then this field is for informational purposes.

For more information on auto letters, see Auto Letter, How DIAMOND Thinks.

Federal Tax ID

R

The Federal Tax ID for the Group.

Manually entered

 

Group ID

R

A unique identifier of the group.

Manually entered or selected from a predefined list

 

Group Type

R

An indicator that determines the type of premium billing records produced.

Selected from predefined list

Select one of the following values:

  • Annual Billing: Produces a single annual bill for each subscriber; customer is the group. Identified as Group Type 4 on the Lookup dialog box.

  • Combined: Produces separate invoices for each subscriber unit. Identified as Group Type 3 on the Lookup dialog box.

  • Group: Produces a single invoice for all members within a group. Identified as Group Type 1 on the Lookup dialog box.

  • Individual: Produces separate invoices for each subscriber unit. Identified as Group Type 2 on the Lookup dialog box.

  • Medicaid: Represents a group type for which there is no premium billing. Identified as Group Type 5 on the Lookup dialog box.

  • Medicare Risk: Represents a group type for which there is no premium billing. Identified as Group type 6 on the Lookup dialog box.

Hold Rsn

O

A value in this field will result in all claims entered for members associated with this group being put on hold prior to payment.

Selected from predefined list

Hold reason codes are maintained in keyword REASN (Reason) and have a type of HD.

Level

R

Identifies whether the record pertains to an administrator, super group, or group. Multiple groups can be associated with a single super group; multiple super groups can be associated with a single administrator.

Members can be associated at level group only.

Selected from predefined list

Select one of the following values:

  • Administrator

  • Group

  • Super Group

To set up a new group, work from the highest level down to the lower level, by entering the administrator first, then the super group, and finally the group.

Name 1 – 2

The complete name of the group.

Manually entered

 

National Employer ID

Unique employer group identification number.

Next OE End
