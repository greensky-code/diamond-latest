CDAPL Field Definitions

Field Name

Req

Description

Source

Comments

Allowed Reason

 

Reason Code to be added to a claim detail line when a pricing method other than standard provider pricing is used.

Selected from predefined list

Valid Allowed Reason Codes are maintained using keyword REASN.

Auth Price used with Submt Modifier - Action

O

Action to take on the claim line when the claim detail line includes a modifier code and authorization/claim detail linking and pricing applies to the line.

Selected from predefined list

Valid values are:

Hold – Hold the detail claim line when the Auth Amount on the auth detail record has been or will be exceeded with the claim detail record.

Deny – Deny the detail claim line when the Auth Amount on the auth detail record has been or will be exceeded with the claim detail line.

Auth Price used with Submt Modifier - Reason

O

Reason code to apply when the claim line is held or denied based on the Auth Price used with Submt Modifier - Action

Selected from predefined list

F5 lookup is available.

Auth Type

R

Authorization Type to which this rule applies

Selected from predefined list

F5 lookup is available. You define valid Auth Types using keyword AUTYP.

To see all CDAPL records for the LOB and Auth Type combination entered, click the Show Records button.

Description

Effective Date

Effective date of this rule

Manually entered

Future effective dates are allowed.

Effective Date (Processing Rules section)

Exceed Auth Detail Amount - Action

O

Action to take on the claim line when the Auth Amount on the auth detail record has been or will be exceeded with the claim detail line

Selected from predefined list

Valid values are:

Hold – Hold the detail claim line when the Auth Amount on the auth detail record has been or will be exceeded with the claim detail record.

Deny – Deny the detail claim line when the Auth Amount on the auth detail record has been or will be exceeded with the claim detail line.

Exceed Auth Detail Amount - Reason

O

Reason code to apply when the claim line is held or denied based on the Exceed Auth Detail Amount - Action

Selected from predefined list

F5 lookup is available.

Exceed Auth Detail Days/Visits - Action

O

Action to take on the claim line when processing of the claim detail line will result in exceeding the matching PRCDR authorization detail record authorized days or visits

Selected from predefined list

Valid values are:

Hold – Hold the detail claim line when the PRCDR procedure code days/visits have been exceeded.

Deny – Deny the detail claim line when the PRCDR procedure code days/visits have been exceeded.

Exceed Auth Detail Days/Visits - Reason

O

Reason code to apply when the claim line is held or denied based on the Exceed Auth Detail Days/Visits - Action

Selected from predefined list

F5 lookup is available.

Hold/Deny Reason

 

Depending on the Pay Action selected, the Not Covered Reason Code or Held Reason Code to be added to the claim detail line when this rule is applied.

 

Selected from predefined list

Used when the Pay Action is Price and Hold or Price and Deny.

Valid reason codes are maintained in keyword REASN. F5 lookup is available.

Include all Rng Rows

O

Indicates whether all the range rows (if there are more than one) on the PRCRA record should be evaluated on a whole range basis or on a single per range row basis

Selected with check box

Valid values are:

Y - Yes, look at all the ranges in the PRCRA record and if a submitted code falls within any of the ranges specified, it is a match to the authorized code

N - No, only if the submitted code falls with the same range row as the authorized code is the submitted code considered a match

LOB

R

Line of business to which this rule applies

Selected from predefined list

F5 lookup is available.

To see all CDAPL records for the LOB entered, click the Show Records button.

Match on Details for Inpt Stay

O

Specifies whether DIAMOND looks for authorizations with details when the claims entered are related to an authorized inpatient stay.

Selected with check box

Valid values:

Y

N

For more information see Authorization/Claim Detail Linking and Pricing, How DIAMOND Thinks, Claims Processing Logic section.

Pay Action

O

Action to take on claim detail lines when this rule is satisfied

Selected from predefined list

Valid values are:

1 - Price with Standard Provider Pricing

2 - Price with Authorized Price

3 - Price with Authorized Procedure

4 - Price and Deny with Standard Provider Pricing

5 - Price and Deny with Authorized Price

6 - Price and Deny with Authorized Procedure

7 - Price and Hold with Standard Provider Pricing

8 - Price and Hold with Authorized Price

9 - Price and Hold with Authorized Procedure

Process Order

R

Order of evaluation of the CDAPL rules

Manually entered

 

Range ID

O

Authorization Procedure Range ID

Selected from predefined list

If you do not select a range, then DIAMOND will only find an exact match on the submitted procedure code.

You enter Authorization Procedure Ranges in keyword PRCRA.

Rule ID

R

Identifier of this Claims Detail Authorization Procedure Link rule

Selected from predefined list

You enter Claims Detail Authorization Procedure Link rules using keyword CDAPR.

Term Date

O

Termination date of this rule

Manually entered

DIAMOND uses system parameter DUPAUTHRULE2 to determine whether the Expire Date or Discharge Date is used to match to the rule Term Date. For more information see PARAM user documentation for DUPAUTHRULE2.

Term Date (Processing Rules section)

O

Termination date of this Claims Detail Authorization Procedure Link rule

Manually entered

 

Tie-Breaker on Price

O

Specifies which of two or more matching in-range priced procedure codes to use to price the claim detail line, based on price.

Selected from predefined list

Valid values:

H - Use Highest Auth Price

L - Use Lowest Auth Price

 

Tie-Breaker on Auth Proc

O

Specifies which of two or more matching in-range priced procedure codes to use to price the claim detail line, based on procedure code.

Selected from predefined list

Valid values:

H - Use Highest Order Auth Procedure

L - Use Lowest Order Auth Procedure
