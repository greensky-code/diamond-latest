LBCDC Field Definitions

Field Name

Req

Description

Source

Comments

Claim Type

(View Existing Discount Calculation Rules)

D

Displays the claim type: institutional, professional, or dental, for this calculation rule.

Displayed.

This field displays information for any discount calculation records that already exist for the specified line of business.

Claim Type

(Create New Discount Calculation Rule)

R

Claim type that this discount rule applies to.

Selected from a predefined list.

 

Valid values are:

  • I – institutional

  • P – professional (default)

  • D – dental

Description

(View Existing Discount Calculation Rules)

D

Displays the description (if any) for this calculation rule.

Displayed.

This field displays information for any discount calculation records that already exist for the specified line of business.

Description

(Create New Discount Calculation Rule)

O

A description of this discount rule.

Manually entered.

Up to 60 alphanumeric characters.

Determinant Column

(View Existing Search Sequences)

D

Displays the determinant column used for the search sequence.

Displayed.

This field displays information for any search sequences that already exist for this discount rule.

Determinant Column

(Create New Search Sequence)

R

Data element to use as a determinant for this discount rule.

Selected from a predefined list.

The values available for this field are dependent upon the determinant table you select.

F5 lookup is available.

Determinant Table

(View Existing Search Sequences)

D

Displays the determinant table used for the search sequence.

Displayed.

This field displays information for any search sequences that already exist for this discount rule.

Determinant Table

(Create New Search Sequence)

R

DIAMOND table from which to select individual discount rule determinants.

Selected from a predefined list.

Depending on the claim type you selected for this discount rule, valid values may be:

  • DENTAL_CLAIM_HEADER

  • GROUP_MASTER

  • INST_CLAIM_HEADER

  • PROFSVC_CLAIM_HEADER

  • PROV_ADDRESS

  • PROV_CONTRACT

  • PROV_MASTER

F5 lookup is available.

Discount Days

R

The maximum number of days allowed for claim payment in order for the claim to be eligible for this discount rule.

Manually entered.

A positive number from 001 – 999.

To meet the criteria for this discount rule, the number of days from the claim’s receipt date to the check or EFT date must be less than or equal to the value entered in this field.

Effective Date

(View Existing Discount Calculation Rules)

D

Displays the effective date for this calculation rule.

Displayed.

This field displays information for any discount calculation records that already exist for the specified line of business.

Effective Date

(Create New Discount Calculation Rule)

R

Date this rule becomes effective.

Manually entered.

 

 

From Value

C

Determinant value or the beginning of a determinant value range.

 

This field is required if you do not define a secondary determinant for this search sequence; however, if you do select a secondary determinant for this search sequence, this field is not available.

Line of Business

R

Line of business code.

Displayed or manually entered.

F5 lookup is available.

 

Operator

C

Determines if DIAMOND searches for claims with a value that matches the determinant or for claims with values that do not match the determinant.

Selected from a predefined list.

Valid values are:

  • Equal to

  • Not equal to

Rate

R

The discount percentage rate to be applied against the claim’s net amount.

Manually entered.

Enter the percentage rate as a positive decimal; for example, .1 is 10 percent, .25 is 25 percent, and 1. is 100 percent.

Reason Code

O

Code used to explain reason for discount amount.

Manually entered.

Reason Code must correspond to Reason Code Type DI – Discount.

Rule Order

(View Existing Discount Calculation Rules)

D

Displays the rule order number assigned to this calculation rule.

Displayed.

This field displays information for any discount calculation records that already exist for the specified line of business.

Rule Order

(Create New Discount Calculation Rule)

R

Order in which this discount rule will be evaluated.

Manually entered.

A number from 001 – 999.

Each discount rule has an order number. When DIAMOND evaluates a claim to see if a discount calculation should be applied, it evaluates the rules by rule order number, lowest number first, highest number last.

Search Seq

(View Existing Search Sequences)

D

Displays the search sequence number

Displayed.

This field displays information for any search sequences that already exist for this discount rule.

Search Seq

(Create New Search Sequence)

R

A number that defines the order in which DIAMOND will evaluate this search sequence.

Manually entered.

A number from 001 – 999 that determines the order in which DIAMOND evaluates this search sequence. DIAMOND evaluates search sequences in numeric order starting with the lowest number.

Each search sequence must have at least one combination of determinant table and determinant column selected.

Sec Determinant Column

O

Data element to use as a secondary determinant for a search sequence.

Selected from a predefined list.

The values available for this field are dependent upon the secondary determinant table you select.

F5 lookup is available.

Sec Determinant Table

O

DIAMOND table from which to select a secondary determinant for a search sequence.

Selected from a predefined list.

Only one secondary determinant and table combination can be selected for each search sequence.

Depending on the claim type you selected for this discount rule, valid values may be:

  • DENTAL_CLAIM_HEADER

  • GROUP_MASTER

  • INST_CLAIM_HEADER

  • PROFSVC_CLAIM_HEADER

  • PROV_ADDRESS

  • PROV_CONTRACT

  • PROV_MASTER

F5 lookup is available.

Term Date

(View Existing Discount Calculation Rules)

D

Displays the termination date for this calculation rule.

Displayed.

This field displays information for any discount calculation records that already exist for the specified line of business.

Term Date

(Create New Discount Calculation Rule)

O

Date this rule is no longer in effect.

Manually entered.

Must be greater than or equal to the effective date.

Thru Value

O

Ending value of a determinant value range.

Manually entered.
