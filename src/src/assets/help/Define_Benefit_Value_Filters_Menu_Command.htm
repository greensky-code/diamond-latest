Description

D

The Description of the selected Benefit Detail.

Displayed

 

Filter Type

R

The type of filter for the selected Benefit Detail.

Manually entered

Hard coded values are:

  • Par

  • Auth Level

  • Rider

From Value

R

The single value or the first value in a range. The From Benefit Value Filter (either Par, Auth Level, or Rider) that will be included or excluded when matching a Benefit Rule to a claim.

 

Manually entered

The From value must be less than the Thru value.

For a single value, enter a From Value with no Thru Value. For a range of values, enter values in both the From Value and Thru Value fields.

Operator

R

Defines whether a value is included or excluded when the system compares Benefits Rules to a claim.

Manually entered

Hard coded values are:

  • I - Include

  • E - Exclude

Proc Order

D

The order in which the Benefit Rule will be evaluated.

Manually entered

 

Proc Seq

D

The sequence number of the selected Benefit Rule

Displayed

 

Thru Value

O

The last value in a range of values. The Thru Benefit Value Filter (either Par, Auth Level, or Rider) that will be included or excluded when matching a Benefit Rule to a claim.

 

The Thru Value must be greater than the From Value.
