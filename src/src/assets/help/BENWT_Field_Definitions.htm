Field Name

Req

Description

Source

Comments

Benefit Weighted Accumulator ID

R

Unique identifier of the Benefit Weighted Accumulator.

Manually entered

F5 lookup is available.

Description

O

Text description of the Benefit Weighted Accumulator.

Manually entered

 

Effective Date

R

The first date for which this procedure code range, primary procedure grouping, or secondary procedure grouping is effective.

Manually entered.

 

Primary Proc Grouping

   O

The primary procedure grouping.

Selected from a predefined list

F5 lookup is available.

You cannot enter a value in this field if you have entered a value in the Procedure Code From field for the same row.

You cannot use a procedure code in more than one row if the Effective Dates of the rows overlap.

Procedure Codes are stored on PROCD (Procedure Codes) screen.

Procedure Code From

O

A single procedure code or the first procedure code in a range of procedure codes.

Selected from a predefined list

F5 lookup is available.

You cannot enter a value in this field if you have entered a value in either of the Primary Proc Grouping or Second Proc Grouping fields for the same row.

You cannot use a procedure code in more than one row if the Effective Dates of the rows overlap.

Procedure Codes are stored on PROCD (Procedure Codes) screen.

 

Procedure Code Thru

O

The last procedure code in a range of procedure codes.

Selected from a predefined list

F5 lookup is available.

The Thru procedure code must be greater than the From procedure code.

You cannot enter a value in this field if you have entered a value in either of the Primary Proc Grouping or Second Proc Grouping fields for the same row.

You cannot use a procedure code in more than one row if the Effective Dates of the rows overlap.

Procedure Codes are stored on PROCD (Procedure Codes) screen.

Second Proc Grouping

O

The secondary procedure grouping.

Selected from a predefined list

F5 lookup is available.

You cannot enter a value in this field if you have entered a value in the Procedure Code From field for the same row.

You cannot use a procedure code in more than one row if the Effective Dates of the rows overlap.

Procedure Codes are stored on PROCD (Procedure Codes) screen.

Term Date

   O

The last date for which this procedure code range, primary procedure grouping, or secondary procedure grouping is effective.

Manually entered

If you leave the Term Date field blank, the procedure code range, primary procedure grouping, or secondary procedure grouping is considered to be infinitely in effect.

The Term Date must be greater than or equal to the Effective Date or blank.

Weighted Accum %

R

The weighted benefit accumulator percentage.

Manually entered

This percentage is used for accumulation of benefits if:

Benefit Rule Type is 20 (Limits) and Limit Type is Quantity,   Days, or Weighted Value.

Or

Benefit Rule Type is 80 (Full Coverage) and Limit Type is Quantity or Days.
