About REGIN

Use REGIN to add, change, and delete Geographic, Price, and Service Region codes. Geographic and Price Regions are used for pricing; Service regions are informational only. Also use REGIN to define ranges of zip codes for use with the Geo Zip pricing calculation method.

Geographic Region Code:

  • Is a required field on the Unit Value (UNITV) window and is used for "unit value" pricing to allow for varying procedure weights (relative values) by geographic area. See UNITV User Documentation for additional information.

  • Is optional on Group Detail (GRUPD) and Provider Contract (PROVC), and can be used to link the provider’s contract pricing detail to the group detail records.

Price Region Code:

  • Is primarily used for fee schedule and unit value pricing methods to vary flat amounts or conversion dollar rates by area.

  • Is a required field on the Procedure Price (PROCP) window and can be used to link PROCP records to Contract Pricing Detail (PROVC) and/or Conversion Factor (CONVF) records. It is optional on both PROVC and CONVF.

  • Is an optional field on the Provider Contracts-Contract Pricing Details (PROVC > Special Functions Menu > Contract Pricing Details) window and can be used to link Contract Pricing Detail (PROVC) records to PROCP and/or CONVF records.

Service Region Code:

  • Is used for "informational" purposes only (has no effect on pricing). You can use this code to categorize records for reporting purposes.

  • Is an optional field on the Provider Contracts-Contract Pricing Details (PROVC > Special Functions Menu > Contract Pricing Details) window and can be used to link Contract Pricing Detail (PROVC) records.

Geo Zip Calculation Method:

  • Is a calculation method that compares the provider's zip code from the claim header to the zip code ranges you define in REGIN to determine what Geo Zip region the rendering provider's zip code applies to.

  • Is selected in the Calculation Method field on PRULD (Price Rule Details) screen.
