Looking up data

In certain fields, you can display the Lookup dialog box to look up data stored in the system. If the Lookup dialog box is available for a field, then a message on the status bar displays "[F5] for lookup." The status bar is located on the bottom of the DIAMOND Client/Server System window.

For example, when you display keyword MEMBR (Member Master), you must enter a subscriber ID. To look up existing subscribers, you can press F5 to display the Lookup dialog box containing the column headings (field names) from the Member Master table. You can use the horizontal scroll bar to move across the column headings. You can also select the fields that display in the Lookup dialog box.

The Lookup dialog box also has a row of search option buttons used to perform your member search. See Figure 8.

Notice in Figure 8 that the Search button is active and appears raised. If you press ENTER or click the Search button at this point, all member records in subscriber ID/person number order display. See Figure 9.

You can scroll through the list of members looking for a last name (for example, Jones), click on the desired member to select the row, and then press ENTER. Jones’ member record in the Member Master window displays.

To look up data

  1. In the field where the Lookup dialog box is available, press F5.

  2. In the column you want to search data, type your search string.

  3. Click Search.

Finding values with an apostrophe

If you want to search for a value with an apostrophe ( ‘ ), then use the _ (underscore) wildcard character in place of the apostrophe. For more information, see About using wildcard characters to search for partial or matching values.

Finding values with a combination of upper and lower case letters

Use the Match All Cases check box to help you find values with a combination of upper and lower case letters. When the Match All Cases check box is selected, DIAMOND processing uses the following rules to find values matching your search string:

  • Exact matches to your search string are always found. For example, McGregor finds McGregor (an exact match).

  • The first letter of a search string can be upper or lower case. DIAMOND processing finds matching values with the same first letter in upper or lower case. For example, mcgregor finds mcgregor (lowercase first letter) and Mcgregor (uppercase first letter).

  • The case of the remaining letters in your search string is ignored and DIAMOND processing only finds matching values in ALL upper or all lower case (except the first letter). For example, mCgREGOR finds Mcgregor (except for first letter, all lowercase), mCgREGOR (exact match), MCGREGOR (all uppercase), and mcgregor (all lowercase). That is, except for exact matches or first letters, values with a combination of upper and lower case letters are not found, regardless of how you typed them. For example, neither mcgregor nor MCGREGOR will find McGregor. Instead, you may want to use a wildcard character in place of an uppercase letter. For more information, see About using wildcard characters to search for partial or matching values.

When the Match All Cases check box is cleared, only exact matching values to your search string display.
