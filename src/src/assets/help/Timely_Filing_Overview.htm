Setting Up Timely Filing Rules in DIAMOND

Timely Filing Processing Notes

Setting Up Timely Filing Rules in DIAMOND

Your organization may choose to encourage providers to submit claims within a certain period of time after the date of the procedure. DIAMOND automatically calculates the age of a claim. Based on criteria that you establish in keyword TFRUL (Timely Filing Rules), DIAMOND applies timely filing rules to each claim. These rules determine if DIAMOND should process a claim with a warning, hold, or deny the claim.

Unique Rule Identifier

Each timely filing rule must be identified by a unique combination of the following fields in keyword TFRUL:

  • Claim Type

  • Rule Order ID

  • Effective Date

For information on each field, see keyword TFRUL user documentation.

Rule Order ID

You determine the order in which DIAMOND searches through timely filing rules by assigning a Rule Order ID to each timely filing rule. DIAMOND evaluates each rule from lowest Rule Order ID number to highest number. Once DIAMOND finds a match, the search stops and DIAMOND uses that rule. You should assign lower numbers to specific rules and higher numbers to more general rules. DIAMOND displays the Rule Order ID in the processing messages assigned to the claim.

For information on the Rule Order ID field, see keyword TFRUL user documentation.

Timely Filing Criteria Menu Commands

TFRUL provides the following commands on the menu bar to include or exclude a single value or range of values:

  • Line of Business Criteria: include or exclude one or more lines of business.

  • Company Code Criteria: include or exclude one or more company codes.

  • Provider Status Criteria: include or exclude one or more provider par statuses.

  • Service State Criteria: include or exclude one or more service states.

  • Authorization Level Criteria: include or exclude one or more authorization levels.

For information on each command, see keyword TFRUL user documentation.

Timely Filing Rule Overrides

You cannot configure DIAMOND to automatically override a timely filing rule. You can manually override a timely filing rule that has been applied to a specific claim.

Timely Filing Processing Notes

This section includes details of how DIAMOND applies timely filing rules.

Calc From Date Field Logic

This value specifies the start date to use when evaluating the age of a claim. The options provided are:

Thru Date of Service: for Professional and Dental Services, uses the "thru" date of service from the claim detail line. For Institutional claims, DIAMOND checks the bill types master to determine which source to use:

  • If the bill type master is configured as an inpatient bill type, the thru date of service on the claim header is used.

  • If the bill type master is configured as an outpatient bill type, the thru date of service on each claim detail line is used.

From Date of Service: for Professional and Dental Services, uses the "from" date of service from the claim detail line. For Institutional claims, DIAMOND checks the bill types master to determine which source to use:

  • If the bill type master is configured as an inpatient bill type, the from date of service on the claim header is used.

  • If the bill type master is configured as an outpatient bill type, the from date of service on each claim detail line is used.

Contract End Date: uses the contract end date for the member’s group. DIAMOND evaluates the contract period that contains the thru date of service, depending on the claim type:

  • Professional dental, and institutional outpatient claims: thru date of service on claim detail line.

  • Institutional inpatient: thru date of service on claim header

Calc Thru Date Field Logic

This value specifies the thru date to use when evaluating the age of a claim. The options provided are:

  • Claim Received Date: uses the received date from the claim header as the calc thru date in the calculation of claim age.

  • Claim Entered Date: uses the system date from the claim as the calc thru date in the calculation of claim age.

Filing Limit Days Field Logic

DIAMOND uses the Filing Limit Days value to determine if the age of a claim is less than, equal to, or over the filing limit. This value comes from the Filing Limit Days field in one of two keywords:

  • PROVC. If you entered a value in this field, DIAMOND uses this field and ignores the value in keyword TFRUL. However, if Ignore Prov Limit Days in keyword TFRUL is checked, DIAMOND does not use the value in keyword PROVC.

  • TFRUL. If the Filing Limit Days field in PROVC is blank, or if Ignore Prov Limit Days in keyword TFRUL is checked, DIAMOND uses this field.

When DIAMOND evaluates a timely filing rule, it compares the Calc From and Calc Thru dates to calculate the age of the claim. DIAMOND then compares the age of the claim with the Filing Limit Days to determine if the claim is under or over the filing limit.

Service Delivery State Evaluation

  • DIAMOND evaluates the service state criteria for a timely filing rule based on the selected provider address, not the vendor address. If there is no state listed on the provider address, the rule evaluation fails and DIAMOND moves on to the next timely filing rule.

  • State names used in the Service State Criteria command must match the state names used in keyword PROVA (Provider Address). For example a timely filing rule applied to state "Calif" will not be triggered for a provider with "CA" in the State field in keyword PROVA.

Term Date Field in GRUPC (Group Contracts) and Timely Filing

The Term Date field in GRUPC is not required by DIAMOND. However, if you want to base a timely filing rule on the Term Date of the contract, this field must be filled in. If this field is left blank, the rule evaluation fails and DIAMOND moves on to the next timely filing rule.

Timely Filing Rule Re-evaluation

Some events in DIAMOND can trigger a new timely filing rule evaluation.

  • Claim re-adjudication: DIAMOND re-evaluates timely filing rules.

  • User invokes Change Header: DIAMOND re-evaluates timely filing rules.

  • Claim adjustment: no re-evaluation.

  • Claim reversal: no re-evaluation.
