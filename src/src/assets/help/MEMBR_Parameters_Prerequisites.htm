MEMBR Parameters & Prerequisites

The following is a brief listing and description of the parameters that affect this keyword. See PARAM Record Map for links to more information about each parameter. How DIAMOND Thinks documents may also give further details on how to configure these parameters for your particular needs.

 

Parameter

Description

Comments

DATEFORMAT

Determines the format for dates for all processing on the DIAMOND server.

Client processing date format is controlled by the Windows Control Panel.

Relative positions of month, day and year must be the same for both client and server formats.

Parameter 1:

Enter one of the following formats:

MM/DD/YYYY

DD/MM/YYYY

YYYY/MM/DD

This parameter is changeable at any time.

DFLTZIP

Determines whether City, State, and, if applicable, County are automatically generated based on the ZIP Code for keywords ALIAS, GROUP, MEMBA, MEMBR, MEMMC, PROVA, SUBMT, and VENDA.

Use ZIPRF to update the ZIP_CODES table.

This parameter should not be changed after implementation.

Parameter 1:

Y Fields are automatically populated using entries in ZIP_CODES.

N or blank: Fields are not automatically populated.

ELIGSTAT

Parameter 1 contains a list of codes which, when entered in the member’s Eligibility Status field, mean "eligible." Parameter 2 applies to claims. When a claim is processed for a member with an Eligibility Status code which is not listed in Parameter 1, DIAMOND changes the claim processing status to H, Held, and enters the hold reason code from Parameter 2 in the claim Hold Reason Code field.

This parameter should not be changed after implementation.

Warning: Premium Billing and Capitation runs ignore this member if the value in the EligSts field does not match the values entered in the ELIGSTAT parameter.

Parameter 1:

Enter a list of 1-digit codes, which mean "eligible."

Parameter 2:

Enter a valid Hold Reason code. If blank, or entry is not a valid Hold Reason code, then DIAMOND will change the claim processing status but leave the Hold Reason Code field blank.

IDCARDORDER

Determines the actions in MEMBR and EDI Member Module that trigger ID card production, the recipients of ID cards, and the number of cards produced for a family.

Parameter 1 tells DIAMOND how many ID cards to produce; Parameter 2 tells DIAMOND which actions trigger an ID card order

 

DIDSEARCH

Parameter used to set dual coverage DIAMOND ID functionality.

Once this parameter is set, a DIAMOND ID must be entered in the MEMBR screen for each new member.

‘Y’ enables dual coverage

‘N’ is the default parameter. If ‘N’ is set, then dual coverage is turned off.

For more information, see Dual Coverage - Overview.

MEMSITEVAL

Used to enable member Site Validation functionality.

If MEMSITEVAL Parameter 1 = No, keyword MEMBR field Site is unavailable.
