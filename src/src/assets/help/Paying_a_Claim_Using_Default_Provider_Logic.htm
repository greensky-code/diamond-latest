Paying a Claim Using Default Provider Logic

Restrictions

Instructions

Restrictions

None.

Instructions

Step

Comment

1. To use DIAMOND’s Default Provider Logic, for the data set up in this example (default provider "global" contract), the line of business associated with the claim must include Dflt Prov+Date Only in the LINBS Contract Search Criteria.

See How DIAMOND selects a contract when pricing a claim documentation.

2. Open a new PSCLM window and enter an out-of-area, non-participating, or one-time provider claim.

  1. Provider = NONSPECIF (Provider ID of the non-specific provider)

  2. Navigate to PSCLM>Special menu>Edit Vendor Info.

  3. Create and save (on-the-fly) a vendor name (VENDR) and vendor address (VENDA) record for the actual provider who submitted the claim.

  4. Complete the claim entry process.

DIAMOND uses Default Provider Logic to locate the default provider and the default provider contract. DIAMOND then prices and adjudicates the claim based on the default provider data.

If there is more than one vendor address on the default contract, DIAMOND will prompt you so select one, and will use this for pricing and adjudicating the claim – only.

DIAMOND issues payment to the vendor name and vendor address you created on-the-fly.
