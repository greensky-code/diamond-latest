About APUPD

Use keyword APUPD (Accounts Payable Update) to run an Accounts Payable (A/P) update job to process a batch of medical claims and capitation transactions.

Running an A/P update job creates A/P transaction records in keyword ACPAY (Accounts Payable) and selects either medical claims or capitation records for payment. At the same time the originating records in claims and capitation are finalized and, once finalized, cannot be changed. When a medical claim detail line is finalized, the Proc Status list is set to F – Final (posted) in the claim detail window. As part of the finalization of the original claims records a transaction number is generated and appropriate general ledger accounts are updated.

Running an A/P update job does the following:

  • Post and finalize specified claim detail line payables to accounts payable (A/P) in a batch mode.

  • Finalize capitation payables in A/P in a batch mode.

  • Assess claims for header holds.

  • Read parameters to determine how partial payment election affects how held claims are handled.

  • Read claims to determine whether Cap/Special Fund Distribution (CFDST), if used, has run.

  • Read the audit status of the claim to determine whether it requires auditing by third-party software.

  • Adds A/P transaction records in keyword ACPAY for claims (capitation A/P transaction records are created in the Capitation module, Post Capitation).

An A/P update job finalizes all claim detail lines and creates A/P transactions or finalizes capitation payables that satisfy the criteria you set up. The A/P update job then selects these A/P transactions for payment. If the run is for claims, then header holds, partial payment details, auditing by third-party software, and Cap/Special Fund Distribution are considered. For example, keyword APUPD only selects to pay claim detail lines if the APC Status box is blank or set to C.

A/P transactions are created for each finalized claim (institutional, professional services, or dental services) detail record with the Claim Status list set to one of the following values on the claim detail window:

  • P – Payable

  • A – Adjustment

  • D – Denied

  • F – Cap Fund (plus a Cap Fund Status of 3)

You can also run an A/P update job to finalize non-payable claim detail lines, which duplicates the functionality of keyword FINAL (Final Non-AP Setup). For more information about keyword FINAL, see FINAL (Final Non-Ap Setup).
