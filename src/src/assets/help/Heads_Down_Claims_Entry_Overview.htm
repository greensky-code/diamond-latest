Heads Down Claims Entry Overview

You use DIAMOND’s Heads Down Claims Entry function to manually enter minimal claims data using the Heads Down Claims Entry windows. The claims Header and Detail entry windows’ layout resembles the HCFA 1500 Claims Form layout to simplify the data entry process. In addition, the Lens Prescription entry window is used to enter optical lens prescription data.

The icon and executable file (hdsdwn.exe) and an initiation file (hdsdwn.ini) for the entry windows reside on your workstation. Double-click the Heads Down Claim Entry executable icon on your workstation’s desktop to display the entry windows.

Note:

DIAMOND displays the lens prescription entry window (after you tab out of the Header entry window) if the hdsdwn.ini PSCLMLENRX parameter = Y. DIAMOND does not display the lens prescription entry window if the hdsdwn.ini PSCLMLENRX parameter = N. Contact your Systems Administrator for additional information.

When you enter claims using these windows, the Heads Down Claims executable builds a claims text file on your workstation (you are not entering claims within the DIAMOND application). When you have finished entering all data, you name and save the text file to your workstation. You can also re-open and edit a claims text file that has been previously saved to your workstation.

Your Systems Administrator is responsible for collecting all claims text files from local workstations and merging them into one consolidated file. This consolidated file becomes the input (source) file for DIAMOND’s Intelligent Data Enhancer (IDE) module. The IDE output file becomes the source (control) file for DIAMOND’s Claims Batch Process. This batch process prices and adjudicates the claims. See Heads Down Claims Processing Examples Heads Down Claims Intelligent Data Enhancer (IDE), and Claims Batch Process — Process EDI (PREDI) for additional information.
