Holding Claims - Overview

You can hold an entire claim or individual claim detail line items. When you hold an entire claim, you prevent any detail lines on that claim from being finalized and selected for payment by Accounts Payable.

Holds that hold entire claims are also referred to as header holds since they are based on information on the claim header.

You can hold entire claims in four ways:

  • By using certain predefined claim holds available in several DIAMOND keywords

  • By using keyword CLHLD (Claim Hold Rules) to set up your own header Hold rules (user-defined header Hold rules)

  • By manually applying a header hold

  • By re-applying the CLHLD user-defined header hold rules using the Re-apply Claim Holds command on the Special menu in keywords INCLM or PSCLM

Some ways of holding an entire claim create a single hold reason code record for claim line #000 and do not affect the individual claim detail line items. Other ways of holding an entire claim do not create a hold reason code record for line #000, but instead create a separate hold reason code record for each individual claim detail line item and sets the Proc Status list for each line to H – Hold.

For information on holding entire claims using predefined rules, see Claim Holds: Sources and Effects.

For information on the latter three ways of holding entire claims, see the following:

  • Create a rule to hold an entire claim

  • Manually hold an entire claim

  • Re-apply a user-defined Hold rules to a claim

For information on how DIAMOND applies claim holds, see Claim Holds - How DIAMOND Thinks.
