Account Status

D

Indicates if user account’s password has been locked by the system.

System generated.

If Account Status is Locked, a DIAMOND user with security access to the system must unlock the password.

Claim Limit

C

Total claim amount, comprising the Determinant fields on all the detail line items, over which user is not authorized to process the claim.

Manually entered

You can type up to 16 digits before the decimal and 2 after the decimal.

Field is required if any fields in the Claim Limit Information section are completed.

Claim detail line items with the Claim Status list set to D – Denied or I – Informational and the Proc Status list is set to F – Final (posted), P – Paid, or N – Paid (no check) are not included in the accumulation toward this limit.

Important: When users with a claim limit access a claim submitted by EDI, and the amount is over their limit, the claim will be put on hold, even if there are no changes made to the claim, because the claim logic is reinvoked.

Claim Type

C

Type of claim to which the claim limit rules apply.

Selected from predefined list

Field is required if any fields in the Claim Limit Information section are completed.

Claim Types are maintained in keyword CODES.

Valid values are:

  • Institutional Claims

  • Professional Claims

Enter a separate claim limit line for each type of claim (that is, a user can have more than one entry in this table)

Determinant

C

Claim detail line field that will be added to determine if the Claim Limit has been exceeded.

Selected from predefined list

Field is required if any fields in the Claim Limit Information section are completed.

Security claim determinants are maintained in CODES with a type SECDETERMINANT.

Valid values are:

  • A - Allowed Amount

  • B - Billed Amount

  • N - Net Amount

Hold/Release

C

Directs DIAMOND to hold claims that exceed the Claim Limit. Hold reason can only be removed by a user with a claim limit exceeding the limit for this user.

Selected from predefined list

Field is required if any fields in the Claim Limit Information section are completed.

Valid values are:

  • Hold - DIAMOND will automatically hold a claim with the entry in Reason when the total of the detail lines, using the entry in the Determinant list, exceeds the Claim Limit entry.

  • Release - Future functionality

Reason Code

C

Hold reason code assigned to all claim detail lines that were added up to exceed the user’s claim limit.

Selected from predefined list

Field is required if any fields in the Claim Limit Information section are completed.

Security hold reason codes are maintained in keyword REASN with a type SC.

Row Level Security

R

Maximum record access/view security code available to this user. Applies to keywords PSCLM, INCLM, DNCLM, CLDSP, and DSPBN, only.

 

A digit from 0-9 may be entered.

Available for entry only if neither the SU Priv nor the Template check boxes are selected.

For example, if you enter 5, then users or templates only have view/change permissions on claims that have 5 or less in the Security Code field.

Template ID

O

If record pertains to an individual user, the security template that applies to this user.

Selected from predefined list

Available for entry only if neither the SU Priv nor the Template check boxes are selected.

Template IDs are maintained in keyword SUSER—they are User IDs with the Template check box selected.
