DRG Grouping and Pricing, How DIAMOND Thinks

You can have DIAMOND perform Diagnosis-Related Groups (DRGs) grouping and pricing in four ways:

  • Accept the submitted DRG code or enter a DRG code on the claim, and have DIAMOND perform DRG whole claim pricing

  • Accept the submitted DRG code or enter a DRG code on the claim, and use an external pricer to calculate the price for DRG whole claim pricing

  • Use an external grouper to determine the DRG code and have DIAMOND perform DRG whole claim pricing

  • Use an external grouper to determine the DRG code and an external pricer to calculate the price for DRG whole claim pricing

For more information on DIAMOND whole claim pricing using DRGs, see Pricing DRGs Without an External Pricer.

For external grouping and pricing, DIAMOND uses the HSS EASYGroupÔ 2000 Optimizer. The HSS Optimizer provides DIAMOND with access to the HSS-supported HCFA Diagnosis-Related Groups (DRG) Grouper and Pricer for online, batch, and API institutional claims processing. The HSS Optimizer determines which version of the HCFA Grouper to use based the claim primary date of service. The Optimizer also interfaces with the single HSS-supported DRG Pricer.

In order to run the HSS Optimizer, you must copy the HSS Optimizer, Grouper, and Pricer executables to the Daemon directory. For more information, see Setting Up the HSS Optimizer.

For more information on DRG grouping using the external grouper, see Grouping DRGs with an External Grouper

For more information on DRG pricing using the external pricer, see Pricing DRGs With an External Pricer.

Calculating the Claim Price (Allowed Amount) Using an External DRG Pricer

The data returned by the external HSS DRG Pricer includes:

  • Base Reimbursement Amount

  • Outlier Payments

  • Alternate Level Care Days Payments

  • Total Reimbursement (Base Reimbursement Amount + Outlier Payments + Alternate Level Care Days Payments)

  • Outlier Type

  • Average Length of Stay

  • Trauma Indicator

  • Congenital Malformation Indicator

  • Short LOS Outlier Trim

  • Long LOS Outlier Trim

  • Additional Mean LOS

  • DRG Weight

  • Cost Reduction Factor

  • Low Per Diem

  • Transfer Indicator

  • Payment Method Indicator

You can view these fields using INCLM Special menu command View DRG Pricer Fields.

When you perform DRG whole claim pricing, DIAMOND inserts the Total Reimbursement as the Allowed Amount on the system generated claim detail line.

HSS Optimizer Interface

A DIAMOND bidirectional interface operating in the background accesses the HSS Optimizer. The interface does the following:

  • Writes inpatient claims data files for patient records to be used by the Optimizer

  • Defines whether to group and/or price each record

  • Reads data returned by the Optimizer

  • Updates corresponding fields in DIAMOND

DIAMOND communicates with external software using the External Process Interface (EPI) Daemon. For more information, see The Daemon and EPI Utilities – Overview.

When Diagnosis Codes Are Longer Than Grouper and Pricer Accept

DIAMOND diagnosis code fields allow you to enter longer values than are permitted by the HSS Grouper and Pricer. When DIAMOND passes a longer diagnosis code than is permitted to one of these third party tools, the tool truncates the value to the permitted length and returns an invalid diagnosis code error.

Passing Newborn Age and Weight to the HSS Optimizer

The DIAMOND application passes the following newborn age and weight data to the HSS Optimizer:

  • Newborn Admit Age (in days)

  • Newborn Discharge Age (in days)

  • Newborn Weight (in grams)

These fields are included on the INCLM menu command UB92 Information.

The DIAMOND application passes these fields according to the following rules:

  • Newborn Admit Age

If you enter a valid age (in days), DIAMOND logic passes the age entered.

If you do not enter a valid age, DIAMOND logic determines what to pass based on the member age. First, DIAMOND logic retrieves the DOB from the member master record of the member entered on the claim header. The member age is then calculated from the member's DOB to the institutional claim header Admit Date. If the member age is less than one year old, the Newborn Admit Age is calculated and passed.   Newborn Admit Age is calculated as the number of days from the member's DOB to the institutional claim header Admit Date. If the member age is greater than or equal to one year, DIAMOND logic passes all 9's.

  • Newborn Discharge Age

If you enter a valid age (in days), DIAMOND logic passes the age entered.

If you do not enter a valid age, DIAMOND logic determines what to pass based on the member age. First, DIAMOND logic retrieves the DOB from the member master record of the member entered on the claim header. The member age is then calculated from the member's DOB to the institutional claim header Admit Date. If the member age is less than one year old, the Newborn Discharge Age is calculated and passed.   Newborn Discharge Age is calculated as the number of days from the member's DOB to the institutional claim header Discharge Date. If the member age is greater than or equal to one year, DIAMOND logic passes all 9's.

  • Newborn Weight

If you enter a valid weight (in grams), DIAMOND logic passes the weight entered.

If you do not enter a valid weight, DIAMOND logic passes all 9's.

Excluding Outpatient Bill Types from DRG Grouping

DRG grouping. During processing, DIAMOND uses the APCBILLTYPES system parameter to determine which bill types are excluded. To define these exclusions, enter all outpatient bill types you want to exclude in Parameters 2 and 3 of the APCBILLTYPE. Separate multiple bill types with a semicolon (;). Typically, bill types 130 through 139 are outpatient bill types.
