About CKPST

Use keyword CKPST (Check Posting) to run a check posting job to post, invalidate, or cancel a check printed by a check print job in keyword CKPRT (Check Print Setup).

You can also use keyword CKPST to post any flagged "P" EFT transactions, which remain in the EFT transaction workfile for exceptions to your customized EFT interface.

When you post checks in keyword CKPST, the following occurs:

  • The CKPRT job ID status is set to Posted

  • For the corresponding A/P transaction record, the Processing Status list is set to P – Paid in keyword ACPAY (Accounts Payable)

  • For the corresponding claim detail line, the Proc Status list is set to P – Paid in keyword DNCLM, INCLM, or PSCLM

  • Posted checks are available for lookup and reconciliation in keyword CHECK (Check Register)

EFT transactions not flagged "P" are deleted from the workfile.

To process checks produced outside the DIAMOND program, use keyword MCHEK (Manual Checks).
