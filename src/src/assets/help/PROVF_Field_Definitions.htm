Auth Class

O

A category of provider that is used for purposes of defining claim situations where the requirement for an authorization is waived.

Manually entered

Authorization waive rules are maintained in AWAVE.

Valid values are A-Z, 0-9.

If both an Admitting and Attending Provider is entered, the Attending Provider is used to determine the auth class value and whether the claim should be waived.

If either an Admitting or Attending Provider is entered, this entered provider is used to determine the auth class value and if the claim should be waived.

See Authorization Claim Links, How DIAMOND Thinks > Waiving the Authorization Requirement > Using the Authorization Class to Waive Authorizations for more information.

O

Indicator of provider’s status with board certification.

Selected from predefined list

Values are

Certified

Eligible

Date of Birth

Birth date of individual provider.

Manually entered

Default IPA ID

Default Vendor Addr

R

The address for the payee (vendor) which, unless an alternate address is selected, will be used.

Selected from predefined list

The vendor address can be selected during claims processing.

Default Vendor ID

R

The vendor ID which, unless a different vendor is selected, will be used as the payee for this provider.

Selected from predefined list

Other vendor IDs are associated with provider contracts (PROVC).

Short Name

R

An abbreviated name used during F5 lookups.

System generated but can be manually overridden

System generated scheme:

If provider is an institution, first 15 characters of the Full Name.

If provider is an individual, either the first 15 characters of Last Nm or the entire Last Nm + as many characters of the first name as necessary to equal 15 characters total.
