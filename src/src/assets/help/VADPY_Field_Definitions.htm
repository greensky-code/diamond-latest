Account Number

R

The unique, identifying, advance pay account number for the combination of vendor and vendor address.

Manually entered

F5 lookup is available.

You can type up to 12 alphanumeric characters.

To automatically generate a new account number, press TAB.

Chk/EFT Date

 

The date of the check or EFT.

Manually entered or system generated.

If you have paid the advance payment with a manual check, then type the date of the check.

If you will use keyword CKPRT (Check Print Setup) to print checks and keyword CKPST (Check Posting) to post checks, then this field automatically displays the check date when it is posted in keyword CKPST.

Chk/EFT Meth

D

Displays the method of check generation.

Displayed

One of the following values display:

  • A

The check was automatically created by DIAMOND processing.

  • M

The check was manually created.

Chk/EFT No

 

The number of the check or the EFT.

Manually entered or system generated.

If you have paid the advance payment with a manual check, then type the check number.

If you will use keyword CKPRT (Check Print Setup) to print checks and keyword CKPST (Check Posting) to post checks, then this field automatically displays the check number when it is posted in keyword CKPST.

Offset By

D

Displays the Offset By value from keyword VADPR.

Displayed

One of the following values display:

  • Vendor Only

Offsets against the advance payment are based on the vendor.

  • Vendor + Provider

The offsets will be any claim/capitation for that vendor and the provider entered in the Provider ID box in keyword VADPY (Vendor Advance Payment Account).

  • Vendor + Provider + Member

The offsets will be any claim that is for that vendor, the provider entered in the Provider ID box, and the member entered in the Member ID box in keyword VADPY.

  • Vendor + Member

The offsets will be any claim that is for that vendor and the member entered in the Member ID box in keyword VADPY.
