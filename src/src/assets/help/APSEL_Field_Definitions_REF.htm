APSEL Field Definitions

Field Name

Req

Description

Source

Comments

Action

D

Action taken by DIAMOND

System generated

Set to "run" because the request type is immediate. (‘Deferred’ is not available.)

Action Type

D

Select the action to take when this batch job is run.

Selected from predefined list

Valid values are:

  • De-select for Pay

De-selects the qualifying payables that were previously selected by running an A/P update job in keyword APUPD. A separate batch job is required to take this action.

  • Reselect for Pay

Reselects the qualifying payables that were previously de-selected in keyword APSEL.

Comments

O

Additional information about the batch job.

Manually entered

Comments are particularly useful on batch jobs that are saved as templates.

Company Code

O

The general ledger company associated with the payable transactions selected.

Selected from predefined list

Leave blank to update qualifying payables for all companies.

F5 lookup is available.

Date

D

Date the batch job is run.

System generated

 

From Due Date

O

Based on the due date in VENDR – Payment Delay Days.

Manually entered

If the vendor has a value in Payment Delay Days on the Vendor Master Record, the due date is calculated by adding that number of days to the Claim Received date.

If no value is in the Vendor Master, the system parameter CLAIMDUEDATE is read and the due date is calculated by adding that number of days to the Claim Received date.

If no value is in the parameter, the due date is the date of the APUPD run.

Enter in MMDDYYYY format. Leave this field blank to update all qualifying payables regardless of the due date.

From Inserted Date

O

The "insert date" of the accounts payable record.

Manually entered

Enter in MMDDYYYY format.

For medical (claims) this is the date the APUPD job ran which selected the payable (keyword APUPD creates the accounts payable records).

For capitation, this is the date CCALC was run (keyword CCALC creates the accounts payable records).

Job ID

R

Unique identifier for this batch job.

System generated

 

Payable Type

R

Type of payable to be processed.

Selected from predefined list

You can select one of the following values:

  • C – Capitation

  • M – Medical

Request Type

R

When the batch job is to be run.

Manually entered

Valid selections are:

  • Immediate: Indicates the job should run immediately when File > Run Job or CTRL+J is entered.

  • Deferred: Future functionality

Request User

D

The logon ID of the user entering the APUPD job request.

System generated

 

Status

D

The status of the batch job.

System generated

Valid values are:

  • New: This batch job is new. The batch job has been saved; however, the job has not been run.

  • Posted: Payables for this batch job have been posted to A/P.

  • Error: An error has occurred during a De-select or Reselect action.

  • Successful Run

Thru Due Date

O

Use this field if a "from due date" is specified.

Manually entered

Must be greater than the "from due date"

Enter in MMDDYYYY format.

Thru Inserted Date

O

The "insert date" of the accounts payable record.

Manually entered

Must be greater than the "From Date."

Enter in MMDDYYYY format.

For medical (claims) this is the date the APUPD job ran which selected the payable (keyword APUPD creates the accounts payable records).

For capitation, this is the date CCALC was run (keyword CCALC creates the accounts payable records).

Vendor ID

O

ID for the vendor receiving the payables

Manually entered

Leave blank to update qualifying payables for all companies.

F5 lookup is available.
