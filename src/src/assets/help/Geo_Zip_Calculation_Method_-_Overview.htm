Geo Zip Calculation Method - Overview

Geo Zip Calculation Method

You can use the Geo Zip calculation method to price professional, institutional (excluding Whole Claim Priced claims), and dental claims using a reimbursement methodology that is based on the rendering provider’s zip code from the claim header. The Geo Zip calculation method is available on the PRULD screen (Price Rule Details). When a provider’s contract is set to price with a Geo Zip calculation method, pricing logic accesses the zip code from the claim header, then compares this zip code to the REGIN (Pricing Regions) screen to determine which Geo Zip Region the rendering provider’s zip code applies to. This Geo Zip Region is then used to find the allowed amount on DIAMOND’s PROCP (Procedure Price) screen. Use PROCP to enter, store, view, and modify Geo Zip fee schedule allowed amounts.

The parameter, GEOZIPMATCH, is used to determine the length of the zip codes you enter when creating Geo Zip Code Regions in REGIN. You can set GEOZIPMATCH to any number from 1 - 5. As an example, if you set the GEOZIPMATCH parameter to 3, you can only enter 3 characters in the From and Through fields of the Geo Zip Code Regions box.

Changes you make to Geo Zip Code Region records (except changes to the description field) are written to AUDIT (Audit Trail).

Geo Zip Code Ranges

Each Geo Zip Code Region can contain several ranges of zip codes. Ranges of zip codes may overlap within one Geo Zip Code Region record but cannot overlap with zip code regions specified in other Geo Zip Code Region records.
