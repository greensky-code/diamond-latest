MCHEK Field Definitions

Field Name

Req

Description

Source

Comments

Action

R

Action that will be taken when Job Run menu command is selected from the File menu.

Selected from predefined list

Hard coded values are:

  • Select: Select payables that match the criteria for the Job ID.

  • Cancel: Cancel a Job ID with a status "Error."

  • Post: Update all claim, capitation, accounts payable, and check register records to reflect the issuance of the manual check or checks. Will include all Passes within the Job ID.

Amount

D

Running total amount of payables selected for payment.

Displayed

 

Balance to Item Count (check box)

D

If selected, number of payables chosen from selected payables must equal the number entered in Balance to Item Count.

Displayed

Automatically selected if parameter MCHEKBAL is activated.

Balance to Item Count (entry)

C

Total number of payables to be included in the manual check.

Manually entered

Available for entry if Balance to Item Count check box is selected.

Balance to Total (entry)

C

Total amount of payables to be included in the manual check.

Manually entered

Available for entry if Balance to Total check box is selected.

Balance to Total (check box)

D

If selected, total amount of payables chosen from selected payables must equal the number entered in Balance to Total.

Displayed

Automatically selected if parameter MCHEKBAL is activated.

Bank Acct

D

Bank Account Code for the Pay Type selected and associated with the Company Code selected.

Displayed

 

Cap Month

R

Capitation month from which payables will be selected for payment.

Manually entered

Enter in MM/YYYY format. The default is 0 (no capitation month selected).

Check Date

R

Date of the manual check.

Manually entered

Default is system date.

Check No.

R

Manual check number.

Manually entered

Must be a higher number than the last manual check entered.

Claim No.

O

Individual claim number from which detail lines are being selected for payment.

Selected from predefined list

If entered, must be consistent with Claim Type.

Claim Type

R

Type of claims from which payables are being selected for payment.

Selected from predefined list

Hard coded values are:

  • Institutional

  • Professional

  • Dental

Company Code

R

General ledger company associated with this set of selection criteria.

Selected from predefined list

Company Codes are maintained in keyword COMPF.

Unless the Manual Cap Pay Acct and Manual Med Pay Acct fields in COMPF have entries, manual checks cannot be processed respectively for capitation and medical (claim) payables.

Entity

O

Capitation entity for which payables will be selected for payment.

Selected from predefined list

Capitation entities are maintained in keyword CPENT.

From Due Date

O

Starting payable due date from which payables will be selected for payment.

Manually entered

Due dates are calculated according to Payment Delay Days in VENDR, or if blank, for claims only, the parameter CLAIMDUEDATE.

From Post Date

O

Starting payable post date, from APUPD, from which payables will be selected for payment.

Manually entered

Available as a criterion for claim payables only.

Include All

O

If checked, all payables displayed in the picklist will be included as paid by the manual check.

Selected from check box

A shortcut to selecting each payable one at a time.

Items

D

Running total of the number of payables selected for payment.

Displayed

 

Job ID

R

Unique identifier of a job.

System generated

This may be overwritten.

Member ID

O

Member for whom payables are being selected for payment.

Selected from predefined list

Member IDs are maintained in keyword MEMBR.

Model

O

Capitation model for which payables will be selected for payment.

Selected from predefined list

Capitation models are maintained in keyword CPMOD.

Pass

D

Unique identifier, within Job ID, of a set of selection criteria.

System generated

If no payables are found when a criteria selection set is processed, a pass number is not assigned.

Payable Type

R

Type of payables that are being paid with an external check for this Job ID.

Selected from predefined list

Hard coded values are:

  • Medical: Payables from PSCLM, DNCLM, and/or INCLM are eligible for selection.

  • Capitation: Capitation payables are eligible for selection.

Pool

O

Capitation pool for which payables will be selected for payment.

Selected from predefined list

Capitation pools are maintained in keyword CPOOL.

Provider ID

O

Provider for whom capitation or claim payables are being selected for payment.

Selected from predefined list

Providers are maintained in keyword PROVF.

Req User

D

User entering the job.

Displayed

 

Request Type

R

Indicator of when job should be run.

Selected from predefined list

Only valid value is Immediate. Value Deferred is reserved for future use.

Run Date

D

System date.

Displayed

 

Start Check No

R

Check number on the manual check.

Manually entered

 

Status

O

Accounts payable status from which payables will be selected for payment.

Selected from predefined list

Valid values are:

  • Selected: Claim or capitation payables that have been both posted and selected for payment using APUPD.

  • Unselected: Claim or capitation payables that have been posted only using APUPD.

  • (blank): Select both posted and selected and posted only payables.

Status (Job ID focus box)

D

Status of Job ID.

Displayed

 

Hard coded values are:

  • New: Job ID saved, payables not selected.

  • Selected: Payables selected, but not posted.

  • Posted: Payables selected and posted; claim or capitation payables, accounts payable records, and check register records have been updated.

  • Error: Run Job for this Job ID had errors, and processing terminated.

  • Canceled: A Job ID with a status "Error" was canceled.

Thru Due Date

O

Ending payable due date from which payables will be selected for payment.

Manually entered

Due dates are calculated according to Payment Delay Days in VENDR, or if blank, for claims only, the parameter CLAIMDUEDATE.

Thru Post Date

O

Ending payable post date, from APUPD, from which payables will be selected for payment.

Manually entered

Available as a criterion for claim payables only.

Vendor Address

R

Address of the payee, either vendor or subscriber.

Selected from predefined list

 

Vendor ID

R

Vendor for whom payables are being paid with the manual check.

Selected from predefined list

Vendor IDs are maintained in keyword VENDR. If a subscriber is being paid, subscriber must have been designated as the payee on the claims being paid. Subscriber vendor identification is the Subscirber ID prefixed with S.
