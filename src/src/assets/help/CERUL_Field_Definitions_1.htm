CERUL Field Definitions

Field Name

Req

Description

Source

Comments

Action

R

Defines the action to be taken against a claim detail line if the claim line matches the criteria defined in this claim evaluation rule.

Selected from a predefined list.

Valid values:

  • W - Warn

  • H – Hold

  • D - Deny

Benef Pkg

O

Determines if you want DIAMOND to use benefit package as a secondary match.

Selected from a predefined list.

Valid values:

  • A – All

  • E – Exact

  • N – None (default)

  • S – Some

The default is none (checkbox is left blank). If you place a check in this box, Exact is displayed. All or Some are displayed if you use the Additional Benefit Packages screen that is accessible from the Special menu.

Claim Type

R

Defines the claim type for this evaluation rule.

Selected from a predefined list.

Valid values:

  • Dental

Description

O

A description of this claim evaluation rule.

Manually entered.

Up to 60 alphanumeric characters.

Determinant Column

R

Data element to use as a determinant for the claim evaluation rule.

Selected from a predefined list.

The values available for this field are dependent upon the determinant table you select.

F5 lookup is available.

Determinant Table

R

DIAMOND table from which to choose individual claim evaluation rule determinants.

Selected from a predefined list.

Valid values are:

  • DENTAL_CLAIM_HEADER

  • DENTAL_CLAIM_DETAIL

F5 lookup is available.

From Value

R

Determinant value or the beginning of a determinant value range.

Manually entered.

 

Active

O

Clearing this checkbox makes this claim evaluation rule inactive; it won’t be evaluated during claim processing.

Manually entered.

Valid values:

  • Checked – the rule is active and will be evaluated.

  • Blank – the rule is not active and will not be evaluated.

This box is automatically checked when you enter a new rule.

Medical Definitions

O

Determines if you want DIAMOND to use medical definitions as an additional filter.

Manually entered.

Valid values:

  • None

  • All

  • Some

The default is none (checkbox is left blank), all other values come from the Additional Medical Definitions screen that is accessible from the Special menu.

Message to Operator

O

Text entered in this field is displayed if the claim being entered matches the criteria defined in this rule.

Manually entered.

 

Operator

R

Determines if DIAMOND searches for claims with a value that match the determinant or that don’t match the determinant.

Selected from a predefined list.

Valid values:

  • Include

  • Exclude

Primary

O

Determines how DIAMOND searches in dental claim history for a match.

Selected from a predefined list.

Valid values:

  • Tooth Number

  • Quadrant (does not consider specific tooth number)

  • Arch (does not consider specific tooth number)

  • None (no primary match selection applies)

Proc Code

O

Determines if you want DIAMOND to use procedure code as a secondary match.

Selected from a predefined list.

Valid values:

  • A – All

  • E – Exact

  • N – None (default)

  • S – Some

The default is none (checkbox is left blank). If you place a check in this box, Exact is displayed. All or Some are displayed if you use the Additional Procedure Codes screen that is accessible from the Special menu.

Prov

O

Determines if you want DIAMOND to use provider as a secondary match.

Selected from a predefined list.

Valid values:

  • A – All

  • E – Exact

  • N – None (default)

  • S – Some

The default is none (checkbox is left blank). If you place a check in this box, Exact is displayed. All or Some are displayed if you use the Additional Providers screen that is accessible from the Special menu.

Reason Code

O

The reason code that is associated with a hold action or a deny action.

Selected from a predefined list.

 

If the Action field for this rule is set to Hold, you can select an HD reason code, if the Action field is set to Deny, you can select a DN reason code.

F5 lookup is available.

No reason code is needed if the Action field is set to warn.

Rule Order ID

R

This number defines the order in which this rule is evaluated.

Manually entered.

Claim evaluation rules are evaluated starting with the lowest rule order number to the highest.

Sequence

R

A number that defines the order in which DIAMOND will evaluate this search sequence.

Manually entered.

Each claim evaluation rule can have several search sequences.

A number from 0001 - 9999 that determines the order in which DIAMOND evaluates this search sequence. DIAMOND evaluates search sequences in numeric order starting with the lowest number.

Each search sequence must have at least one combination of determinant table and determinant column selected.

Thru Value

O

Ending value of a determinant value range.

Manually entered.

 

Timeframe

R

Define the time period DIAMOND will use when it considers how far back in claims history to search when evaluating this rule.

Selected from a predefined list.

Valid values:

  • Y – calendar year

  • C – contract period

  • E – elapsed year

  • L – lifetime

  • D – elapsed days

Years/Days

R

Specify the number of years or days, depending on the value you entered into the Timeframe field.

Manually entered.

A number from 000 – 999.

If, in the timeframe field you specified:

  • Calendar year or elapsed year, this field will default to years

  • Elapsed Days, this field will default to days

  • Contract Period, this field defaults to number of periods

  • Lifetime, this field is not available
