MEMBB Field Definitions

Field Name  Req Description Source Comments

Eligibility information identifying individual family members

System generated based on Subscriber ID.

 

(Eligibility data)

S

Each eligibility period for the member selected above.

System generated based on individual family member.

 

Billed Through

D

A system calculated date indicating the span of coverage for which the member has been billed.

System calculated and displayed if the subscriber’s group’s Type is either Individual or Combined.

When a new subscriber record is added and a date entered in Billing Begin, this field automatically displays the day before the subscriber’s eligibility effective date.

When a new subscriber record is added and the associated group has a common billing date, this field automatically displays the day before the common billing date entry.

Thereafter, this field is automatically updated with the last day of the last premium billing that was posted for the subscriber unit.

Billing Begin

O

For subscribers in Groups with Group Type of Individual or Combined, the billing start date which applies to the entire family.

Manually entered.

This field is protected if a Common Billing Date is entered in keyword GRUPB or if the Group Type is Group.

For the subscriber’s first eligibility period DIAMOND populates the Billing Begin date with the subscriber’s effective date.

The month and day of the Billing Begin date can be used to determine when new premium rates should apply.

DIAMOND ID

O

Unique Identifying Number

There is one identifier for every instance of a person in the Member Master records

Manually entered

A DIAMOND ID enables dual coverage in DIAMOND.

If this person has more than one insurance coverage, the other instances of the person appear in a popup and the appropriate one can be selected from the list.

For more information, see Dual Coverage, How DIAMOND Thinks.

Ovr Amount

O

A dollar amount, which will be treated according to the Override Code above.

Manually entered.

You can type 16 digits before the decimal and 2 digits after the decimal.

Ovr Code

O

A code describing how the override premium amount is treated arithmetically.

Selected from a predefined list

Valid values are:

A  Add to calculated plan premium

R Use the override amount in place of the calculated premium

Ovr Step

O

Associated rate step to be overridden

 

 

Paid Through

D

Automatically updated date from Cash Receipts Entry when the subscriber unit’s posted payment amount is over the threshold percentage for the billing period.

System displayed.

As a result of the ARCSH threshold calculation process, this field may be updated and the Claim Hold Code on the member’s record assigned to all claims with dates of service after the Paid Through date.

Subscriber ID

R

Unique identifier of a subscriber unit (family).

Selected from a predefined list

 

Use EFT

O

If checked, it indicates that the member pays premiums through Electronic Funds Transfer.

Manually checked.

This field must be manually maintained when a member is reinstated.
