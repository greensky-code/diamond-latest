PROCD Field Definitions

Field Name

Req

Description

Source

Comments

Allowed Reason

O

Allowed reason code associated with this procedure.

Selected from predefined list

Allowed reasons are maintained in keyword REASN with a type of AL.

Arch Req

O

If checked, makes the Arch field on DNCLM, PRCDR, THIST, and TRULE a required field for this procedure.

Manually entered

 

Asterisk Procedure

R

Indicator that procedure is an asterisk procedure as defined in CPT.

Manually entered

Asterisk procedures are surgical procedures that do not include pre- and post-operative services.

Code Class

O

Category of procedures; when class is A, claims processing prompts for anesthesia time.

Selected from predefined list

Valid values are:

S Surgical

P Per Diem

O Observation

A Anesthesia

Count As Days

R

Indicator that the procedure should be counted as a day for authorization and benefit rule accumulations and for whole claim pricing calculations.

Selected from check box

 

Effective Date

O

First date this procedure is valid.

Manually entered

 

Follow Up Days

O

The number of follow up days allowed for the specified procedure code

Manually entered

 

From Age

O

Youngest age at which a member could appropriately receive the service represented by this procedure code.

Manually entered

An error message displays when the member’s age is not consistent with this entry.

For information on how DIAMOND uses the PROCD From Age and To Age to allow or not allow a procedure based on the member’s age, see Procedures and Pricing HDT>Allowing Procedures Based on Member’s Age documentation.

Full Description

O

Complete description of the code.

Manually entered

 

Hold Date

O

First date of claims entry on which the Hold Reason will be applied. If no date is entered, Hold Reason will be applied as soon as this PROCD record has been saved.

Manually entered

 

Hold Reason

O

Reason code that will be entered on the claim detail line when this procedure is used.

Selected from predefined list

Hold reasons are maintained in keyword REASN with a type of HD.

ID Qualifier

R

Unique product identification qualifier.

Selected from predefined list

F5 lookup available.

Hard coded values are:

CJ Current Procedure Term (CPT)

ER Workers Comp Jurisdiction Codes

HC CPT and HCPCS Codes

IV Home Infusion Codes (HEIC)

NU UP92 Revenue Codes

RB UB82 Revenue Codes

ZZ Mutually Defined Codes

User defined values are maintained in keyword CODES.

Inst. Auth Detail Link

O

If selected, indicates that this procedure code records is included in Institutional claims authorization/claim detail linking and pricing.

Selected from check box

 

Oral Cav Req

O

If checked, makes the Oral Cavity field on DNCLM, PRCDR, THIST, and TRULE a required field for this procedure.

Manually entered

 

Patient Gender

O

Gender for whom this service is appropriate.

Selected from predefined list

An error message is displayed when the member’s gender is not consistent with this entry.

Price Indicator

O

Indicates that when this procedure is entered, INCLM should use the alternate procedure to price the detail line rather than the primary procedure.

Selected from check box

 

Primary Grouping

O

Implementation specific category of procedures.

Manually entered

 

Procedure Code

R

Unique code representing a medical service.

Manually entered

Typically a CPT, HCPCS, ASA, or UB92 code; ICD surgical codes are maintained in keyword DIAGN.

Quadrant Req

O

If checked, makes the Quadrant field on DNCLM, PRCDR, THIST, and TRULE a required field for this procedure.

Manually entered

 

Secondary Grouping

O

Implementation specific category of procedures.

Manually entered

 

Short Description

O

Short text description displayed automatically when the code is used in claims and authorizations.

Manually entered

 

Surface Req

O

If checked, makes the Surface field on DNCLM, PRCDR, THIST, and TRULE a required field for this procedure.

Manually entered

 

Target Revenue Code

O

Indicates that alternate CPT/HCPCS codes have been entered in menu command Procedure Determinant.

Selected from check box

When this procedure (typically a UB92 revenue code) is entered and this indicator is checked, INCLM requires alternate procedure(s) be entered.

Invocation of this feature and its results are defined in the provider’s PROVC Detail record.

Term Date

O

Last date this procedure is valid.

Manually entered

 

To Age

O

Oldest age at which a member could appropriately receive the service represented by this procedure code.

Manually entered

An error message is displayed when the member’s age is not consistent with this entry.

For information on how DIAMOND uses the PROCD From Age and To Age to allow or not allow a procedure based on the member’s age, see Procedures and Pricing HDT>Allowing Procedures Based on Member’s Age documentation.

Tooth # Req

O

If checked, makes the Tooth # field on DNCLM, PRCDR, THIST, and TRULE a required field for this procedure.

Manually entered

 

User Date 1 and 2

O

Use of these fields is determined by your organization

Manually entered

The literals for these fields are defined in keyword UDTXT (User Defined Fields).

User Defined 1 and 2

O

Use of these fields is determined by your organization

Manually entered

The literals for these fields are defined in keyword UDTXT (User Defined Fields).

Weighted Value

O

The number of points that are assigned to this procedure.

Manually entered

A number between 000 – 999.

As an example, you can use the weighted value field to assign points to different x-ray procedures, then use the Limit type benefit rule to set a maximum for the number of x-ray points allowed for a one year period.
