CKPRT Field Definitions

Field Name

Req

Description

Source

Comments

Action

R

Action taken for this check print job.

Selected from predefined list

Hard coded values are:

  • Calc: Calculates transactions; one amount for each vendor within vendor address within back account within company code.

  • Print: Prints checks, one for each vendor/vendor address combination.

  • Cancel: Cancels the job.

Bank Acct

D

Bank account associated with the Company Code entered.

Displayed

Bank Accounts are maintained in keyword BANKA (Bank Account).

Calc Admin Fee

R

Indicates whether administrative fees are included in the check print job.

Selected from predefined list

You can select one of the following values:

  • E – Exclude Admin Fee

Exclude administrative fees when running a check print job.

  • I – Include Admin Fee

Include administrative fees when running a check print job.

  • O – Only Admin Fee

Only process administrative fees when running a check print job. Other payables are ignored.

Text that will be printed on the check.

Date

System date.

Displayed

 

From Due Date

O

Starting date in the range of due dates for payables calculations.

Manually entered

If the Vendor has a value in Payment Delay Days on the Vendor Master Record, the due date is calculated by adding that number of days to the Claim Received date.

If no value is in the Vendor Master, the system parameter CLAIMDUEDATE is read and the due date is calculated by adding that number of days to the Claim Received date.

If no value is in the parameter, the due date is the date of the APUPD run.

Enter in MMDDYY format. Leave this field blank to update all qualifying payables regardless of the due date.

Start Check No.

R

Next available check number in the sequence when checks are printed. The first valid check will be numbered with the sum of the Start Check No. + the number of alignment checks.

Displayed

DIAMOND will display the next available check for the Bank Account; may be overridden.

The number of alignment checks is determined manually during Action, Print.

This field is required for printed checks and EFT runs.

Manually entered number must be higher than the last check in the check register, CHECK.

Status

D

Current status of the check print job.

Displayed

Hard coded values are:

  • New: New job has been saved, but no check print/EFT records have been calculated.

  • Calculated: Check print/EFT records have been calculated for the batch job but haven’t been printed.

  • Printed: Checks have been printed for the batch job but have not been posted.

  • Posted: Check/EFT transactions for the batch job have been posted; accounts payable, claim detail, and check register is updated.

  • Error: An error occurred during check print/EFT transaction record calculation or during check printing. Job must be canceled and a new job submitted for the same action.

  • Canceled: This check print job was canceled and must be re-run.

Important - You can change the data entered in the Item Selection Criteria section box only if the batch job status is New.

Sub/Dep Minimum

D

Minimum amount for which a check will be written to a subscriber or dependent.

Displayed

Selecting the Minimum Override check box will allow you to type over the value in this field. You can type up to 16 digits before the decimal and 2 digits after the decimal.

Sub/Dep Minimum is entered in keyword COMPF. Subscriber or dependent payments that are less than the Sub/Dep Minimum will be held until the next CKPRT run to which they apply.

If no value is in the Vendor Master, the system parameter CLAIMDUEDATE is read and the due date is calculated by adding that number of days to the Claim Received date.

If no value is in the parameter, the due date is the date of the APUPD run.

Enter in MMDDYY format. Leave this field blank to update all qualifying payables regardless of the due date.

Thru Post Date
