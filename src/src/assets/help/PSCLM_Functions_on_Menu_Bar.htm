PSCLM Commands on the Menu Bar

 

Command             Menu   Description

Adjust Interest/
Penalty/Discount Amount

Details

S

Use this command to manually apply a discount, penalty, or interest amount and associated reason code to an adjusted claim detail line.

Adjust Line

S

Creates a new claim detail line for entering data to adjust an existing finalized or unfinalized line.

Administrative Fee Info

S

Displays the administrative fee, if charged to the health plan, for a claim detail line.

Allowed Factor

S

Enters a percentage which will be applied to the calculated allowed amount; otherwise, 100% of the calculated allowed amount is used.

Anesthesia Time

S

Automatically prices anesthesia services based on time data.

Attach EOB/RA Remarks

S

Add or delete EOB/RA remarks to the claim header or detail line.

Audit Display

Details

W

Displays audit trail records for the selected record.

Authorization Match/Waive

S

Includes the following functions:

  • Manual Waive: Manually waives requirement for an authorization. Populates the Method field with M.

  • Undo Waive: Undoes a manual or automatic waiver of the requirement for an authorization. May result in claim requiring re-pricing or re-adjudication.

  • Reapply Waive: Reapplies the rules from AWAVE to an unfinalized claim. Required when new AWAVE rules have been added since the claim was initially entered.

  • Reapply Match: Reapplies rules for automatic matching of an authorization to the claim.

  • Reapply Match Waive: Reapplies both automatic matching of an authorization to the claim and the AWAVE rules.

Change Default Batch

S

Changes the default received date and batch number of the claim currently entered, and all claims subsequently entered in the batch.

Change Header

S

Enables you to change any of these claim header fields on unfinalized claims:

Svc Date; Thru Date; Member ID/Person Number; provider ID; Provider Address; Provider Zip code; Vendor ID; Vendor Address.

For detailed information including restrictions and affects, see Change Header Menu Function - Overview.

Claim Audit

S

Triggers detail line editing by GMIS ClaimCheck.

Claim Audit Details

Details

S

For the claim and claim line selected in the keyword, displays summary information for:

  • Claim detail lines that were impacted by the CodeReview evaluation

  • Claim detail lines that caused CodeReview to change the claim.

Claim Detail Indicators

Details

S

Stores additional data associated with the detail line.

Claim Lookup

S

Alternative to pressing F5 to search for a claim.

Claims Interest/
Penalty/Discount Information

Details

S

Displays the claim’s interest calculation, penalty calculation, and discount calculation information.

Enables you to enter an override for the calculation date used in interest calculations and to override the number of unclean days used in interest calculations.

COB Information

Details

S

Allows override of the OC Carrier Code, COB Allowed Amt rule and COB Allowed Amt Rsn. Only available if the COB calculation method selected on the corresponding group detail record (GRUPD) is Non-Duplication of Benefits or Integration of Benefits.

Covering Provider Information

Details

S

Displays and/or changes data that governs how a provider of service is reimbursed when serving as a covering provider for the member’s primary care provider.

Delete

 

E

Deletes the ACPAY record and the benefit accumulator records associated with the claim line.

Edit Vendor Info

S

Changes or adds vendor (VENDR) and vendor address (VENDA) information.

Hidden Fields

S

Displays the following additional data associated with the detail line:

Auth Detail: The authorization detail number.

Hidden User 1-2: Reserved for future use.

Hold Reason Display

Details

W

Displays hold reasons and associated data (user, last update, source, etc.) applied to the claim.

Hold Reasons

Details

T

Use to enter a manual hold at either the header or detail level, or to view, change, delete, or mark as historical an existing hold. Displays all header and detail line holds, whether generated automatically or entered manually.

When you add and save a hold reason using this command, DIAMOND automatically changes the Proc Status on the line to Held. However, in order to remove any corresponding benefit accumulator records, you must manually save the line while in the claim Detail File window.

Lens Prescription

Details

T

Stores lens prescription data.

Measurements Information

Details

S

Stores measurement values and associated process type.

NDC Codes

Details

S

Use to add or change a National Drug Code (NDC) to this detail line item.

Notes

Details

N

This command is only available if the PSCLM window has been associated with at least one note type in NOTEW.

Takes you to the Notes Entry window where you can view a list of notes or add, view, or edit a note detail.

For information on setting up notes functionality, see NOTES, How DIAMOND Thinks; NOTET, Note Type; and NOTEW, Note Window user documentation.

Original Procedure

S

Stores the Billed Procedure, Billed Modifier, and Billed Allowed Amount, in cases when the submitted code has been changed for purposes of adjudication. For example when an obsolete procedure code has been billed.

Other Amounts Information

S

Records the amount paid by the patient or subscriber, and displays the balance due calculated by subtracting the patient paid amount from the total billed amount.

Other Claim Info

Details

T

Records additional information related to the claim but not directly required for benefit adjudication or pricing.

Override Cap Fund Dates

S

Overrides the requirement that the date of service for a claim detail line fall within the Cap Fund Month in order to be included in the next CFDST run.

Patient Liabilities

Details

T

Updates financial liabilities of the member after a claim detail line has been adjudicated.

Pay Subscriber

S

Use to pay the subscriber instead of the provider. If the claim is finalized, then this command is not available.

Pay Dependent

S

Use to pay the dependent of a subscriber instead of the provider. If the claim is finalized, then this command is not available.

Payment Detail

Details

S

Displays detailed payment information about a selected claim line.

Pricing and Adjudication

S

Prices and/or adjudicates claims or claim detail lines; with the following options:

  • Price Detail Line: Prices an unfinalized detail line (any processing status other than P- Paid, F-Final/Posted, or N-Paid/No Check).

  • Price Claim: Prices all unfinalized detail lines on a claim.

  • Adjudicate Detail Line: Adjudicates an unfinalized claim detail line.

  • Adjudicate Claim: Adjudicates all unfinalized detail lines on a claim.

  • Price and Adjudicate Detail Line: Prices and adjudicates an unfinalized claim detail line.

  • Price and Adjudicate Claim: Prices and adjudicates all unfinalized detail lines on a claim.

Using this command overwrites any manual pricing or adjudication.

Print Flag Info

S

Displays the current print flag and status for the claim and allows for a manual override from a claim detail line. Status selections are "A" for Automatic and "M" for Manual. The "M" status will not change, unless the automatic print flag evaluation is reinvoked by using the "Auto" button on this window.

Processing Messages

Details

W

Displays header and detail processing messages generated automatically by DIAMOND.

Reapply Claim Holds

S

Reapplies header holds according to the rules entered in CLHLD; marks existing holds as historical and applies CLHLD holds again.

Reset Audit Status

S

Resets the claim header Audit Status to Y, Yes. Resets the claim detail lines Audit Status as follows:

  • If the claim detail line Audit Status is C, Complete, set it to Y, Yes

  • If the claim detail line Audit Status is N, No, leave it as N, No

  • If the claim detail line Audit Status is S, set it to Y, Yes

  • If the claim detail line Audit Status is E, set it to Y, Yes

  • If the claim detail line Audit Status is A, Added Line, delete the line

You can only use Reset Audit Status if parameter CAUDTOOL is 1, ClaimCheck, the existing claim header Audit Status is C, Completed, and no claim detail line has a Processing Status of F - Final (posted), P - Paid, N - Paid (no check), or C - Check issued (paid).

Reset Claim Indicator

S

Changes current setting of the Unclean Indicator from No to Yes or from Yes to No. This command is not available if the claim line has been finalized.

This command is not available for adjusted claim lines.

For information on the Unclean Indicator, see Claims Discount and Interest Calculations, How DIAMOND Thinks.

Reverse Line

S

Creates a new claim detail line that contains all the data of an existing line, finalized or unfinalized, with all quantity and dollar amounts reversed (for example, if the original dollar amount is positive, the reversed amount is negative).

Spinal Manipulation Information

Details

S

Stores Spinal Manipulation condition codes and descriptions associated with claim.

Submitted Authorization

S

Displays the authorization or secondary authorization number originally entered on the claim and the authorization or secondary authorization that was subsequently matched to the claim.

Submitted Claim Detail Data

Details

S

Enables you to enter or view data elements that were submitted with invalid data.

Submitted Claim Information

S

Includes the following functions:

  • Claim Header Information: Enables you to enter or view data elements that were submitted with invalid data. Details

  • Provider/Vendor Information: Enables you to enter or view data elements that were submitted with invalid data. Details

View Administrative Fee Info

S

Use to view the administrative fee details, if any, for the displayed claim.

View Claim Totals

S

Displays a list of all claim totals from the Claim Detail sections from INCLM or PSCLM. See INCLM or PSCLM for information about the Claim Detail totals displayed here.

See Billed Minus Allowed Amount, How DIAMOND Thinks for information about the Billed Minus Allowed total in this window.

Note: if DIAMOND displays the error message "70894: Totals for a claim that is whole claim priced, cap fund or capitated may not be accurate," this indicates that, depending on the claim status of each claim line, the net amount displayed in the Claims Display Totals may not reflect the amount paid in the check run.

View Claims with Auth Number

S

Displays a list of other professional services and institutional claims with the same authorization number on the claim header as the current claim. You can select a claim on the list and view it in an active Claims Display (CLDSP) window. To sort the list, click on a column header.

Use this command any time after you enter an authorization number on the claim header. The current claim is not included in the list.

View COB History

S

Displays the member’s COB history information.

A dependent member with other coverage may have their own COB history record(s) or may be associated with their subscriber’s COB history using the COBHS Other Family Coverage all/include/exclude logic.

If the dependent has their own COBHS records, DIAMOND displays those records. If the dependent’s other coverage is associated with a subscriber COB history record, you cannot view it under the member’s person number. You can review the other family members’ COBHS records by scrolling through the picklist to view all COBHS records to locate an all, include or exclude COBHS record.

View IPA Info

S

Displays the member’s associated IPA contract information, including Delegated Responsibilities, Special Programs, Service Area, and the Postal Codes included/excluded in this Service Area. Available from both the claim header and the claim detail screens.
