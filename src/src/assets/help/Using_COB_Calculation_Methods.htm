Using COB Calculation Methods

Restrictions

Instructions

Frequently Asked Questions

You can have DIAMOND automatically calculate COB for each claim line on a claim using a COB calculation method. You select COB calculation methods for an employer group for a specific effective period on the Group Detail (GRUPD) record. You can establish the same or different COB calculation methods for institutional claims, dental services claims, and professional services claims. DIAMOND automatically applies COB calculation methods only if the claim is automatically adjudicated by DIAMOND, not if the claim is manually adjudicated. (Manual adjudication is when you manually enter an amount in one or more patient liability fields.)

Restrictions

  • To apply a Coordination of Benefits calculation method during claims adjudication, you must set the group to adjudicate claims. If your arrangement with the employer group calls for adjudication, set the GRUPD field Group Req = A - Adjud. (If the GRUPD field Group Req is set to P - Price, DIAMOND does not automatically adjudicate the claim and hence does not perform COB processing.)

  • Only one COB calculation method can be used to process and adjudicate a claim detail line.

Instructions

Step

Comment

1. Review the employer group agreement and determine COB-related specifications.

  • Claims adjudication by DIAMOND

  • COB calculation methods for professional services and institutional claims

  • Dependent determination rule

  • Inclusion/exclusion of capitated claims

  • Amount to which to apply logic to calculate patient liability amounts

  • Whether or not to adjust benefit accumulators for COB (Integration of Benefits and Non-Duplication of Benefits only)

 

2. Enter a COB history record for the member’s additional coverage. If the required Carrier Code doesn’t exist, enter a Carrier record including Carrier Code and Carrier Type (Medicare, Commercial, Other) using CARCD.

COBHS records can be established for Medicare or Commercial carriers. For COB processing, DIAMOND considers all carriers as commercial except Medicare.

3. On the desired group detail (GRUPD) record

  • Select a Professional COB Calculation method, Dental COB Calculation method, and Institutional COB Calculation method as needed

  • If the employer group agreement stipulates excluding capitated claims, in the Exclude Capitated Claims field select Both, Institutional or Professional as appropriate.

  • If the employer group agreement stipulates a dependent determination rule, select the rule in the dependent Determination rule drop-down list.

On the Group detail record, the same or different COB calculation methods can be selected for Institutional Claims and Professional claims.

See Coordination of Benefits – How DIAMOND Thinks for more information on how these elements are used in COB processing.

4. On the benefit package record (BENEF) for the package selected in the GRUPD Benef Pkg field, select a value in the Apply Patient Liability To field per your employer group agreement.

Tells DIAMOND what value to use when calculating patient liability amounts when performing automatic adjudication using COB calculation methods.

5. (Integration of Benefits and Non-Duplication of Benefits only) On the corresponding BENEF detail records, specify whether or not to adjust benefit accumulators in the field Adjust.

Tells DIAMOND whether to adjust benefit accumulators when performing automatic adjudication using COB calculation methods Integration of Benefits or Non-Duplication of Benefits. For more information on how DIAMOND adjusts benefit accumulators, see Coordination of Benefits – How DIAMOND Thinks for examples of these COB calculation methods.

6. (Integration of Benefits and Non-Duplication of Benefits only) If you have provider contract agreements to use a Commercial COB Allowed amount rule, enter the Commercial COB Allowed amount rule and COB Allowed Reason code on the corresponding provider contract records using PROVC.

A Commercial COB allowed amount rule stipulates whether to use the DIAMOND allowed amount or the lesser of the DIAMOND Allowed Amount or the OC Allowed Amount when applying COB for these two methods during adjudication.

The rule and reason code default to claims where these PROVC records apply.

If no rule is specified, DIAMOND uses the DIAMOND Allowed Amount.

If you need to override the Commercial COB Allowed Amount rule and reason code defaulted for a particular claim, you may do so in INCLM, DNCLM, or PSCLM. From the claim header click COB Info from the Special menu, select the correct rule in the Com COB Allowed field and the correct reason code in COB Allowed Rsn field, and save the header.

Frequently Asked Questions

  • The COB calculation method I selected is not being applied during claims adjudication. What should I do?

  • Check if the GRUPD field Group Req = A - Adjudication. If not and your agreement with the employer group calls for adjudication, correct this field value.

  • If the claim is a capitated claim, check if corresponding Group detail record is set to exclude this type of capitated claim. That is, check if the GRUPD field Exclude Capitated Claims is set to exclude Both, Institutional or Professional claims. If so and your agreement with the employer group allows you to include this type of capitated claim in COB, change this field value accordingly and reprocess the claim.

  • Verify that there is other primary coverage for the effective dates of the claim. Check for primary COB History (COBHS) records for the member with effective dates corresponding to the claim detail.
