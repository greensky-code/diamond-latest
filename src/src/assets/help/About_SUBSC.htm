About SUBSC

Use keyword SUBSC (Subscriber Update) to update consolidated eligibility history for each subscriber/family to include membership transactions that have occurred since the last premium billing run. In addition, this keyword creates an historic "snapshot" of the subscriber/family eligibility as of each premium billing run. (Premium bills are calculated based on the subscriber, or family, unit.) DIAMOND processing uses the consolidated eligibility history (subscriber_snapshot table) to calculate current and retroactive premium bills.

Important

When you run a premium billing job in keyword GPBIL (Premium Billing Setup), an SUBSC job automatically runs (before the premium billing job) if there are outstanding membership transactions that have not been included in a previous premium billing run.
