Hold Payment Flag

O

If selected, payments to this vendor are calculated but not released.

Selected from check box

If you select the Hold Payment Flag check box, DIAMOND prevents the accounts payable records for this vendor from being included in a check.

DIAMOND does not create hold reason records nor change the Processing Status to Held on the claim detail lines, nor is this condition explicitly reflected on the corresponding accounts payable records.

For more information, see Claim Holds, An Introduction and Claim Holds, How DIAMOND Thinks. For links to additional holds information, see keyword CLHLD (Claim Hold Rules).

IRS Tax ID

O

The unique IRS ID number of the vendor.

Manually entered

IRS Tax IDs are maintained in keyword VENTX (Vendor TIN).

This is the number used to assemble and report 1099 totals. Multiple vendors may share a tax ID.

National Provider ID

O

Unique National Provider ID as established by the Health Care Financing Administration.

Vendor ID

R

The unique identifier of the vendor.

Manually entered or system assigned.

You cannot add a Vendor ID prefixed with "S."

A Vendor ID prefixed with "S" indicates a system-generated vendor created to make payments to a subscriber.

A Vendor ID prefixed with "SD" indicates a system-generated vendor created to make payments to the dependent of a subscriber.

Vendor Type

R

A category of vendor.

Selected from predefined list

Hard coded entries are:

  • Agent

  • Medical

  • Other
