Entering Heads Down Claims

If desired, for verification purposes, process a test claim using keyword PSCLM. Retain the processed test claim for comparison against a similar claim to be processed using DIAMOND’s Heads Down Claims Entry process.

Step

Comment

1. Run the Heads Down Claims executable (hdsdwn.exe). The executable is included in DIAMOND’s directory on the client (PC) workstation. You can create a short-cut to the executable and put it on your desk top.

Perot Systems Healthare recommends that you save all claims text files to local workstation hard drives rather than to a network drive. This prevents possible uppercase/lowercase problems when moving (FTP) the file to UNIX.

2. Use the Heads Down Claims Entry window to create a batch (text file) of claims on your workstation. See Heads Down Claims Entry Windows for additional information.

 

3. Save your Heads Down Claims text file on your workstation’s hard drive in an appropriate directory. Save the file with a .dat extension (<filename>.dat).

 

Moving Claims Text File to UNIX

The UNIX directories, necessary to process Heads Down Claims and run the Intelligent Data Enhancer (IDE), are created when you install the current release of DIAMOND.

You must use FTP to move the Heads Down Claims text file (<filename>.dat) from your workstation to a UNIX directory. You cannot click and drag the file using Windows Explorer.

Step

Comment

1. Open a DOS window.

 

2. At the DOS prompt, change to the workstation directory where <filename>.dat claims text file is stored.

 

3. Prompt = DOS dir prompt>

Enter = ftp <host name>

4. Prompt = username:

Enter = <host username>

5. Prompt = password:

Enter = <host password>

6. Prompt = ftp>

Enter = cd <host directory containing IDE script and control file>

7. Prompt = ftp>

Enter = put <filename>.dat

8. Prompt = ftp>

Enter = quit

You have moved your claims text file (<filename>.dat) to the UNIX directory containing the IDE script (ideexec.sh) and the IDE control file (hdsdwn.ctl).

When you run the IDE script, <filename>.dat, ideexec.sh, and hdsdwn.ctl must be in the same directory.

Connecting to Host

Step

Comment

1. Use TNVTPlus or Tiny Term to connect to the host.

 

2. Prompt = username:

Enter = <host username>

3. Prompt = password:

Enter = <host password>

Running the IDE Script

You must run the IDE script from within the directory in which it is stored. In addition, the <filename>.dat and hdsdwn.ctl files must be in the same directory as the IDE script (ideexec.sh).

Step

Comment

1. To run the IDE script from the host

Enter = ideexec.sh

2. Prompt = username:

Enter = <DIAMOND 950CS username>

3. Prompt = password:

Enter = <DIAMOND 950CS password>

4. Select the Heads Down Claims control file (hdsdwn.ctl).

If you are entering lens prescription data, you must also select the Lens Prescription control file (hdlenrx.ctl).

5. Enter the name of the claims text file.

Enter = <filename>.dat

Enter the <filename> only. Do not enter a full path.

6. A "script completion" message is displayed.

The output of the IDE script is a new data file named D_<filename> (where <filename> is the same as the <filename>.dat claims text file). The D_<filename> file is the source (input) file for DIAMOND’s Claims Batch Process.

You must process each D_<filename> file completely through DIAMOND’s Claims Batch Process. You cannot concatenate separate D_<filename> files and attempt to use the concatenated file as the source (input) for the Claims Batch Process.

Running Claims Batch

Step

Comment

1. Make sure DIAMOND’s EPI Daemon is running for your environment.

 

2. Use DIAMOND’s Process EDI (PREDI) function to run the Claims Batch Process.

 

3. PREDI field = Submitter ID

Select = any Submitted ID.

4. PREDI field = File Type:

Select = Profsvc Claims

5. PREDI field = Input File Name;

Enter = /<path>/D_<filename>

You must enter the full path to D_<filename>.

6. PREDI field = Conversion Program:

Enter = psclmn01.ctl (unpriced claims)

7. PREDI field = Next Action:

Click = Receive

8. Press CTRL+J (Run Job) to log the job.

Wait for job log action to complete.

9. PREDI field = Next Action:

Click = Edit

10. PREDI field = Job Run Options:

Click = Pricing and/or Adjudication (as required).

11. Press CTRL+J (Run Job) to run the Claims Batch Process job.

After the job runs, check the PREDI Claims Critical Error Report and the Claims Detail Reports to verify that all claims have been processed.. This report sorts all errors by error category.

12.  If no "critical error" claims are reported and no error correction editing is necessary, go to Step 15.

 

12. If "critical error" claims are reported, use DIAMOND’s Work Table Edit (EDIED) function to edit the claim record(s). You can go directly to a specific claim using EDIED, Special menu > Go to Claim. See EDIED User Documentation for additional information.

 

 

13. After editing the claim record(s) access the PREDI function, select the appropriate job record and click on the PREDI — Re-Edit option button.

 

14. Press CTRL+J (Run Job) to re-run the Claims Batch Process.

Re-check the PREDI Claims Critical Error and the Claims Detail Reports to verify that all claims have been processed.

15. If all claims have been processed, use the PREDI function to post the Claims Batch Process job.

 PREDI field = Job ID

Select = the job that processed successfully.

16. PREDI field = Next Action

Click = Post

17. Press CTRL+J (Run Job) to post the Claims Batch Process job transactions.
