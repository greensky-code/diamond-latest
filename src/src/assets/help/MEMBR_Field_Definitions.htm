MEMBR Field Definitions

Field Name  Req Description Source Comments

Addr 1

O

First line of the member’s address.

Manually entered

You can type up to 55 alphanumeric characters.

Addr 2

O

Second line of the member’s address.

Manually entered

You can type up to 55 alphanumeric characters.

Addr Type

D

Type of member address.

Displayed

Valid value:

  • R1 Member Residential/Primary Address

This field always displays a value of R1. To add other types of addresses, use keyword MEMBA (Member Address).

Alt Memb ID

O

Second identification number that links two different, valid eligibility records together when they reference the same person.

Manually entered

There must be a reciprocal entry in the member’s other record.

A common use of this field is to link two different records for a single individual who has dual coverage, however, as of Release 5.1, dual coverage functionality can be implemented by the use of DIAMOND IDs. For more information, see Dual Coverage, How DIAMOND Thinks.

Ben Start Date

O

Entered on the subscriber’s record only, but can pertain to accumulations of benefits for the family.

Manually entered

This field, in combination with entries in keyword BRULE (Benefit Rule), overrides DIAMOND’s use of the group’s contract effective date (in keyword GRUPC). For more information, see Benefit Rules and Packages, How DIAMOND Thinks.

For more information on benefit start dates, see Notes on Benefit Start Date.

Bus Ph

O

Member’s business telephone number.

Manually entered

 

Case Mgmt

O

An informational indicator of whether the member requires clinical case management.

Selected from check box

This field is for informational purposes only.

City

O

Member’s city.

Manually entered

Depending upon the values in keywords ZIPRF, UDATT, and the DFLTZIP parameter in keyword PARAM, this field automatically is filled based on the ZIP Code you entered.

COB Ind

O

Indicates whether one or more COB History (COBHS) records exist for this member

System generated

Valid values:

  • <Selected> - If one or more COB History records exist for this member (no date comparison is done)

  • <Clear> - If no COBHS records exist for this member

Cannot be used in any determinant based logic. For example, COB Order assignment rules (COBOR) or COB Hold/Deny rules (COBHD).

DIAMOND selects the COB Indicator only on the MEMBR record of the individual member specified on the COBHS record, even if you select a COBHS Other Family Coverage field value of Include, Exclude, or All.

Country

O

Member’s country.

Selected from predefined list

Depending on the setup in UDATT, this field may not be displayed.

CS Ind

O

Customer Service indicator noting member has confidential or sensitive information.

Selected from predefined list

Valid Values:

CO - Confidential

VP - V.I.P

Date of Death

O

Member's date of death

Manually entered

Unavailable if Termination Date is blank. Else, optional.

If a Termination Date is entered, when you enter a date of death DIAMOND logic validates it according to the LINBS Date of Death Rule.

Dept

O

A code representing the department where a member works.

Selected from predefined list

Department codes are maintained in keyword CODES (System Codes Maintenance) with a type MELIGDEP.

DIAMOND ID

O

Unique Identifying Number

If you use this feature, then there is one identifier for every instance of a person in the Member Master records.

Manually entered

A DIAMOND ID enables dual coverage.

If this person is covered by more than one insurance, then the other instances of the person appear in a popup window and the appropriate one can be selected from the list.

For more information, see Dual Coverage, How DIAMOND Thinks.

Dist. Meth.

O

Displays the preferred method of auto letter distribution.

Selected from predefined list

Select the member’s preferred method of receiving Auto Letter correspondence: printed, faxed, or sent in e-mail.

For more information on auto letters, see Auto Letters, How DIAMOND Thinks.

DOB

R

Member’s date of birth.

Manually entered month, day, century and year.

 

Dual Cov

R

Indicates dual coverage, (whether there is more than one Member Master record for this person)

System generated

When this box is checked, it indicates dual coverage.

For more information, see Dual Coverage, How DIAMOND Thinks.

Eff Date

R

The first date on which the member is eligible for the coverage described in the eligibility section.

Manually entered

DIAMOND does not allow overlapping eligibility periods.

EligSts

R

Indicator of whether the member is eligible during the period described in this eligibility record.

System generated for new members; can be overridden or, for existing members, entered.

If a new member record, then DIAMOND defaults to the first character of the ELIGSTAT record in keyword PARAM.

If the entry in this field matches an entry in ELIGSTAT parameter, then the member is eligible. If the entry is not found in ELIGSTAT, then the member is ineligible.

Warning: Premium Billing and Capitation runs ignore this member if the value in this field does not match the values entered in the ELIGSTAT parameter.

Email ID

C

The member’s e-mail address.

Manually entered

If in the Dist.Meth list you selected E-mail, then this e-mail address is used to send auto-letter correspondence. If E-mail is not selected in the Dist.Meth list, then this field is for informational purposes.

For more information on auto letters, see Auto Letters, How DIAMOND Thinks.

EmpNo

O

Member’s employee identification number.

Manually entered

 

Emp Stat

O

Member’s employment status

Selected from predefined list

Select the status of the member’s employment from the following list:

  • 1 - Employed full time

  • 2 - Employed part time

  • 3 - Not employed

  • 4 - Self employed

  • 5 - Retired

  • 6 - On active military duty

  • 7 - Terminated

  • 9 - Unknown

  • N - None

Valid values are maintained in EMPSTAT in the CODES keyword.

Fax Number

C

The member’s fax telephone number.

Manually entered

If in the Dist.Meth list you selected Fax, then this telephone number is used to fax auto-letter correspondence. If Fax is not selected in the Dist.Meth list, then this field is for informational purposes.

For more information on auto letters, see Auto Letters, How DIAMOND Thinks.

First

R

The member’s first name.

Manually entered, case sensitive.

 

Gender

R

Member’s gender.

Selected from predefined list

Valid values:

  • F Female

  • M Male

  • U Unknown

Group ID

R

The group with which this member is identified or enrolled.

Manually entered

F5 lookup is available.

Hire Date

O

The member’s date of hire.

Manually entered

 

Home Ph

O

Member’s home telephone number.

Manually entered

 

IPA

O

The Independent Practice Association with whom the member is enrolled.

Selected from predefined list

 

Lang

O

Member’s language.

Selected from predefined list

 

Last Nm

The member’s last name.

Manually entered

You can type up to 35 alphanumeric characters. This box is case-sensitive.

LOB

O

The line of business associated with the group/plan in which the member is enrolled.

Displayed

 

Location

O

A code representing the location of the member.

Selected from predefined list

Location codes are maintained in keyword CODES, Sys Code Type MELIGLOC.

Marital Sts

O

Member’s marital status.

Selected from predefined list

Valid values:

  • D Divorced

  • M Married

  • O Other

  • S Single

  • W Widowed

MCareSts

O

The member’s Medicare status.

Manually entered

Formatted according to the Medicare program format for each position as follows:

  • State Code

  • County Code

  • Medicaid

  • Disability

  • Hospice

  • Institution

  • ESRD

  • Working aged

 

MedRec

Manually entered

 

Medicaid

Manually entered

 

Medicare

Member’s Medicare number.

MI

Manually entered

 

Other Sts

O

A second status code.

Manually entered

This field is not edited.

Panel ID

O

A code associated with a panel of providers that are associated with the group in which the member is enrolled.

Selected from predefined list

 

PCP Change Reason

C

The reason code for this PCP change.

Selected from predefined list

 

PEC End Date

O

Pre-Existing Condition End Date.

Type the last day of the pre-existing condition restriction.

Manually entered

It is only necessary to enter a date in this box when the member’s pre-existing condition end date is different than the group end date.

PEC Waived

O

Pre-Existing Condition Waived.

Click to select if the pre-existing condition restriction is waived.

Selected from check box

When PEC Waived is selected, DIAMOND skips all pre-existing condition (PEC) logic, including all member and group PEC logic.

PCP ID

C

The ID of the member’s primary care provider.

Selected from valid provider IDs.

This field is required if the line of business displayed so requires; otherwise, it’s optional.

DIAMOND performs a provider contract search, and, if Provider Member Enrollment functionality is activated, evaluates the provider’s enrollment rules (PROVE).

Person No

R

The unique identifier of an individual member, in combination with the subscriber ID.

Manually entered

The person number of the subscriber is initially 01 (may change later in certain membership circumstances); other members of the family are numbered 02-99.

Plan Code

R

A code representing a benefit plan available for this group and member.

Manually entered

F5 lookup is available.

Privacy On

O

Designates whether the member prohibits release of personal health information to outside entities.

Selected from check box when creating a new member. Otherwise, display only.

If checked, member’s information may not be released to outside organizations.

For more information, see About the Privacy Indicator

Prov2

O

The ID of a secondary provider associated with the member, such as an OB/GYN for a female adult member.

Selected from predefined list

 

Rate Type

An optional code to indicate a specific rate type to be used during premium billing rate calculation.

Selected from a predefined list after user defined values are set up using CODES.

Rate Type codes are maintained in CODES. Examples of user defined rate types might be:

Rate0

Rate1

Rate2

Rate3

For more information, see the following:

  • Premium Billing, How DIAMOND Thinks, the section Member-by-Member (Whole Family) Family Billing.

  • Rate Matrix Examples, the section Member-by-Member Family Rates by Member Age/Rate Type

Reason

C

Hold reason code to be applied to the claim detail line when the PEC end date is different than the group end date

Selected from predefined list

Required if there is a date entered in the PEC End Date field.

Reason codes are maintained in keyword REASN (Reason).

Riders

Codes which represent add-on benefits available for this group, typically dental, pharmacy, vision and the like.

Manually entered

F5 lookup is available.

Salary

The member’s salary.

Manually entered

Site

O

Unique identifier of the site (facility).

Selected from predefined list

Unavailable if parameter MEMSITEVAL Parameter 1 = No or if the corresponding line of business record Site check box is clear.

Required if the corresponding line of business Site check box is selected.

F5 lookup is available.

For more information, see Using Site Validation and Site Validation, How DIAMOND Thinks.

The title by which the member is addressed.

Subs Person No

Subscriber’s person number.

Displayed

Subscriber ID

R

The unique identifier of a subscriber and subscriber unit (family).

Manually entered or automatically generated for a new subscriber.

You can have DIAMOND automatically generate a Subscriber ID for a new subscriber. For information on activating this option, see Automatic Subscriber ID Generation, How DIAMOND Thinks.

Term Date

O

The last date on which the member is eligible for the coverage

Manually entered

Date must be greater than the Start Date.

DIAMOND does not allow overlapping eligibility periods.

Term Rsn

O

A code that explains the reason for the termination.
