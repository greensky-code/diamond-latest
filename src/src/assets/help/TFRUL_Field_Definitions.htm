TFRUL Field Definitions

Field Name

Req

Description

Source

Comments

Action

R

Action to apply to claim that does not meet timely filing rule.

Selected from a predefined list.

Available values and restrictions:

Warn: must enter text in Message to Operator field.

Hold: Reason Code is required. Can enter text in Message to Operator field.

Deny: Reason Code is required. Can enter text in Message to Operator field.

Authorization Level

D

Authorization levels (all, some, none) covered by the timely filing rule.

System displayed.

Default value is None. You can set this value using the Authorization Level Criteria command. You can create new provider par statuses using the AUTHLEVEL code in CODES.

Calc From Date

R

Specifies start date to use when evaluating the age of a claim.

Selected from a predefined list.

Available values:

Thru Date of Service (default)

From Date of Service

Group Contract End Date

Calc Thru Date

R

Specifies calc through date to use when evaluating the age of a claim.

Selected from a predefined list.

Available values:

Claim Received Date (default)

Claim Entered Date

Claim Type

R

Type of claim for selecting timely filing rule.

Manually entered.

Values are populated from CLAIMTYPE code in CODES. Hard-coded values:

P (Professional)

I (Institutional)

D (Dental)

Company Code

D

Company codes (all, some, none) covered by the timely filing rule.

System displayed.

Default value is None. Set this value using the Company Code Criteria command.

Description

O

Description of the rule.

Manually entered.

Optional field, limited to 60 characters.

Effective Date

O

Effective date of the rule.

Manually entered.

 

Filing Limit Days

R

Maximum number of days.

Manually entered.

Value used when determining if the claim age is less than, equal to, or greater than the filing limit.

This value is overridden by the Filing Limit Days field in PROVC if the following conditions are true

The Filing Limit Days field in PROVC has a value (is not empty).

Ignore Prov Limit Days is not checked.

Ignore Prov Limit Days

O

Prevents Provider Contract Override from occurring for timely filing rule.

Selected by check box.

 

Line of Business

D

Lines of business (all, some, none) covered by the timely filing rule.

System displayed.

Default value is None.

Set this value using the Line of Business Criteria command.

Message to Operator

O

Text to display on claims screen (INCLM, PSCLM, DNCLM) if timely filing rule is invoked.

Manually entered.

Required if Action field is Warn. Optional if action is Hold or Deny.

Field is limited to no more than 60 characters.

Provider Par Status

D

Provider par statuses covered by the timely filing rule.

System displayed.

Default value is None. You can set this value using the Provider Status Criteria command. You can create new provider par statuses using the PARSTATUS code in CODES.

Reason Code

O

Reason Code to apply to a claim with a Hold or Deny action.

Selected from a predefined list.

Required if Action field is Hold or Deny. Only Reason Codes corresponding to the Action field will be available.

Rule Order ID

O

Determines order in which rules are evaluated.

Manually entered.

May be up to 4 numeric characters, from 0001 to 9999.

Service Delivery State

D

Service delivery states (all, some, none) covered by the timely filing rule.

System displayed.

Default value is None.

Set this value using the Service Delivery State Criteria command. Note that the state in this field must match exactly the state value in PROVA.

Term Date

O

Date that rule is terminated.

Manually entered.

Optional field; only required if rule has a specific duration.
