Finding a matching provider contract and site address when adding a provider address

When you add a provider address that includes a Site Code, DIAMOND logic validates that a matching provider contract and site address exist for the Provider ID and Site Code.

Specifically, when you press F5 in the Site field and then click Search, DIAMOND Site Validation logic performs the following steps:

1. Find all provider contracts for the entered Provider ID.

2. Compare all site address records to the provider contracts. Find any site addresses that match a provider contract by satisfying both of the following criteria:

  • Line of business, IPA, and Panel of the Site Code Master record for the Site Code on the site address record match the provider contract

  • Effective periods of the site address and provider contract overlap

3. Display all matching site address records in the Lookup window.

If a single match or multiple matches are found, the DIAMOND application displays the matching site addresses and you can select the desired address.

When you select one of these site addresses, DIAMOND logic displays the Site Code in the Site field and copies Name 1, Cert, address, office hours, and contact information from the site address record to the provider address record. These fields are unavailable after being so populated. Subsequent changes to these fields in keyword SITEA automatically update the keyword PROVA fields.

If no match is found, an error message is generated.

Entering a Site code on a provider enrollment rule

You can enter a Site code when adding a provider enrollment rule in two ways:

  • In the Site field, type a Site code

  • In the Site field, press F5 for lookup

If you type a Site code, DIAMOND logic validates that the Site code entered satisfies Site Validation criteria. If Site Validation criteria are met, DIAMOND logic accepts the Site code, and displays the Panel, IPA, site name, Certification and Address.

If you press F5 for lookup, DIAMOND logic displays records that satisfy the Site Validation criteria. When you select a record, DIAMOND logic displays the Site code, Panel, IPA, site name, Certification and Address.

The Site Validation criteria applied here include:

  •   The Provider ID on the provider address record for the Site Code must match the Provider ID on the provider enrollment rule. (It is standard keyword PROVE logic that the Provider ID on the provider address must match the Provider ID on the provider enrollment rule. This logic just uses Site Code to locate the correct provider address record for the site.)

  • The effective date of the site address linked to this provider address must be prior to or equal to the effective date of the provider enrollment rule.

  • The line of business, Panel, and IPA of the Site Code master record of the Site on the provider address record must match the provider enrollment rule.

    • If the Lookup window Match All Contracts check box is clear, the match on IPA and Panel includes matching blanks on the provider enrollment rule. That is, a blank IPA on the provider enrollment rule is only matched by a blank IPA on the Site Code master record and a blank Panel on the provider enrollment rule is only matched by a blank Panel on the Site Code master.

    • If the Lookup window Match All Contracts check box is selected, the match does not include IPA and Panel blanks matching blanks. Only entered values must match. A blank IPA or Panel on the provider enrollment rule matches any IPA or Panel value.

Finding PCPs Using Site Validation

When you add an initial member eligibility period including the PCP ID, and Site is required by the member's line of business, DIAMOND logic includes Site Validation. Site Validation is also included when you change the PCP on an existing period where Site is required by the line of business.

Specifically, when you press F5 to lookup on the PCP ID field and then click Search, in addition to the normal provider contract search and provider enrollment rules evaluation, DIAMOND logic filters the PCP records and only displays those that satisfy Site Validation requirements. Specifically, the PCP must have a provider enrollment rule where:

  • The provider enrollment rule line of business, IPA, and Panel matches the member eligibility record.

    • If the Lookup window Match All Contracts check box is clear, the provider enrollment rule IPA and Panel must match the member eligibility record, including blanks matching blanks. That is, if IPA or Panel is blank on the member eligibility record, it must be blank on the provider enrollment rule.

    • If the Lookup window Match All Contracts check box is selected, the provider enrollment rule must match any entered IPA and Panel values. Only entered values must match, a blank IPA or Panel on the member eligibility record matches any IPA or Panel value on a provider enrollment rule.

  • The provider enrollment rule includes a Site code (since Site is required by the member's line of business).

  • The effective period of the provider enrollment rule encompasses the effective period of the member eligibility record.

  • The provider enrollment rule meets Site certification requirements:

    • If the member's line of business has the Cert check box selected and no Age From and Age Through are entered, the provider enrollment rule must have the Cert check box selected.

    • If the member's line of business has the Cert check box selected and Age From and Age Through are also entered, and the member's age falls within this Age From/Age Through range, then the provider enrollment rule must have the Cert check box selected.

  • The effective date of the site address linked to the provider address used on the provider enrollment rule is prior to or equal to the effective date on the member eligibility record.

  • The member's age falls within the provider enrollment rule's Age From/Age Through range.  

  • The provider contract corresponding to the provider enrollment rule has both the PCP Flag and Accepts New Pat check boxes selected. (This is standard PCP assignment logic.)

In the F5 lookup window, when you select a record, DIAMOND logic displays the values from the selected record in the PCP ID, Site, IPA, and Panel fields.

You can also type a PCP ID. If you do, DIAMOND logic performs the same validations with two exceptions. The Match All Contracts check box is considered clear since you did not use the F5 Lookup, and the IPA and Panel are not automatically populated.

Finding Sites Using Site Validation

You can press F5 to lookup in the Site field before entering a PCP ID. If you do, the DIAMOND application applies the same logic and displays the same selections as when you use an F5 lookup in the PCP ID field. See Finding PCPs that satisfy Site Validation criteria when adding or changing a member eligibility period.

Using Match All Contracts

The Match All Contracts check box is available on the Lookup window when you press F5 in the following fields:

  • Keyword PROVE field Site

  • Keyword MEMBR fields PCP ID and Site

  • Keyword MELIG field Site

Selecting or clearing the Match All Contracts check box changes the logic that returns matching records when you click Search in the Lookup window, as follows:

  • If the Lookup window Match All Contracts check box is clear, the match on IPA and Panel includes matching blanks. That is, a blank IPA on the current keyword is only matched by a blank IPA on the records being searched, and a blank Panel on the current keyword is only matched by a blank Panel on the records being searched.

  • If the Lookup window Match All Contracts check box is selected, the match does not include IPA and Panel blanks matching blanks. Only entered values must match. A blank IPA or Panel on the current keyword matches any IPA or Panel value on the records being searched.

The Match All Contracts check box is only available if Site Validation logic is enabled in system parameter MEMSITEVAL, with one exception. The exception is that the Match All Contracts check box is always available for keyword MEMBR field PCP ID, regardless of whether Site Validation logic is enabled in system parameter MEMSITEVAL.

Site Validation Limitations

Certain limitations apply to Site Validation:

  • Site Validation logic does not apply to batch, DDS (DIAMOND Data Services), or back-end processing.

Incoming Member Batch or DDS transactions do not include a Site code. Member records submitted using Member Batch or DDS where the line of business requires a Site do not produce an error until you access the record online and try to save. You can use a 3rd Party query tool to report on members added where the line of business requires Site. You can then add Site using keyword MELIG.

  • You should not change data on an existing member eligibility record using keyword MELIG that could impact Site assignment, such as changing the PCP, IPA or Panel, with one exception. If you do, you can cause errors in DIAMOND processing for such functions as Premium Billing. The exception is that you can add Site to an existing eligibility record using keyword MELIG. Other than that, you should use the appropriate MEMBR Special menu commands to first terminate the existing record and then add a new eligibility segment.

  • You must use the keyword MEMBR Special menu command Provider Change to change an existing PCP or Site code. Site Validation logic is not available for the keyword MEMBR Special menu command Modify Existing Provider.
