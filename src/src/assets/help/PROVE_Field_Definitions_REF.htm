PROVE Field Definitions

Field Name

Req

Description

Source

Comments

Accepts Existing Patients

O

An information only indicator that the provider accepts or does not accept existing patients.

Selected with a check box.

Selected = provider accepts existing patients (default).

Cleared = provider does not accept existing patients.

Accepts New Patients

O

Indicator that the provider accepts or does not accept new patients.

Selected with a check box.

Selected = provider accepts new patients (default).

Cleared = provider does not accept new patients.

Address

R

Provider’s address associated with the enrollment rule.

Selected from a predefined list

Only one enrollment rule record for a provider ID/effective date/termination date/LOB/IPA/panel/Site/provider address combination is allowed.

Age From

O

Youngest age of patients accepted by the provider.

Manually entered.

 

Age Through

O

Oldest age of patients accepted by the provider.

Manually entered.

 

Available for Auto Assign

R

Indicates if this PROVE record is available for PCP Auto Assignment.

Manually selected

Once a value is saved, you cannot change it.

Cert

D

Certification value from the linked provider address record.

Displayed

Unavailable if system parameter MEMSITEVAL Parameter 1 = N.

Only displayed if the provider address is linked to a site address record.

Current Enrollment

D

Number of patients assigned to the provider for the highlighted enrollment rule record.

Calculated by DIAMOND.

PROVENROL parameter must be activated. See PARAM user documentation for additional information.

May be greater than the Enrollment Limit.

Effective Date

R

Effective date of the enrollment rule record.

Manually entered.

 

Enrollment Limit

O

Maximum number of patients accepted by the provider.

Manually entered.

 

Gender

R

Gender of patients accepted by the provider.

Selected from a predefined list.

Valid values:

Both (default)

Female

Male

IPA

O

IPA associated with the provider contract record for the selected LOB.

Selected from a predefined list of valid IPAs for the LOB selected.

Only one enrollment rule record for a PCP provider ID/effective date/termination date/LOB/IPA/panel/provider address combination is allowed.

Opened for entry after a valid LOB is entered.

LOB

R

Line of business associated with the enrollment rule.

Selected from a predefined list of valid LOBs for the provider.

Only one enrollment rule record for a PCP provider ID/effective date/termination date/LOB/IPA/panel/provider address combination is allowed.

The provider’s PROVC – PCP Flag check box must be selected.

Panel

O

Panel associated with the provider contract record for the selected LOB.

Selected from a predefined list of valid Panels for the LOB selected.

Only one enrollment rule record for a PCP provider ID/effective date/termination date/LOB/IPA/panel/provider address combination is allowed.

Opened for entry after a valid LOB is entered.

Provider ID

R

Valid provider ID.

Selected from a predefined list (Provider Master).

 

Site

C

Unique identifier of a site (facility)

Manually entered

Unavailable if system parameter MEMSITEVAL Parameter 1 = N.

If available, required if the corresponding LINBS record has the Site check box selected. Else, optional.

F5 lookup is available. For more information on this F5 lookup, see Site Validation, How DIAMOND Thinks section Entering a Site code on a provider enrollment rule.

When you select a Site, the Site code, site name, Panel, IPA, and Cert check box value display, and the Address field is populated.

Term Date

R

Termination date of the enrollment rule.

Manually entered.

 

Term Reason

C

Reason for the enrollment rule termination.

Selected from a predefined list.

Required if date entered in Term Date.

Protected from entry if no date entered in Term Date.

User Defined 1

O

Definition is implementation specific

Manually entered

Can be used in PCP Auto Assignment.

Once a value is saved, you cannot change it.

You define the field literal in keyword UDTXT (User Defined Fields).

User Defined 2

O

Definition is implementation specific

Manually entered

Can be used in PCP Auto Assignment.

Once a value is saved, you cannot change it.

You define the field literal in keyword UDTXT (User Defined Fields).
