Setting Up the Non-Specific Provider

Restrictions

Instructions

Restrictions

None.

Instructions

Step

Comment

1. In keyword PROVF, create the non-specific provider.

  1. Provider ID = NONSPECIF (clearly identifies the non-specific provider)

  2. Last Nm = NonSpecific

  3. First Nm = Provider

  4. Type = DEFAULT

  5. Select the Non Specific Prov check box.

  6. Specialty Type = DEFLT

  7. Primary = Yes

  8. Accepts Patients = Yes

You are creating the non-specific provider master record.

You will use this non-specific provider record when you pay out-of-area, non-par, or one-time provider claims.

You do not have to create any PROVC Contract Pricing Details for the non-specific provider record as DIAMOND will use the default provider’s (DEFAULTP) contract record pricing data when you pay out-of-area, non-par, or one-time provider claims.

You do not have to create an address record for this provider. As you enter out-of-area, non-par, or one-time claims using this non-specific provider, you will enter the actual provider’s name and address, on-the-fly, as the claim’s vendor name and address.
