Member/PCP Assignment Considerations

If you have activated DIAMOND’s provider/member enrollment functionality, the following apply to both member online and member batch processing (except where noted):

  • DIAMOND performs automated provider enrollment rule evaluation when you select a PCP for the member (using MEMBR > PCP ID).

  • DIAMOND also performs provider enrollment rule evaluation when you use MEMBR > Special menu > Add Dependent, Terminate/Unterminate, Reinstate, Provider Change, or Modify Existing Provider.

  • DIAMOND’s provider/member enrollment functionality does not include provider changes made using the Member Eligibility window (MELIG); that is, MELIG provider changes are not validated against PROVE records. In most cases you should not use MELIG to make changes in member eligibility. See the MELIG user documentation for additional information.

  • If there are no enrollment rules for the PCP ID selected as of the member’s effective date (MEMBR > Eff Date), DIAMOND only performs a provider contract search to verify the existence of a provider contract.

  • DIAMOND does not perform provider enrollment rule evaluation for the provider ID you enter in the MEMBR > Prov2 field.

  • The MEMBR > Eff Date is evaluated against the provider’s enrollment rules effective dates (PROVE > Effective Date, Term Date) for new member enrollments and for MEMBR, Special menu > Add Dependent, Terminate/Unterminate, Reinstate, Provider Change, or Modify Existing Provider actions.

  • If the MEMBR > Eff Date is not within the PROVE > Effective Date, Term Date (for example, the rule has expired), DIAMOND only performs a provider contract search to verify the existence of a provider contract.

  • During online member enrollment, if a provider has multiple enrollment rules (rules for different addresses), DIAMOND prompts the operator to select the appropriate enrollment rule (based on provider address).

  • During member batch processing, if a provider has multiple PROVE records for different addresses, DIAMOND evaluates the rule associated with the provider’s "primary address."

  • DIAMOND performs the following enrollment rule validations before assigning a PCP to a member -

The provider accepts new patients (PROVE > Accepts New Patients box is checked).

The provider has not reached the enrollment limit (PROVE > Enrollment Limit). The provider enrollment limit and member count toward the enrollment limit are evaluated as of the effective date of the requested change.

The provider accepts patients of the member’s gender (PROVE > Gender).

The member’s calculated age (as of the current system date) is within the provider’s acceptable age range (PROVE > Age From/Age Through).

  • If any of the enrollment rules do not pass validation and there are no override edits (PROVE, Special menu > Override Enrollment Edits), DIAMOND displays an appropriate error message and the PCP is not assigned to the member.

  • If any of the enrollment rules do not pass validation and there are applicable override edits (PROVE, Special menu > Override Enrollment Edits), DIAMOND displays an appropriate warning message that the PCP will be assigned to the member because of the override.

  • If all enrollment rules pass validation, DIAMOND assigns the PCP to the member’s eligibility record (MELIG) and the PROVE > Current Enrollment count is incremented as of the effective date of the change.

Provider/Member Enrollment Utility - Background

To begin using DIAMOND’s automated provider/member enrollment functionality, we strongly urge you to activate the PROVENOROL parameter and run the Provider/Member Enrollment Utility scripts as outlined in the D950C/S R5.1 Software Installation Guide.

After you run this utility, each time you assign a PCP to a member, DIAMOND evaluates the provider’s enrollment rule(s) (PROVE) to determine if the provider will accept the member being assigned. DIAMOND also updates the provider’s PROVE > Current Enrollment counts as members are assigned to the provider and as specific changes are made to member/provider records.

Provider/Member Enrollment Utility - Overview

When you activate the PROVENROL parameter and run the Provider-Member Enrollment Utility scripts, DIAMOND:

  • Creates a generic PROVE record based on the PROVE record creation criteria selected (see Provider/Member Enrollment Utility section PROVE Record Creation for details).

  • Associates each eligible member, who has an existing, assigned PCP, with a PROVE record.

  • Increments the associated provider’s enrollment count (PROVE > Current Enrollment) in all appropriate PROVE records. This includes enrollment count updates for retroactive member/provider assignments.

After you run the utility, DIAMOND updates the associated provider enrollment count as you make specific changes to member/provider records; for example, member adds, terminations, unterminations, or reinstatements.

We strongly urge you to keep the PROVENROL parameter activated after running the utility. If you manually change the PROVENROL parameter back to ‘N’ after you have run the utility, DIAMOND stops evaluating provider enrollment rules as you assign PCPs to members; however, DIAMOND does perform a provider contract search to verify the existence of a provider contract. Since provider enrollment rules are no longer evaluated, DIAMOND does not update the associated provider enrollment counts. As a result, these counts may not be correct.

Provider/Member Enrollment Utility, PROVE Record Creation

When you run the utility for the first time, there are no existing PROVE records in the PROV_ENROLLMENT_RULES table. DIAMOND prompts you to select one of the following enrollment rule record creation criteria:

  • LOB + Provider Address

  • LOB + IPA + Provider Address

  • LOB + IPA + Panel + Provider Address

  • LOB + Panel + Provider Address

After you select the criteria, the utility creates a generic PROVE record (based on the PROVE record creation criteria selected) for PCPs who do not have at least one PROVE criteria based record for a provider contract record. The utility associates the PROVE record with the provider’s primary address. If the provider does not have a primary address, the utility does not create a PROVE record.

The utility then associates each eligible member, who has an existing, assigned PCP, with the corresponding provider enrollment rule record and increments the record’s enrollment count. That is, the PROVE seq_enrollment_rule ID is added to the member’s membership eligibility history record. The utility does not enforce or evaluate enrollment rules during this process since the provider was previously assigned to the member.
