Batch Claim Hold Release Overview

You can release and retest multiple claim holds at one time using the Batch Claim Hold Release process. This process allows you to define rules and determinants to select particular holds to release and retest. These release rules and determinants, along with settings in keyword RHJOB (Claim Hold Release Setup Job), enable you to define criteria for selecting particular holds to release and retest in the batch process. These criteria include:

  • Hold Reason Code

  • Claim type

  • Claim header level only, or header and detail level

  • Entered Date or Received Date

  • Selected determinants, selected from the Group Master table, Institutional Claim Header table, Institutional Claim Detail table, Professional Services Claim Header table and Professional al Services Claim Detail table

Certain claims and claim detail lines are exempt from selection by this process, since processing in this manner would adversely affect other processing or functionality. For example, claims are excluded which have been whole claim priced, where COB calculations or third party processing (e.g. Auto Audit) have been performed, or which have been manually adjudicated. See Batch Claim Hold Release, How DIAMOND Thinks for more information.

The Batch Claim Hold Release process also reprocesses the selected claims and claim detail line using much of the normal claims processing logic. For example, this reprocessing includes reapplying hold rules, pricing and adjudicating the selected claim detail lines, performing authorization waive and auth/claim matching, and more. Again, see Batch Claim Hold Release, How DIAMOND Thinks for more information.
