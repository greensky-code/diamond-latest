MEMBR Commands on the Menu Bar Command  Menu  Description

Add Dependent

S

Use to add a dependent to a subscriber unit (family).

If you are using DIAMOND IDs, and there are multiple members under one DIAMOND ID, then a pop-up window displays and you can select the correct ID.

For more information, see Add a dependent to a subscriber.

Add DIAMOND ID

S

Use to add a DIAMOND ID to a member record. This command can be used whether or not the parameter DIDSEARCH is set to ‘Y’. If there is more than one member with this DIAMOND ID, then a popup window displays the others so you can select the correct member. The popup window displays the Sub ID, Person Number, Last Name and First Name.

Add New Subscriber to DID

S

Use to add a new subscriber to an existing DIAMOND ID. Display the original record of the subscriber using that subscriber’s DIAMOND ID, and on the Special menu, click Add New Subscriber to DID. The Name, Gender, Address, and Date-of-Birth populate the MEMBR screen if a subscriber match is found for a new member who is the second member with that DIAMOND ID. This command is not enabled if the DIDSEARCH parameter is set to ‘N’.

Address Ripple

S

Use to change the address of the member along with all other members of the subscriber unit who have the same current address. Member address ripple applies only to the subscriber’s primary/residential address, not to additional addresses entered using keyword MEMBA.

To have the same current primary/residential address, the following address fields for all members in the subscriber unit must be identical:

  • Addr 1

  • Addr 2

  • City

  • State

  • ZIPPOST

  • Home Ph

If any of these fields are not identical in a member's MEMBR window address fields (or fields are blank), then that member's address information will not change as a result of applying an Address Ripple. However, the Address Ripple will affect other members in a subscriber unit that have identical address information.

When the Address Ripple does not affect a member of a subscriber unit because their address information is different, a "No" will appear in the Address Ripple popup box for that member's person number.

For more information, see Change the address of a subscriber.

Audit Display

Details

W

Displays audit trail records for the selected record.

Claim Hold Codes

S

Adds a claim hold reason code and date to the member’s record.

D/C Information

Details

S

Use to enter debit and credit account information. This field is available if the Use EFT check box is selected in keyword MEMBB.

ID Card Request

S

Use to manually order an ID card and view the member’s ID card history.

Letter Request

S

Use to request a letter to be sent to the member. For more information, see LTEMP, Letter Template and LTPRT, Letter Print.

M+C Information

T

Use to access keyword MEMMC.

Medicare Risk Info

S

Includes the HCFA accretion date, deletion date, risk refused date, comments, user defined text fields, and user defined date fields.

Member Address

T

Use to access keyword MEMBA.

Member COB Verification Information

Details

S

Enter or view coordination of benefits (COB) verification information obtained from the member or another carrier. Includes user defined text fields and user defined dates.

Member Lookup

S

Use to search through your database of members.

Member User Fields

S

Opens a window with 20 user defined fields and 20 user defined date fields. Field literals and validation can be changed for the 20 user defined fields using keyword UDTXT (User Defined Fields). For the 20 user defined date fields, the field literals can be changed using UDTXT but validation cannot be added.

Modify DID

S

Changes the DIAMOND ID for one person and can remove the Dual Coverage Indicator that shows that this person had dual coverage. This command is only available when the parameter DIDSEARCH is set to ‘Y’. For more information, see Dual Coverage, How DIAMOND Thinks.

Modify Existing Provider

S

Use to correct errors in the member’s PCP or second assigned provider.

Modify Privacy Indicator

Details

S

Use this command to terminate an eligibility record due to a change in the privacy preference of the member.

Notes

Details

N

This command is only available if the MEMBR window has been associated with at least one note type in NOTEW.

Takes you to the Notes Entry window where you can view a list of notes or add, view, or edit a note detail.

For information on setting up notes functionality, see NOTES, How DIAMOND Thinks; NOTET, Note Type; and NOTEW, Note Window user documentation.

Provider Change

Details

S

Use to change the member’s primary care provider.

Reinstate

S

Establishes a new eligibility history record for a member. For more information, see Notes on Benefit Start Date.

Subscriber Designation

S

Use to make an existing dependent within a subscriber unit the subscriber. This is sometimes necessary when, for example, the subscriber dies and the spouse assumes the subscriber designation.

Subscriber ID Change

Details

S

Use to change the member’s subscriber ID.

For more information, see Subscriber ID Changes, How DIAMOND Thinks.

Terminate/ Unterminate

S

Use to end a member’s eligibility or remove an eligibility end date from the member’s current period of eligibility.

Verify Student/ Dependent Status

S

Use to enter a dependent verification status indicator, dependent verified through date, user defined text fields and user defined dates for the member.

View Auto Letter History

S

Use to display the Auto Letter correspondence files generated for this member. Only those auto letters created from templates with view history saved display.

View DID Members

S

Shows all members with the same DIAMOND ID. This command is not available if the DIDSEARCH parameter is set to ‘N’.

View IPA Info

S

Displays the contract current as of the system date for the IPA associated with the member. Information is from IPACT.

Working Aged

T

Use to access keyword MEMWA.
