PROVC Field Definitions

Field Name

Req

Description

Source

Comments

Address

R

The address to which fee-for-service payments are sent.

Manually entered

 

Board Status

D

Indicates if the provider is board certified or eligible for certification for the selected provider contract.

Manually entered

Warning - Field data is informational only and specifically for the selected provider contract. A similar field exists in keyword PROVF; however, this field is not populated with existing data on PROVF>Board Status.

If you add or change this field data, the corresponding field in keyword PROVF is not updated. Use PROVF>Board Status to make data changes that may affect claims processing.

See PROVF user documentation for additional information.

Cap Vend Addr

O

The address to which payments to the Capitation Vendor are sent.

Selected from predefined list

 

Cap Fund Model

C

The Model ID used for this provider contract for Cap/Special Fund Distribution.

Selected from predefined list

Required if Def Contr Type is set to F.

Capitation fund models are maintained in keyword CFMOD.

Cap Fund Sub Model

O

The Sub Model ID used for this provider contract for Cap/Special Fund Distribution.

Selected from predefined list

Used only if there is an entry in the Cap Fund Model field.

Capitation fund sub-models are maintained in keyword CFMOD.

Cap Vend ID

O

Unique identifier of payee for capitation payments made under this contract.

Selected from predefined list

 

Com COB Allowed

O

Indicates the allowed amount to be used when a member has other commercial coverage and the claim is automatically adjudicated using a Professional or Institutional COB Calculation method of Integration of Benefits or Non-Duplication of Benefits.

Selected from predefined list

Values are

A Use Allowed Amount

L Use lesser of Allowed Amount or Other Carrier Allowed Amount

Descriptions of these values are maintained in CODES; logic is hard coded.

Com COB Rsn

O

A reason code explaining the basis for the Com COB Allowed field value for this provider contract.

Selected from predefined list

Reason codes are maintained in keyword REASN.

Used when a member has other commercial coverage and the claim is automatically adjudicated using a Professional or Institutional COB Calculation method of Integration of Benefits or Non-Duplication of Benefits.

If the Com COB Allowed rule is used in claims, the corresponding Com COB Rsn displays in the COB Alwd Amt Rsn field on the Special menu command COB Information window.

Cov Group

O

Identifier of a covering provider group with which this provider, under this contract, is associated.

Selected from predefined list

 

Def Contr Type

R

The contract type used by capitation processing to identify capitated providers when the selection criteria (in CPENT) is Contract Type and the capitation model (CPMOD) has "Use Prov Contract" check box selected.

Selected from predefined list

Valid contract types are

C Capitated

F Cap/Special fund

P Payable (fee-for-service)

S Staff

Dflt

O

Indicates whether this address is the default address for the provider.

Selected from check box

At least one address per vendor must be specified as the default.

Dir Inc(lude)

(Specialty Section)

D

An indicator of whether the provider/specialty should be included in a provider directory for the selected provider contract.

Manually entered

Warning - Field data is informational only and specifically for the selected provider contract. A similar field exists in keyword PROFV; however, this field is not populated with existing data on PROVF>Dir Inc.

If you change this field, the corresponding field in keyword PROVF is not updated. Use PROVF>Dir to make data changes that may affect claims processing.

See PROVF user documentation for additional information.

Dir Inc

(Vendor Section)

O

An indicator of whether the vendor address should be included in a provider directory.

Selected from check box

 

Eff Date

R

The effective date of this contract.

Manually entered

 

Exclude Adm Fee

O

Indicates whether the contract is excluded from administrative rules.

Selected from check box

If this check box is selected, then the contract is excluded from administrative fees.

If this check box is cleared, then the contract is eligible for administrative fees.

Fed Tax ID

R

The Federal Tax ID for the Provider Contract record.

Manually entered

 

Filing Limit Days

O

The filing limit days for the provider.

Manually entered

Value used when determining if the claim age is less than, equal to, or greater than the filing limit.

This value is overridden by the Filing Limit Days field in PROVC if the following conditions are true

The Filing Limit Days field in PROVC has a value (is not empty).

Ignore Prov Limit Days is not checked.

IPA

O

The provider IPA with which this contract is associated.

Manually entered

 

LineBus

O

The line of business with which this contract is associated.

Manually entered

 

MCare Assign

O

Indicates if, for this contract, the provider agrees to accept the Medicare allowed amount rates as payment in full.

Selected from check box

Valid values are:

<Selected> - Provider agrees to accept Medicare allowed amount rate as payment in full

<Cleared> - Provider does not agree to accept Medicare allowed amount rate as payment in full

Panel

O

The provider panel with which this contract is associated.

Manually entered

 

Par Flag

R

Indicator of whether the provider is considered Participating under this contract.

Selected from predefined list

Hard coded values are

P Participating

N Non-participating

Implementation specific codes are maintained in CODES with a type PARSTATUS.

Member benefits and capitation arrangements may depend on the participation status of the rendering provider.

PCP Flag

O

Indicates that provider is a primary care physician.

Selected from check box

Members can only be assigned to PCPs.

Primary (Specialty Type)

D

Indicates if this specialty type is the provider’s primary specialty for the selected provider contract.

Manually entered

Warning - Field data is informational only and specifically for the selected provider contract. A similar field exists on PROFV; however, this field is not populated with existing data on PROVF>Primary.

If you change this field, DIAMOND does not update the corresponding field on PROVF. Use PROVF>Primary to make data changes that may affect claims processing.

See PROVF user documentation for additional information.

Primary (Taxonomy Type)

C

Indicates if this taxonomy code is the provider’s primary taxonomy code for the selected provider contract.

System generated or manually entered.

This field is required if the Taxonomy Code field contains a value.

Valid values are:

 Y Yes

 N No

If the provider contract record has one or more Taxonomy Codes specified, one and only one must be marked as primary; otherwise, you will receive an error when you attempt to save the record.

Print RA

O

Indicates how DIAMOND will produce a remittance advice to the provider when payments are made under this contract.

Selected from predefined list

Hard coded values are:

1 Paper RA for Inst /Prof/Dental

2 No RA for Inst/Prof/Dental

3 Electronic RA for Inst & Prof only

4 Electronic RA for Inst only

5 Electronic RA for Prof only

6 Paper RA for Inst only

7 Paper RA for Prof only

8 Paper RA for Inst & electronic RA for Prof/Dental

  1. Paper RA for Prof/Dental & electronic RA for Inst

A Electronic RA for Dental only

B Paper RA for Dental only

The entry here is overridden for services delivered to members who are ineligible; in those cases, whether or not a remittance advice is printed is governed by the DFLTPRINTSTAT parameter.

Provider ID

R

The unique identifier of the provider.

Manually entered, or selected from predefined list

 

Specialty Type

D

The provider’s specialty for the selected provider contract.

Manually entered

Warning - Field data is informational only and specifically for the selected provider contract. A similar field exists on PROFV; however, this field is not populated with existing data on PROVF>Specialty Type.

If you change this field, DIAMOND does not update the corresponding field on PROVF. Use PROVF>Specialty Type to make data changes that may affect claims processing.

See PROVF user documentation for additional information.

Taxonomy Code

R

One or more provider taxonomy codes used to classify the provider by type of service, type of provider, and area of specialization or focus.

Selected from predefined list

F5 lookup is available.

Each provider contract may contain between 0 and 999 taxonomy codes.

Each taxonomy code must be 10-characters.

There is no pricing or adjudication associated with this field; it is informational only.

Term Date

O

The last date this contract is in force.

Manually entered

 

Term Rsn

O

A code explaining the termination date of the contract.

Selected from predefined list

Termination reason codes are maintained in keyword REASN with a type TM.

Type / Description (Taxonomy Code)

D

The type and description of the selected taxonomy code.

Displayed

Displays the type and description text from the TXNMY window for the selected taxonomy code.

This field may be blank if no taxonomy type or taxonomy description exists in keyword TXNMY for the specified taxonomy code.

User Def 1, 2

O

Implementation defined code.

Manually entered

Valid values for these fields may be defined in keyword UDTXT; otherwise, any entry is valid.

Vendor

R

Unique identifier for payee for fee-for-service (payable) payments.

Selected from predefined list
