PROCP Field Definitions

Field Name

Req

Description

Source

Comments

Allowed Amt

O

Dollar amount that will be entered on the allowed amount field on the claim detail line. Sometimes referred to as the fee.

Manually entered

You can type 16 digits before the decimal and 2 digits after the decimal.

Contract Override

O

Value which takes precedence over the entry in the provider’s PROVC Pricing Details contract type field.

Selected from predefined list

Used to administer partial capitation arrangements where some procedures are reimbursed fee for service, and others are covered by a capitated payment.

Valid values are

P Payable

C Capitated

S Staff

This field affects the Claim Status field on the claim detail line. Unless the detail line is denied, applicable to a cap/special fund arrangement, or an adjustment, Claim Status will be set to equal the Contract Override value.

Effective Date

R

First date on which the information in this record applies.

Manually entered

Overlapping dates are not allowed for each unique Price Schedule/Pricing Region/Modifier Code/Effective Date record.

Hold Date

C

First date the Procedure Hold is effective

Manually entered

Required if Procedure Hold is entered.

Modifier Code

O

Procedure suffix representing a subset of the service or other refinement of meaning.

Manually entered

Modifiers are maintained in keyword MODIF (Modifier).

Note that of the four Modifier codes on the professional services claim detail line, DIAMOND only uses Modifier 1 in pricing and adjudication.

Pct Of Billed

O

Percentage which, when the allowed amount is not applicable, is applied to the billed amount to calculate the entry for the allowed amount field on the claim detail line.

Manually entered

An entry here overrides the percent of billed on the provider’s PROVC contract record, and/or overrides the percent of billed on the member’s group detail, GRUPD.

Per Diem Flag

O

Indicator of whether procedure price represents a per diem amount.

Manually entered

 

Price Schedule

R

Code representing a set of prices.

Manually entered

Price schedules are maintained in keyword PSCHD (Price Schedule).

Procedure Code

R

Unique identifier of a medical service.

Manually entered

 

Procedure Hold

O

Reason code that explains the hold on a claim line when this procedure record applies.

Selected from predefined list

Hold reasons are maintained in keyword REASN (Reason) with the Reason Code Type list set to HD – Hold.

Region

R

Code representing a geographic region.

Manually entered

F5 lookup is available.

Regions are maintained in keyword REGIN (Region).

Region Type

R

The region type that corresponds to the code entered in the region field.

Selected from a predefined list.

Valid values are:

P - Price

Z - Geo Zip

Term Date

O

Last date on which the information in this record applies

 

Overlapping dates are not allowed for each unique Price Schedule/Pricing Region/Modifier Code/Effective Date record.

User Defined 1-5

O

Implementation specific values.

Manually entered

 

Withhold Pct

O

Percentage that is applied to the entry in the claim detail allowed amount. The result is entered in the withhold field.

Manually entered

An entry here overrides the percent withhold on the provider’s PROVC contract record.
