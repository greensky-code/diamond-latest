Adj for OC Pd

O

If selected, benefit accumulators are adjusted to account for other carrier paid amounts. Applies only to COB calculation methods Integration of Benefits and Non-Duplication of Benefits.

Selected from check box

See Coordination of Benefits, How DIAMOND Thinks > COB Calculation Methods and Examples documentation for examples of the application of this field.

Apply Auth Level

D

Indicates if an Auth Level will or will not apply to the specific values defined in the Define Benefit Value Filters special function.

 

Displayed

Hard coded values are:

  • Some - Indicates that the user has defined values in the Benefit Value Filter special function.

  • None - The user has not created Benefit Value Filters.

  • X - not applied.

  • All - All values apply (The Auth Req. field is Y, but Benefit Value Filters are not defined so all apply).

Apply Par

D

Indicates if Par Benefit Filters have been defined for Tilter Type Par.

Displayed

Hard coded values are:

  • X - not applied.

  • Some - Indicates that the user has defined values in the Benefit Value Filter special function.

Apply Pat Liability To

R

Dollar field or amount on the claim detail line that is used to calculate patient liability. Applies only to COB calculation methods Integration of Benefits and Non-Duplication of Benefits.

Selected from predefined list

Hard coded values are:

  • Billed

  • Allowed

  • Lsr B/A - Lesser of the billed or allowed amount

  • Lsr B/A/OA - Least of billed, allowed, or other carrier allowed

  • Lsr A/OA - Lesser of allowed or other carrier allowed

See Coordination of Benefits, How DIAMOND Thinks > COB Calculation Methods and Examples documentation for examples of the application of this field.

Apply Riders

D

Indicates if Rider Benefit Filters have been defined for Filter Type Rider.

Displayed

Hard coded values are:

  • X - not applied.

  • Some - Indicates that the user has defined values in the Benefit Value Filter special function.

Auth Req.

O

Indicates if the system evaluates if an authorization is associated with the claim to determine if the rule should be applied.

 

Selected from a predefined list

Hard coded values are:

  • Y - Apply rule if an authorization, regardless of level, is associated with the claim.

  • N - Apply rule only if no authorization is associated with the claim.

  • X - Apply rule regardless of whether or not an authorization is associated with the claim.

  • L - User-defined auth level. User has defined the Auth Level that applies to the Benefit Rule (see Define Benefit Value Filters special function).

If Benefit Filters have been defined, N or X cannot be selected.

 

To apply a rule depending on the level of the authorization associated with the claim, select a Level Code. Level Codes are   maintained in CODES.

Proc Order

D

Order in which the selected benefit rule, according to its type, is evaluated.

Displayed

Processing order values are retrieved from the BORDR record entered in Processing Order ID.

Processing order + processing sequence (Proc Seq) are combined by DIAMOND into the overall order in which a rule is evaluated. Each rule has a unique overall order that is retained even after rule is terminated.

Proc Seq

R

Sequence in which a rule is evaluated within Processing Order.

System calculated; can be manually overridden.

Processing order + processing sequence (Proc Seq) are combined by DIAMOND into the overall order in which a rule is evaluated. Each rule has a unique overall order that is retained even after a rule is terminated.

Processing Order ID

R

Record in BORDR that determines the order in which types of rules are applied.

Selected from predefined list

Individual benefit rules (from BRULE) are applied in the rule type order identified here. Rules types are Copayments, Coinsurance, Deductibles, Limits, etc.

Rule ID

R

Unique identifier of a benefit rule included in this package.

Selected from predefined list

One benefit rule (BRULE) may be linked to the same benefit package (BENEF) multiple times to accumulate benefits for various participation statuses (BENEF>PAR) and authorization levels (BENEF>AUTH). Doing this has an impact on how DIAMOND displays benefit accumulations on DSPBN. See DSPBN>Rule ID reference documentation for additional information.

Start Date

O

First date on which this rule is applied within this benefit package.

Manually entered

When there are multiple entries in a package for same rule ID, start and end dates cannot overlap or duplicate each other.

Sub Flag

O

Indicates that this rule will be applied when the patient’s relationship code (on MEMBR) matches the code entered here.

Selected from predefined list

Hard coded values are:

  • H Sub - Apply rule if member is the subscriber. (Regardless of whether member has relationship code H for Head of Household.)

  • S Spouse - Apply if member has relationship code of S for Spouse or E for Domestic Partner.

  • D Dep - Apply if member has any relationship code other than S, E, or H.

  • A All Except Sub - Apply if member is not the subscriber.

  • (blank) - Apply to all family members.
