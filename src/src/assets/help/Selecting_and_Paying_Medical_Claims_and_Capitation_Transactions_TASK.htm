Selecting and Paying Medical Claims and Capitation Transactions

To select and pay medical claims and capitation transactions, you must do the following:

1.  Create and run an A/P update job

In keyword APUPD, create and run a new A/P update job or update and run an existing batch job.

A header hold will prevent claim detail lines from being sent to accounts payable. A header hold will also prevent claim lines from being finalized.

A partial payment election will be determined during APUPD preprocessing.

Use of Cap/Special Fund Distribution is determined during APUPD processing.

Evaluation of third-party audit software status will take place during APUPD processing.

To maximize performance, Perot Systems Healthcare recommends using minimal selection criteria when running APUPD. Additional filtering can then be applied when running a job in keyword CKPRT (Check Print)

Learn how to run an A/P update job.

2.  De-select or re-select A/P records for payment

In keyword APSEL, de-select or re-select A/P records for payment. Transactions can be de-selected for payment based on:

  • Company code

  • Vendor ID

  • Due Date

  • Entered Dates

When a transaction is de-selected for payment, it will not proceed through keyword CKPRT; it must be re-selected to re-enter the process.

Learn how to de-select claims or capitation transactions for payment.

Learn how to re-select claims or capitation transactions for payment.

3.  Run a check print job

In keyword CKPRT, calculate and print a check print job.

Learn how to calculate a check print job.

Learn how to print a check print job.

4.  Post checks

In keyword CKPST, post checks for payment.

Learn how to post checks.
