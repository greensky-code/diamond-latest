Comp Dt.

D

Indicates if the date of the ACCUM record is compared against claim dates.

Displayed

If the benefit detail line is the result of a manually entered ACCUM record, and the Rule Type is Limit and the Limit Type is Days, or the Rule Type is Full Coverage and the Full Coverage Type is Days, then this field is populated.

Valid values are:

  • C - Compare the date of the ACCUM record against claim dates and if they match, then don't include the ACCUM record in the benefit accumulation calculation.

  • I - Ignore the date of the ACCUM record and count it in the benefit accumulation calculation regardless or whether or not it is the same date as a claim.

Rule ID

D

The benefit rule(s) ID associated with the member’s benefit package.

System generated

One benefit rule (BRULE) may be linked to the same benefit package (BENEF) multiple times to accumulate benefits for various participation statuses (BENEF>PAR) and authorization levels (BENEF>AUTH). Doing this has an impact on how DIAMOND displays benefit accumulations on DSPBN.

In these cases, DIAMOND displays a benefit accumulation line for every occurrence of the same benefit rule in the benefit package and accumulates the corresponding expended benefit amounts (DSPBN>Accum Amt) for each line; however, the remaining amount (DSPBN>Remain Amt Mbr/Fam) is calculated based on Accum Amt – Rule Limit for each line which distorts the true (total) remaining amount for the rule.

To arrive at the actual remaining amount for all lines of the same rule, add up all Accum Amts and subtract from the rule Limit.
