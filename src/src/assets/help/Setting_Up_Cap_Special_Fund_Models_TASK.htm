Setting Up Cap/Special Fund Models

In order to pay providers from a specified fund of dollars, called a Cap/Special Fund in DIAMOND, a Cap/Special Fund Distribution model must be set up. Each CFD model should reflect an agreement that defines the particulars of the fund payment arrangement between the health plan (or other business entity such as payor) and the included providers.

Key particulars used by CFD models are the payment period (monthly), the basis of payment (claims activity – Allowed Amount or Net Amount), minimum and maximum allocation percentages, and possibly the fund allocation amount. You can use maximum allocation percentages to place an upper limit on the ratio of fund payment prorations to actual claim activity (limiting payor risk). You can use minimum allocation percentages to put a lower limit on the ratio of the fund payment to actual claim activity (limiting provider risk).

You must specify a value for all of these key elements on each model, except for the actual periodic allocation amount. This allocation amount is specified for each Cap/Special Fund batch run.

For more information about how to determine the key values for your models, see Capitation/Special Fund Distribution - How DIAMOND Thinks.

Instructions

Step

Comment

1. Create Cap/Special Fund Models and Submodels using CFMOD.

A CFD model reflects a negotiated agreement between the health plan and certain providers. These agreements stipulate the particulars of CFD payment for these providers.

A single model may be used to define CFD payment particulars for one or multiple provider contracts.

Cap/Special Fund Submodels are optional and used for reporting purposes only.

2. Create Provider Contracts, using PROVC, for the providers included in the Cap/Special Fund Distribution.

PROVC:

Def Contract Type = F (Capitation Fund)

Cap Fund Model ID = <Selected Model ID from CFMOD>

Cap Fund Submodel ID (optional) = <Selected Submodel ID from CFMOD>

Other contract particulars per agreement between the health plan and the provider (for example, effective period)

3. Create Provider Contract Details record

PROVC > Special menu command Contract Pricing Details (appropriate vendor must highlighted):

Contract Type = F - Cap Fund
