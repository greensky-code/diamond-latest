About Medical Definitions

Use keyword MEDEF (Medical Definitions) to define categories of medical services such as office visits, lab tests, or surgeries, called medical definitions, using one or more claim characteristics or determinants. For example, a combination of the procedure rendered and the place of service can define an office visit. Surgeries may be defined by the procedure alone. Chiropractic services may be defined by the diagnosis plus the type and specialty of the provider.

The medical definition code assigned to a claim detail line drives the claim adjudication process, capitation processing for stoploss and deduction pools, general ledger claim expense accounts, and is handy for reporting.

Medical definition codes are first defined in keyword MEDLU (Medical Definition Lookup). Then, for each medical definition code entered in keyword MEDLU, a set of determinants and the appropriate determinant values are defined in keyword keyword MEDEF. The maximum number of Medical Definition codes you can assign in MEDLU is 9,999. The maximum range of order numbers you can assign to a Medical Definition in MEDEF is 9,999. The maximum allowed combinations of Medical Definition Codes and Order Numbers is 99,999.

Making Changes to the MEDEF File

Because a snapshot is created of the medical definitions at the time each user logs on to the system, changes made during the production day are not recognized by a user’s session until he or she logs off and logs on again. Since changes to the medical definitions should be thoroughly tested before they are moved into production, we recommend against making ad hoc changes to these rules. However, be advised of this feature and institute processes to make sure medical definition assignment changes are rolled into the production environment in a predictable and controlled fashion.

Universal Medical Definitions and Exceptional Circumstances: Alternate and Global Searches

In most cases, you will find that medical definitions are applicable for all claims, such as, an office visit is an office visit. In some cases, however, you may find it useful to have certain medical definitions apply in very specific or exceptional circumstances, such as, a service that is defined according to provider status differently for different lines of business.

You can define medical definitions so that they apply to all claims, and also allows you to set certain claim situations apart for special handling. These features are called Global Search and Alternate Search. Global Search, for medical definitions, which apply across the board, is simple to set up, simply click to select the Include in Global Search check box on each medical definition that so applies.

Exception medical definitions require a multi-step set-up process. You establish alternate search criteria (such as line of business or plan) and then link each alternate search criterion to a specific medical definition and its determinants defined in keyword MEDEF.

Alternate Search Set Up

  • Use keyword MEDCO (Alternate Search Columns) to define alternate search criteria that may be applied to one or more medical definitions.

  • Use keyword MEDOR (Alternate Search Order) to define sequential Alternate Search rules by linking an alternate search criterion defined in MEDCO to each exceptional medical definition in MEDEF.

  • DIAMOND applies alternate search rules in numerical order as follows:

1. By MEDOR priority

2. Within MEDOR priority, by MEDOR search order. Each unique MEDOR priority + search order links to a specific MEDEF order.

3. Within MEDEF order, by MEDEF search sequence.

Using Both Alternate and Global Medical Definitions

When an Alternate Search rule is applied, the MEDCO criterion must be satisfied before the MEDEF rule and its determinants are applied. All Alternate Search rules are applied in sequential order (according to MEDOR) prior to applying all Global Search rules in sequential order (according to MEDEF Assigning Order and Search Sequence Number). A single MEDEF rule may be part of an Alternate Search rule and marked "Include In Global Search" and applied in the Global Search.

Global Search Setup

For Global Search medical definition rules, the following outlines how Order and Search Sequence numbers are defined in MEDEF to establish the sequence order of application.

In MEDEF, each medical definition code is assigned an Order, which

  • Determines DIAMOND’s order of Global Search medical definition evaluation during claims processing and,

  • Groups multiple determinants within each medical definition code.

The methodology used to assign order numbers is critical. Within each claim type (Dental Services, Institutional, or Professional Services) DIAMOND searches for the MEDEF record for the lowest Order which is flagged "Include in Global Search" and evaluates the determinants associated with the medical definition code. If corresponding values in the claim line match the determinant and determinant values, DIAMOND assigns that medical definition to the claim detail line and stops searching. If a match is not found, DIAMOND proceeds to the next lowest Order flagged "Include in Global Search."

Therefore, the most specific medical definition codes should have the lowest Order numbers, so that more general definitions are not found and assigned to the claim line when a more specific medical definition is more accurate.

Using a Default Medical Definition

We recommend you create a "default" medical definition as the last, and therefore most general, medical definition in your processing sequence. This default medical definition code should be something that will stand out, such as "9999." If this code appears on a claim detail line, you can research why DIAMOND assigned this medical definition instead of a more specific medical definition, most commonly due to keying errors or errors in the submitted claim.

Using Diagnosis Codes as Determinants

If you want to use a diagnosis code as a determinant when creating a medical definition in keyword MEDEF, then it may be convenient to use DIANGOSIS_CODE_MASTER as the determinant table and DIAGNOSIS_CODE as the determinant. Typically, when you enter a claim, you enter the diagnosis codes in the Dx 1, Dx 2, Dx 3, or Dx 4 boxes on the header window. Then, when you enter a claim detail line on the detail window, you can use the Dx Ptr boxes to point to the diagnosis code on the header window that you want to become the "primary" diagnosis code for the claim detail line. That way, the primary diagnosis code on the claim detail line is what is used as the medical definition determinant value.
