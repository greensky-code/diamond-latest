Paying claims to subscribers instead of providers, How DIAMOND Thinks

You can pay a claim to the subscriber instead of the provider. DIAMOND will then use the subscriber’s name and address when creating the A/P record and payment check.

When you select to pay the subscriber, the Pay Sub box will be set to Y on the claim header window. When you save the claim, DIAMOND does the following:

  • Prices the claim according to the original provider selected on the claim.

  • Adds a vendor record in keyword VENDR (Vendor Master) with the subscriber’s name from keyword MEMBR (Member Master). The Vendor ID is prefixed with an "S." Any future payments to this subscriber or this subscriber’s dependents will be based on the information for this vendor record.

  • Adds a vendor address record in keyword VENDA (Vendor Address) with the subscriber’s address from keyword MEMBR (Member Master). If you later change the subscriber’s address in keyword MEMBR, then the subscriber’s vendor address record in keyword VENDA will automatically change.

DIAMOND uses the subscriber’s name and address when adding the A/P record in keyword ACPAY (Accounts Payable) and payment check in keyword CKPRT (Check Print Setup). DIAMOND changes the RA/EOB Print Flag list to 1 – Paper EOB and No RA on the A/P record in keyword ACPAY.

If you change information on the claim by using the Change Header command on the Special menu, then the Pay Sub box will be set to N. To pay the claim to the subscriber, again you must select the Pay Subscriber command on the Special menu.

For more information, see Paying a claim to the subscriber instead of the provider.
