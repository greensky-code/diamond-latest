Claims Processing with Submitted Invalid Data Overview

Submitting Invalid Data to DIAMOND

DIAMOND performs edits on all incoming claims; certain fields in each claim must be valid for DIAMOND to correctly process the claim. If any of these fields contain invalid data (such as a non-existent Member ID), the claim generates a critical error and is not inserted into DIAMOND. By substituting a placeholder for an invalid code, DIAMOND enables you to enter a submitted claim that has invalid codes. DIAMOND stores the original invalid code(s) separately.

Once a claim has been inserted into DIAMOND, you can view the claim information, including the invalid code(s), and replace the placeholder(s) with valid value(s).

Invalid Field Substitution

DIAMOND provides parameters to store the placeholder values that will substitute for a field containing invalid data. For each placeholder value, you must configure its associated parameter. For some parameters, you must provide a placeholder value in Parameter 2. For other parameters, DIAMOND inserts a null or other value if Parameter 1 is set to Y.

When creating placeholder values, you should use a value that does not resemble real data. For example: for invalid member IDs, you could create a member ID record named "INVLMB01", or a similar value. The value should not resemble a real value.

The following parameters require a placeholder value:

  • Admitting Provider ID: INVALIDADMPV

  • Attending Provider ID: INVALIDATTPV

  • Bill Type: INVALIDBILTY

  • Diagnosis Code: INVALIDDIAG

  • DRG Code: INVALIDDRGCD

  • Facility ID: INVALIDFACID

  • Member/Person Number: INVALIDMBRID

  • Modifier Code: INVALIDMODIF

  • Place of Service: INVALIDPOS

  • Procedure Code: INVALIDPROC

  • Provider ID: INVALIDPRVID

  • Referring Provider ID: INVALIDREFPV

  • Vendor ID: INVALIDVENDR

The following parameters insert a null (or blank) or other system-generated value:

  • Admit Hour: INVALIDADMHR (inserts a null value)

  • Admit Source: INVALIDADMSO (inserts a null value)

  • Admit Type: INVALIDADMTY (inserts a null value)

  • Discharge Hour: INVALIDDISHR (inserts a null value)

  • Occurrence Code: INVALIDOCCCD (inserts a null value)

  • Patient Status: INVALIDPATST (inserts "XX") The "XX" value is pre-defined in DIAMOND as "Invalid Patient Status."

For information on each parameter, see PARAM user documentation.

Note: If a replacement value has not been set up in DIAMOND (not just in the parameter), the replacement will not occur.

Hold Status for Claims with Invalid Fields

DIAMOND can place a Hold on a claim header that was submitted with invalid data. To enable DIAMOND to do this, set the parameter INVALIDHOLD. For information on this parameter, see PARAM user documentation.

If the parameter INVALIDHOLD is set, Holds are placed as follows

  • If the claim was submitted via the DDS API, DIAMOND holds the claim automatically.

  • If the claim is submitted online, and you cannot resolve the invalid data, you select the Invalid Data check box in the appropriate claims header (keyword PSCLM, INCLM, or DNCLM). DIAMOND then places a hold on the claim. You can then capture the submitted invalid data using the available commands on the menu bar.

After you have corrected the invalid codes in the claim, you can deselect Invalid Data in the claims header to indicate that the claim no longer contains invalid data.

Resolving Invalid Data in DIAMOND

DIAMOND provides the following commands on the Special menu to store data for claims that were submitted with invalid data:

INCLM, Institutional Claims

  • Submitted Claims Header/UB92 Data

  • Submitted Claims Provider/Vendor

  • Submitted Claims Detail Data

PSCLM, Professional Claims

  • Submitted Claims Header Data

  • Submitted Claims Provider/Vendor

  • Submitted Claims Detail Data

DNCLM, Dental Claims

  • Submitted Claims Header/Other Claim Info Data

  • Submitted Claims Detail Data

For information on each command, see the keyword reference documentation for each claim type.
