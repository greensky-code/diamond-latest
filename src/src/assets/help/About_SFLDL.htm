About SFLDL

Use keyword SFLDL (Field Level Security) to set up field level security for DIAMOND users. You can either set up field level security for a single user or set up a template that can be applied to multiple users. Using SFLDL, you specify which database columns a user may view but not change, and which database columns a user may not view.

Because field level security is applied to a database table and column, the security level applies to all fields that display data from the column. For example, restricting access to a field in a keyword also restricts access to the same field in an associated view, such as CLDSP (Claims Display). Note that access to fields in DIAMOND Special Functions cannot be restricted using Field Level Security.
