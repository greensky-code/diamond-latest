Benefit Accumulation Using Weighted Values - Overview

Accumulating benefits using weighted values is an option that enables you to accumulate a member's benefits at a reduced or an increased amount. For example, you might want to accumulate benefits at 50% of the standard rate or at 150% of the standard rate. (The standard accumulation rate is one accumulation point for every unit.)

You can set weighted value benefit accumulations at the Benefit Rule or the Procedure Code level. A member's weighted accumulated benefits are appropriately adjusted when a claim line is adjusted or reversed.

Create Benefit Weighted Accumulator records in BENWT (Benefit Weighted Accumulators). Here, you define one or more detail lines for each record. Each detail line includes procedures - you can define a single procedure code, a range of procedure codes, a single primary procedure group, or a single secondary procedure group. For each detail line, you also define an effective date range and a weighted accumulator percentage.

The Benefit Weighted Accumulator ID field on the BENEF screen links Benefit Weighted Accumulator records to Benefit Rules within a Benefit Package.   If you have not defined an accumulator, the system searches PROCD to find a weighted value greater than 00.00 and if not found, applies standard one-to-one accumulation.

On BENEF detail, if Benefit Weighted Accumulator ID is selected, the system compares the claim's procedure code and date of service against the Benefit Weighted Accumulator records. To be considered a match, the claim's procedure code must match either the procedure code, procedure code range, primary procedure grouping, or secondary procedure grouping and the claim's date of service must fall within the record's effective date period.

Screens Used to Set Up Weighted Value Benefits:

BRULE (Benefit Rules)

Use BRULE to set up benefit rules - see requirements in table above

 

BENWT (Benefit Weighted Accumulators)

Use BENWT to define Benefit Weighted Accumulators, which include procedure codes, effective date ranges, and weighted accumulator %.

 

BENEF (Benefit Packages)

Use BENEF to link Benefit Weighted Accumulators to benefit packages

 

PROCD (Procedure Codes)

Use PROCD to specify weighted values for procedure codes.
