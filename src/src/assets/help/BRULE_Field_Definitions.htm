All Groups?

O

Indicates whether the member’s group is relevant to accumulating toward the following:

  • Maximum Copayment

  • Maximum Coinsurance

  • Deductible

  • Out of pocket maximum, or

  • Full coverage dollar or quantity limit.

Selected from check box

Displayed values are:

  • Cleared: The accumulated benefit records must have a group code that matches the group on the current claim.

  • Selected: The accumulated benefit records need not have a group code that matches that on the current claim.

Amount

O

Quantity or dollar amount which, with Limit Type, define the limit to this benefit.

Manually entered

If you type a dollar amount, then you can type up to 16 digits before the decimal and 2 after the decimal.

If you type a quantity, then you can type 14 digits before the decimal and 1 after the decimal. Unless the insertion point is in this box, then only 4 digits before the decimal and 1 after the decimal display.

If the quantity or dollar amount on the current claim line straddles the Amount, then the claim is held and the message "Manual adjudication required" displays.

Once the full coverage dollar or quantity limit amount is reached, DIAMOND continues to search for other applicable benefit rules

Apply only if Quant>=and Quant <=

O

Minimum and maximum number of services for which the copay is applied.

Manually entered

You can type 14 digits before the decimal and 1 after the decimal. Unless the insertion point is in this box, then only 4 digits before the decimal and 1 after the decimal display.

If the quantity on the current claim line straddles minimum or maximum, the claim is held and the message "Manual adjudication required" is displayed.

Carryover

C

Period by which the time frame is extended prior to its beginning. Used in conjunction with No. Years/Days.

Selected from predefined list

Required unless the Time Frame list is set to Life.

Valid values are:

  • None

  • Days

  • Years

Children Count Individually?

O

Indicates whether each child within a family must meet the individual deductible.

Selected from check box

Used only if an amount was entered in the Individual Deductible box.

Claim Type

R

Type of claim applicable to accumulators used to determine if the

  • Maximum Copayment

  • Maximum Coinsurance

  • deductible or

  • Out of pocket maximum

  • Full coverage limit

have been reached.

Selected from predefined list

Valid values are:

  • Dental: Only dental services claims (DNCLM).

  • Prof Svcs: Only professional services claims (PSCLM).

  • Institutional: Only institutional claims (INCLM).

  • Prof + Inst: Professional services and institutional claims.

  • Prof + Dental: Professional services and dental claims.

  • Inst + Dental: Institutional and dental claims.

  • Prof + Inst + Dental: Professional services, institutional, and dental claims.

Coinsure %

O

Percentage applied to the claim detail line allowed amount (net of other benefit rules applied before the current coinsurance rule, per the order specified in the benefit package) to calculate the amount of coinsurance.

Manually entered

Sample entries:

.2 = 20%

.35 = 35%

Copay Amount

O

Dollar amount of the copayment.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Count Toward Max

R

Indicator of whether reason codes have been identified as either included or excluded in the accumulation of copayments or coinsurance toward the Maximum Copayment or Maximum Coins.

System generated

Displayed values:

  • Some: Specific reason codes have been entered using the Count Toward Max command on the Special menu. Only benefit accumulators which meet the conditions specified using the Count Toward Max command (inclusions or exclusions), the Med Def command, and the Other Med Defs command specifications are counted toward the out of pocket maximum or deductible.

  • All: No reason codes have been entered using the Count Toward Max command on the Special menu. This results in no accumulators being selected for evaluation, because no line other than the current line has a blank reason code.

This field is disabled for Type 80, Full Coverage Limit Rules.

Family Deductible

O

If deductible applies to the family as a whole (Member/Family field), the amount of that deductible.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.

From

C

Claim date from which DIAMOND calculates benefit renewal periods (Institutional claims only).

Selected from predefined list

Valid values are:

  • Adm Date (from claim header)

  • Disch Date (from claim header)

For more information, see Benefit Rules and Packages, How DIAMOND Thinks.

Full Coverage Type

R

Nature of the full coverage benefit. Specifies which accumulators DIAMOND uses to determine whether the limit has been reached.

Selected from predefined list

Valid values are:

  • Allowed: Total allowed less the not covered amounts are summed.

  • Quantity: Quantity of services from the claim detail line is summed.

  • Billed: Billed Amounts from the claim detail line are summed.

  • Lesser of Billed or Allowed: The sum of Billed or Allowed, whichever is less.

  • Days: The number of unique service dates from the claim detail lines is summed.

Hold Reason Code

O

If applicable, the hold reason that is entered on the claim detail line when this rule is applied.

Selected from predefined list

Hold reason codes are maintained in keyword REASN (Reason) with the Reason Code Type list set to HD – Hold.

Individual Deductible

O

If deductible applies to the individual member only (Member/Family field), the maximum amount of that deductible.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Limit Type

R

Nature of the benefit limit. Specifies which accumulators DIAMOND uses to determine whether the limit has been reached.

Selected from predefined list

Valid values are:

  • Quantity: Quantity of services from the claim detail line is summed.

  • Billed: Billed Amounts from the claim detail line are summed.

  • Allowed: Not Covered: The Allowed Amounts minus the Not Covered Amounts are summed.

  • Net: The Net Amounts from the claim detail lines are summed.

  • Days: The number of unique service dates from the claim detail lines is summed.

  • Weighted Values: X-ray points from the claim detail lines are summed.

Max Out of Pocket Amount

O

Maximum out of pocket expense applicable to this member or family.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.

If more than one amount on the current claim line (copayment, coinsurance, or deductible) causes the maximum to be reached, the amounts are processed in the following order until the maximum is reached:

  • Deductible

  • Copayment

  • Coinsurance

Maximum Coins

O

Maximum total of this coinsurance applicable to this member or family.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.

DIAMOND reviews member or family accumulators to determine if the maximum has been reached; when reached, the copayment on the detail line is set to $0.00; the Copay Rsn box is left blank.

Held and denied detail lines are not included in determining whether this maximum has been reached.

Maximum Copayment

O

Maximum total of this copayment applicable to this member or family.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.

DIAMOND reviews member or family accumulators to determine if the maximum has been reached; when reached, the copayment on the detail line is set to $0.00; the Copay Rsn box is left blank.

Held and denied detail lines are not included in determining whether this maximum has been reached.

Med Def

D

Displays Some after you have selected one or more medical definition codes to trigger this benefit rule. To select a medical definition code, on the Special menu, click Med Def.

Displayed

You must select at least one medical definition code to include or exclude on a benefit rule.

Member/Family

R

Indicates whether to include an individual member’s accumulated benefits only or the combined accumulated benefits for all members within the family in determining whether the following has been reached:

  • Maximum Copayment

  • Maximum Coinsurance

  • Deductible

  • Out of pocket maximum, and/or

  • Full coverage limit

Selected from predefined list

Valid values are:

  • Member: Include the member’s accumulated benefits only.

  • Family: Include the member’s and all associated family member’s accumulated benefits.

Message

O

Message that is displayed to the claims processor when this rule is applied.

Manually entered

The corresponding Message ID 2054 must be set up for display in keyword MSGMT (Message Master Maintenance) and PMSCT (Processing Messages Control).

Multiply by Quantity?

Rule Type 00

O

If selected, the copayment amount is multiplied by the number of services on that detail line. Otherwise, a single copayment is applied regardless of the quantity.

Selected from check box

 

Multiply by Quantity

Rule Type 20

O

Identifies the accumulation requirement when Limit Type = Weighted Value.

Selected from check box

This field is only accessible if the Rule Type is 20 (Limits) and the Limit Type is Weighted Value.

Checked - the weighted value is multiplied by the claim quantity.

Not checked - Accumulation is based on the weighted value and the quantity is not included in the calculation when accumulating benefits.

Narrative

O

Full text description of the rule.

Manually entered

 

No. Years/Days

C

1. The number of days or years applied to Time Frame.

2. The number of days or years applied to Carryover.

Manually entered

Required unless the Time Frame list is set to Life or the Carryover list is set to None.

Two separate fields with this label appear on the screen, one associated with Time Frame, one associated with Carryover.

Although you can type 0 (zero), this will cause unpredictable results.

Other Med Defs

D

Indicator of whether other medical definitions have been identified through the Special menu command, Other Med Defs.

System generated

Displayed values:

  • Some: Specific medical definitions have been entered using the Other Med Defs command on the Special menu. Only benefit accumulators that meet the conditions specified using the Other Med Defs command and Med Def command that meet the Reason code conditions defined using the Count Toward Max command are counted towards the deductible amount specified in this rule.

  • All: No medical definitions have been entered using the Other Med Defs command.

This field is disabled for Type 80, Full Coverage Limit Rules.

Per Admit Deductible

O

Indicates if deductible is applied to each inpatient hospital admission.

Selected from check box

Applied only to institutional claims with bill type starting with 11.

Per Provider

O

Indicates that a dental benefit limit be applied on a per member per provider basis (instead of the usual per member basis).

Selected from check box

 

Reason Code

R

Reason code that will be automatically applied to the detail line when this rule is applied.

Selected from predefined list

Reason codes are maintained in keyword REASN with a type:

  • CP: Copayment and coinsurance

  • DD Deductible

  • FC Full Coverage

  • NC: Not Covered, Out of Pocket Max & Hold Reasons

We recommend a different Reason code be set up for each rule.

Renewal

C

Period of time category DIAMOND will use when calculating benefit renewal periods (Institutional claims only).

Selected from predefined list

Valid values are:

  • None (default) - The benefit does not have renewal periods.

  • Elapsed Days*

  • Months*

  • Years*

For more information, see Benefit Rules and Packages, How DIAMOND Thinks.

Rule ID

R

Unique identifier of the rule.

Manually entered

You can type up to 10 alphanumeric characters.

Short Desc

R

Brief description of the rule.

Manually entered

 

Time Frame

R

Type of time span over which DIAMOND reviews accumulated benefits to determine if the following have been reached:

  • Maximum Copayment

  • Maximum Coinsurance

  • Deductible

  • Out of pocket maximum, or

  • Full coverage limit

Used in conjunction with the No. Years/Days box.

Selected from predefined list

Valid values are:

  • Calendar Yr: January 1 through December 31.

  • Contract Yr: If the member has a Benefit Start Date, the year defined by that date plus 364 days (or, for a leap year, 365 days). If no Benefit Start Date, the year defined by the Effective Date and Term Date on the group contract (GRUPC) record.

  • Elapsed Yrs: A number of years prior to the detail claim line date of service.

  • Life: All accumulated benefits in DIAMOND.

  • Elapsed Days: A number of days prior to the detail claim line date of service.

Type

R

Category of benefit rule that determines what fields are displayed for entry.

Selected from predefined list

Valid values are:

  • 00 Copay

  • 10 Coinsurance

  • 20 Limits

  • 30 Deductible

  • 40 Out-of-Pocket Maximum

  • 50 Message & Hold

  • 60 Exclusions

  • 70 Utilization Review

  • 80 Full Coverage/First Dollar

YrsMonthsDays

Manually entered
