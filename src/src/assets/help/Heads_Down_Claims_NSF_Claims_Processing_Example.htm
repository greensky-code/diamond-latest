Receiving NSF Claims

Clearinghouses and/or partners may submit claims to your organization using the National Standard Format (NSF). You must use third-party software, Sterling® for example, to re-format NSF claim files before the file is processed through the IDE module. This process re-maps the NSF fields and creates a DIAMOND readable IDE module input file (<filename>.dat).

Each claim within the NSF file must have a unique claim number. If possible, it is recommended that all claims received from a specific source, over a period of time, have unique claim numbers.

Processing NSF Claims

The UNIX directories, necessary to process NSF claims and run the Intelligent Data Enhancer (IDE), are created when you install the current release of DIAMOND.

Step

Comment

1. Process the NSF claims file using third-party software to create a re-mapped claims file (<filename>.dat).

 

2. If necessary, move (FTP) the re-mapped <filename>.dat claims file to a UNIX directory containing the IDE script (ideexec.sh) and the NSF control file (psclmnsf.ctl).

When you run the IDE script, <filename>.dat, ideexec.sh, and psclmnsf.ctl must be in the same UNIX directory.

Running the IDE Script for NSF Claims

You must run the IDE script from within the directory in which it is stored. In addition, the <filename>.dat and hdsdwn.ctl files must be in the same directory as the IDE script (ideexec.sh).

Step

Comment

1. To run the IDE script from the host

Enter = ideexec.sh

2. Prompt = username:

Enter = <DIAMOND 950CS username>

3. Prompt = password:

Enter = <DIAMOND 950CS password>

4. Select the NSF control file (psclmnsf.ctl).

 

5. Enter the name of the claims text file.

Enter = <filename>.dat

Enter the <filename> only. Do not enter a full path.

6. A "script completion" message is displayed.

The output of the IDE script is a new data file named D_<filename> (where <filename> is the same as the <filename>.dat claims text file). The D_<filename> file is the source (input) file for DIAMOND’s Claims Batch Process.

You must process each D_<filename> file completely through DIAMOND’s Claims Batch Process. You cannot concatenate separate D_<filename> files and attempt to use the concatenated file as the source (input) for the Claims Batch Process.
