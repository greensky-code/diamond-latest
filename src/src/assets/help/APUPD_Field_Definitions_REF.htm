APUPD Field Definitions

Field Name

Req

Description

Source

Comments

Action

D

Action taken by DIAMOND.

System generated

Set to "run" because the request type is immediate. (‘Deferred’ is not available.)

Admin Fee

R

Indicates whether administrative fees are included in the A/P update job.

Selected from predefined list

You can select one of the following values:

  • E – Exclude Admin Fee

Exclude administrative fees when running an A/P update job to process medical claims.

  • I – Include Admin Fee

Include administrative fees when running an A/P update job to process medical claims.

 

 

The dates entered in the Entered Date, Thru Entered Date, From Due Date, and Thru Due Date boxes are ignored for administrative fees

  • O – Only Admin Fee

Only process administrative fees when running A/P update job. Medical claims are ignored.

 

 

The dates entered in the Entered Date, Thru Entered Date, From Due Date, and Thru Due Date boxes are ignored for administrative fees

Comments

O

Additional information about the batch job.

Manually entered

Comments are particularly useful on batch jobs that are saved as templates.

Company Code

O

The general ledger company associated with the payable transactions selected.

Manually entered

F5 lookup is available.

Leave blank to update qualifying payables for all companies.

This field is required if the Parameter 1 box is set to Y in keyword PARAM for the APUPDCOMPCD parameter.

Date

D

Date the batch job s run.

System generated

 

From Due Date

O

Based on the due date in VENDR – Payment Delay Days.

Manually entered

If the Vendor has a value in Payment Delay Days on the Vendor Master Record, the due date is calculated by adding that number of days to the Claim Received date.

If no value is in the Vendor Master, the system parameter CLAIMDUEDATE is read and the due date is calculated by adding that number of days to the Claim Received date.

If no value is in the parameter, the due date is the date of the APUPD run.

Enter in MMDDYY format. Leave this field blank to update all qualifying payables regardless of the due date.

From Entered Date

O

For claims, the date that the claim detail line was entered in the system.

For capitation, the Insert Date on the ACPAY record.

Manually entered

Enter in MMDDYY format.

If a date is entered in this box, but a date is not entered in the Thru Entered Date box, then keyword APUPD will just use the date entered in this box.

From Received Date

O

If you want to include medical claims received by the health plan on or after a particular date, then type the from received date.

The date a claim was received by a health plan displays in the Date Recv box on the claim header window in keyword DNCLM, INCLM, or PSCLM.

Manually entered

Type in MMDDYY format.

Include FINAL

O

Indicates whether to finalize informational, capitation, or staff claim detail lines; same functionality as keyword FINAL (Final Non AP Setup).

Selected from check box

If you click to select this check box, claim detail lines with the Claim Status list set to I-Informational, C-Capitation, or S-Staff in the claim detail window are finalized.

Claim lines are finalized according to the logic used in keyword FINAL.

Job ID

R

Unique identifier for this batch job.

System generated

 

Payable Type

O

Type of payable to be processed.

Selected from predefined list

Types include:

  • C Capitation

  • M Medical

  • A Agent Commission (future functionality)

Post Month

O

Type the G/L month for which you want to post the job.

Manually entered

 

Request Type

R

When the batch job is to be run.

Selected from predefined list

Valid selections are:

  • Immediate: Indicates the job should run immediately when File>Run Job or CTRL+J is entered.

  • Deferred: Future functionality

Request User

D

The logon ID of the user entering the APUPD job request.

System generated

 

Status

D

The status of the batch job.

System generated

Status indicators:

  • New: This batch job is new. The batch job has been saved; however, the job has not been run.

  • In Process

  • Successful Run

  • Error

Thru Due Date

O

Use this field if a "from due date" is specified.

Manually entered

Must be greater than the date in the From Due Date box.

Type in MMDDYY format.

Thru Entered Date

O

For claims, the date that the claim detail line was entered in the system.

For capitation, the Insert Date on the ACPAY record.

Manually entered

Must be greater than the From Entered Date box.

Type in MMDDYY format.

Thru Received Date

O

If you want to include medical claims received by the health plan through a particular date, then type the through received date.

The date a claim was received by a health plan displays in the Date Recv box on the claim header window in keyword DNCLM, INCLM, or PSCLM.

Manually entered

Must be greater than the date in the From Received Date box.

Type in MMDDYY format.

Vendor ID

O

ID for the vendor receiving the payables.

Selected from predefined list

F5 lookup is available.

Leave blank to update qualifying payables for all companies.

Up To Amount

O

Type the maximum payable amount to be included in this update. This amount is the net amount from claims.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.
