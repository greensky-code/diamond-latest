BORDR Field Definitions

Field Name

Req

Description

Source

Comments

Benefit Type

R

The benefit type corresponding to the processing order.

Selected from predefined list

Valid values are:

  • Coinsurance

  • Copay

  • Deductible

  • Exclusion

  • Limit

  • Message and Hold

  • Out of Pocket Maximum

  • Utilization Review (future)

Default

O

Indicator that this processing order set is the default when setting up a benefit package (BENEF).

Selected from check box

A default is recommended. If no default is established, a Processing Order ID must be selected for each benefit package. Only one default is allowed.

Description

O

Text description of this processing order set.

Manually entered

 

Processing Order

R

The order (ascending) in which the selected benefit type will be evaluated during claims adjudication.

Selected from predefined list

If you enter the same order for all benefit types, each benefit rule is evaluated in the sequence order defined in the benefit package (BENEF).

Valid values are:

  • 10

  • 20

  • 30

  • 40

  • 50

  • 60

  • 70

  • 80

Processing Order ID

R

Unique identifier of this processing order set.

Manually entered
