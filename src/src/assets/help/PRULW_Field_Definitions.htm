PRULW Field Definitions

Field Name  Req Description Source Comments

Allowed Factor

R

Amount entered will be multiplied by the amount calculated by this rule.

Manually entered.

 

Allowed Reason

O

Reason code automatically applied to the detail line generated by the Base Price Rule.

Selected from a predefined list.

Allowed reasons are maintained in keyword REASN (Reason) with the Reason Code Type list set to AL – Allowed.

Amt Type

R

Definition of the From and Thru Amt fields.

Selected from a predefined list.

Valid values are:

Billed

Allowed

Net

Net + Withhold

Apply Bucket

R

Temporary holding place, or bucket, in which DIAMOND stores the Base Price Rule calculated amount during calculation of the claim using this rule.

Selected from a predefined list.

Valid values are:

Results to Bucket A

Results to Bucket B

<blank>: Calculated amount not stored.

Bucket Logic

R

Relationship of the calculation results from this rule and the amounts stored in bucket as instructed in Apply Bucket.

Selected from a predefined list.

Valid values are:

(+) Add To: Current calculation is added to the appropriate bucket.

(-) Subtract: Current calculation is subtracted from the appropriate bucket.

R Replace: Replace the current calculation with the amount in the appropriate bucket.

Calc Method

R

Calculation method used when this rule is employed. Different methods are available for selection depending on whether the rule type is Base Price, Outlier or Cap.

Selected from a predefined list.

Valid values are listed below. The subset of methods available for each rule type is shown:

Base Price Rules:

01 Per Diem or Fee Schedule

02 Case Rate

03 Whole Claim % of Billed

04 DRG Pricing

05 Tiered Per Diem

06 Tiered Allowed Amount

18 > of bucket A or B

19 < of bucket A or B

20 Pay A

21 Pay B

23 Sum of Allowed

98 Message

Custom Do not use.

Outlier Rules:

%Bill PROVC: The Pct of Billed from PROVC, PROCP, or GRUPD is applied to the billed amount that exceeds the threshold.

%Bill WCPR: The Allowed Factor is applied to the billed amount that exceeds the threshold.

Flat Amount: The Allowed Amt from the PROCP record corresponding to the Sys Proc Code is the Outlier payment amount.

Per Diem: The Allowed Amt from the PROCP record corresponding to the Sys Proc Code is multiplied by the number of days above the threshold. If the Calc Method is 23, Per Diem, then the Threshold Qnt field must be greater than zero.

Multiply DRG: The DRG Outlier % of Billed from PROVC is multiplied by the billed amount that exceeds the threshold. A message is displayed to the operator and no outlier payment is calculated.

Custom. Do not use.

Cap Rules:

Neg clm line: The generated claim line shows an allowed amount equal to the cap amount. A message is displayed to the operator and no cap adjustment is calculated.

Custom. Do not use.

Cap %

R

If cap is based on a percentage, further defined in Method (Cap %), the percentage.

Manually entered.

Field is only used if cap is a percentage amount; not used if cap is a dollar amount.

Cap Amount

R

If cap is based on the total dollar amount, further defined in Method (Cap Amt), the dollar amount.

Manually entered.

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Field is only used if cap is a dollar amount; not used if cap is a percentage.

Cust Prog Name

 

Not used.

Not used.

 

Description

O

Description of the rule.

Manually entered.

 

Effective Date

R

Effective date for the whole claim price rule record.

Manually entered.

 

From Amt

O

Lowest amount of the dollars defined in Amt Type In a range of dollar amounts to which this rule applies.

Manually entered.

You can type up to 16 digits before the decimal and 2 digits after the decimal.

If blank, application of this rule is not based on total dollar amount, by Amt Type, of the claim.

From Days

O

First day of an inpatient stay in a length of stay range.

Manually entered.

If blank, indicates rule is not applied based on the day of the admission.

Hold Reason

O

Hold reason code automatically applied to the detail line generated by the Base Price Rule.

Selected from a predefined list.

Hold reasons are maintained in keyword REASN with the Reason Code Type list set to HD – Hold.

LOS Method

C

Method used to calculate the total length of stay for the claim.

Selected from a predefined list.

Required if From Days has an entry.

Valid values are:

H Days on Header: DIAMOND calculates the LOS by subtracting the claim discharge date (INCLM>Disch Date) from the claim admit date (INCLM>Adm Date). If the difference falls within the From Days/Thru Days range entered above, DIAMOND applies the base rule to the claim.

D Days on Detail: DIAMOND calculates the LOS by summing all claim procedures that have been defined as "count as days" procedures (PROCD>Count As Days) on DIAMOND. If the sum of day procedures falls within the From Days/Thru Days range, DIAMOND applies the base rule to the claim.

Method ($ Threshold)

R

Method used to determine if the $ threshold has been met.

Selected from a predefined list.

Valid values are:

02 Billed > Allowed by X: If the billed amount equals or exceeds the allowed amount by at least the Threshold $, the outlier rule is applied.

03 Billed > X: If the billed amount equals or exceeds the Threshold $, the outlier rule is applied.

04 Allowed > X: If the allowed amount equals or exceeds the Threshold $, the outlier rule is applied.

05 Allowed > Billed by X: If the allowed amount equals or exceeds the billed amount by the Threshold $, the outlier rule is applied.

08 Billed > DRG by X: If the billed amount exceeds the DRG Base Outlier times the DRG Base Weight by at least as much as the Threshold $, the outlier rule is applied.

Method (Cap %)

R

Method used to determine if the Cap % has been reached.

Selected from a predefined list.

Valid value is

03 Cap if Allowed is > Billed by X%: If the claim allowed amount exceeds the claim billed amount, the allowed amount is adjusted with a generated detail line to equal billed.

Method (Cap Amt)

R

Method used to determine if the Cap Amount has been reached.

Selected from a predefined list.

Valid values are:

01 Cap at Dollars Allowed: If the claim allowed amount is greater than or equal to the cap amount, the adjustment line that reduces the allowed amount is generated.

02 Cap if Allowed is X>Billed: When the claim allowed amount exceeds the claim billed amount, the adjustment line is generated which reduces the allowed amount to one cent ($.01) greater than the billed amount.

Method (LOS Threshold)

R

Method used to calculate the length of stay (which is then compared to Threshold Qnt to determine if an additional payment is allowed).

Selected from a predefined list.

Valid values are:

Admit-Discharge: The discharge date is subtracted from the admit date. If the difference is =/> the days entered in the Threshold Qnt, DIAMOND will apply the Outlier rule.

# Claim lines with Proc: Total number of detail lines with a procedure code which is counted as a day (see PROCD reference documentation)

Method (Pct Threshold)

R

Method used to calculate the Threshold Pct.

Selected from a predefined list.

Valid values are:

01 Billed > X% of Allowed: If the claim billed amount is greater than the allowed amount by the Threshold Pct, the outlier rule is applied.

05 Discount Exceeds X%: If the allowed amount exceeds the billed amount by the Threshold Pct, the outlier rule is applied.

Msg to Oper

O

Message which is displayed to the claim processor during claim entry.

Manually entered.

If claims are processed using EDI Claims, this message is printed on the EDI transaction reports.

Patient Age From

O

Earliest age of the member age range to which this rule applies.

Manually entered.

 

Patient Age To

O

Oldest age of the member age range to which this rule applies.

Manually entered.

 

Pct Allowed?

O

If selected, the Percent Allowed feature will be applied to the amount calculated by this rule.

Selected by checkbox.

 

Price Determinant

R

Code representing the type of claim to which this rule applies.

Selected from a predefined list.

Price Determinants are maintained in keyword PDTER (Price Determinant Maintenance).

Rule Type

R

One of three kinds of whole claim price rules.

Selected from a predefined list.

Valid values are:

Base Price: Defines the primary pricing method; generates an additional detail line with the allowed amount, reason code, and procedure code.

Outlier: Defines a method of calculating an additional payment when the claim has exceeded a threshold dollar amount.

Cap: Defines a method of calculating and applying an adjustment when the Base Price exceeds a ceiling.  

Seq No.

R

Sequence in which DIAMOND will evaluate each Rule Type within the price rule.

Manually entered.

Within a price rule, rules types are evaluated in the following order: Base Price, Outlier, Cap. Within each rule type, rules are evaluated in Seq No. order.

Summing Dtls

R

Identifies which claim detail lines are included in automatically calculated claim totals of Billed Amount, Allowed Amount, Net Amount, Withhold Amount, and Quantity.

Selected from a predefined list.

Valid values are:

All but denied: Sum all claim detail lines except those that are denied.

Info Only: Sum all informational claim detail lines only.

All but denied or info: Sum all detail lines except informational and denied lines.

<blank>: No lines are summed.

Sys Proc Code

C

Procedure code which will be automatically assigned to the generated detail line.

Selected from a predefined list.

Procedures are maintained in PROCD.

Required if Calc Method is 01-06, 20, 21, 23, 99. Not used if Calc Method is 18, 19, 98.

Term Date

O

Last date the rule is effective.

Manually entered.

 

Threshold $

R

Dollar amount, further defined by Method ($ Threshold) over which an additional payment will be allowed.

Manually entered.

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Field is only used if threshold is based on a dollar amount; not used if Threshold Qnt or Threshold Pct is entered.

Threshold Pct

R

Dollar amount, further defined by Method (Pct Threshold), over which an additional payment will be made.

Manually entered.

Field is only used if threshold is based on a percentage; not used if Threshold Qnt or Threshold $ is entered.

Threshold Qnt

R

Number of days in the stay over which an additional payment will be allowed.

Manually entered.

Field is used only if threshold is based on length of stay; not used if Threshold $ or Threshold Pct is entered.

If the Calc Method is 23, Per Diem, then the Threshold Qnt field must be greater than zero.

Thru Amt

O

Highest amount of the dollars defined in Amt Type In a range of dollar amounts to which this rule applies.

Manually entered.

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Thru Days

O

Last day of an inpatient stay in a length of stay range.

Manually entered.

 

Tiered Amt From

C

Lowest dollar amount for the range associated with Tiered Seq no.

Manually entered.

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Required if Calc Method is 06, Tiered Allowed Amount, and not used if any other Calc Method is entered.

Tiered Amt Thru

C

Highest dollar amount for the range associated with the Tiered Seq no.

Manually entered.

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Required if Calc Method is 06, Tiered Allowed Amount, and not used if any other Calc Method is entered.

Tiered Days From

C

First day of the range of days associated with the Tiered Seq no.

Manually entered.

Required if Calc Method is 05, Tiered Per Diem, and not used if any other Calc Method is entered.

Tiered Days Thru

C

Last day of the range of days associated with the Tiered Seq no.

Manually entered.

Required if Calc Method is 05, Tiered Per Diem, and not used if any other Calc Method is entered.

Tiered Seq No.

C

"Step" order for calculation of allowed amounts based on dollar levels or length of stay ranges.

Manually entered.

Required for Base Price rules; not used for Outlier or Cap rules.

The first Tiered Seq No. must be 00.

The values for the tiers are entered in Tiered Days From, Tiered Days Thru, Tiered Amt From, and Tiered Amt Thru fields.
