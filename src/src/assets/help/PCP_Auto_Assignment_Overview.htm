PCP Auto Assignment - Overview

PCP Auto Assignment is a batch process by which you can:

  • Validate/Replace a submitted PCP assignment

  • Automatically assign a suitable PCP to a current member eligibility record

PCP Auto Assignment validates and makes PCP assignments according to sets of rules you define. You define individual PCPAA rules once and then combine them into sets with appropriate values for the applicable lines of business. PCPAA is a supplement to the existing Provider Enrollment Rules (PROVE) functionality.

To use PCP Auto Assignment, you first identify the lines of business and PCPs for which you want to use PCP Auto Assignment functionality. Then for each line of business and PCP you determine and enter appropriate values, based on your business requirements, for the key data used exclusively by PCP Auto Assignment. (Additional existing member and provider data is of course used as well.) Lastly, you run a PCP Auto Assignment batch job to assign a PCP or validate a submitted PCP for selected member eligibility records.

PCP Auto Assignment Setup - Overview

At a high level, the main steps for setup of key data required for PCP Auto Assignment includes the following:

1. Determine and enter appropriate values for the system parameters:

  • ENROLLLIMITS (PARAM) - Use to specify what enrollment limits (PROVE, PROVF, LINBS) to evaluate and in what order. Default is PROVE only.

  • PCPAAFAMUSRD (PARAM) - Use to enable use of MEMBR user-defined fields to identify family affiliations.

2. Determine and enter for use across included lines of business:

  • Rules for PCP Auto Assignment to validate or replace a submitted PCP (PCPRL)

  • Rules for PCP Auto Assignment to automatically assign a PCP (PCPRL)

3. Determine and enter for each individual included line of business:

  • A set of rules for PCP Auto Assignment to validate or replace a submitted PCP (PCPAA)

  • A set of rules for PCP Auto Assignment to automatically assign a PCP (PCPAA)

  • A PCP maximum enrollment limit and threshold applicable to each individual PCP for the line of business (LINBS)

  • Default PCP, No Auto PCP Found, ID Card Reinst Days, and Family Affiliation specifications (LINBS)

  • Overrides to enrollment thresholds (PCPAA)

  • Priority PCP assignment selections (PCPAA)

  • Pediatric, OB/GYN, and other Specialty restrictions by age and gender (PCPSI)

4. Determine and set up for each included provider:

  • PROVE Available for Auto Assign checkbox (PROVE)

Note:

The default is for the check box to be selected, indicating that the provider is available for PCP auto assignment. You must manually clear the check box for providers you want to exclude from auto assignment.

  • Provider PCP enrollment limits at the provider enrollment and provider master levels (PROVE, PROVF)

  • LINBS (Line of business)

PCP Auto Assignment Processing Logic - Overview

When you run a PCP Auto Assignment batch job using PCPJB, DIAMOND performs the following steps.

1. Select member eligibility records for PCP Auto assignment per PCPJB criteria

2. Process each selected member eligibility record in turn as follows

3. If the member eligibility record includes a PCP (who is not the LINBS Default PCP), go to Event 1 - Validate/Replace Submitted PCP.

4. If the member eligibility record does not include a PCP (or includes the LINBS Default PCP), go to Event 2 - Replace a Default PCP or No PCP Selection.

Event 1 - Validate/Replace Submitted PCP:

Validate the Submitted PCP. If not valid, search for a valid PCP to replace the Submitted PCP.

1. Evaluate the submitted PCP against key fixed criteria, PCPAA Basic Rules for the member’s line of business, and PROVE, PROVF, and LINBS enrollment limits as specified in system parameter ENROLLLIMITS. (Threshold capacities are automatically overridden for the Submitted PCP.)

If satisfy all criteria, the submitted PCP assignment is valid. Set internal flags to reflect that PCP Auto Assignment has been performed for this member eligibility record and the date it was performed.

If don’t satisfy all criteria, search for another PCP as follows.

(Search for PCP to Replace Submitted PCP)

2. Evaluate priority selections in the PCPAA Evaluate Possible Selections section box of the PCPAA record for the member’s line of business. (For example, you can specify a priority selection of Member Reinstated to assign the same PCP as the reinstated member had for a prior eligibility record.)

If one or more records qualify as a possible priority assignment, continue and validate the qualifying records for potential assignment.

If no records qualify as a possible priority assignment, search for a PCP. See Search for PCP to Replace Submitted PCP below.

3. Validate the priority selection records against the PCPAA Basic Rules for the member’s line of business, the PROVE/PROVF/LINBS enrollment limits and threshold capacities, and the Override Threshold Capacity Levels selections. (For example, you can specify to override the enrollment level threshold capacities if the selected PCP was the Member’s previous PCP on a prior eligibility record.)

If only one record satisfies all criteria, assign the PCP on the record to the current member eligibility record. Update the PCP’s PROVE enrollment count and set an internal flag that PCP Auto Assignment has been performed for this member eligibility record.

If more than one record satisfies all criteria, apply the Tiebreaker Rules for the member’s line of business to the PCPs on the qualifying records. If a single PCP is selected using the tiebreakers, assign the PCP to the member’s eligibility record. If a single PCP is not selected by the Tiebreaker Rules, use a random process to pick a single PCP and then make the assignment. Also update the submitted and assigned PCPs PROVE enrollment counts and set internal flags to reflect that PCP Auto Assignment has been performed for this member eligibility record and the date it was performed.

If no records satisfy all criteria, proceed to Event 2 - Replace a Default PCP or No PCP Selection below.

Event 2 - Replace a Default PCP or No PCP Selection

Search for an appropriate PCP to replace the Default PCP or if no satisfactory PCP is submitted.

1. Evaluate potential PCPs against key fixed criteria.

If one or more PCPs satisfy all criteria, continue.

If no PCPs satisfy all criteria, assign the No Auto PCP Found ID from the member’s line of business record to the member’s eligibility record and set an internal flag indicating that PCP Auto Assignment has been performed for this record.

2. Evaluate potential PCPs against the PROVE/PROVF/LINBS enrollment limits and threshold capacities, and the Override Threshold Capacity Levels selections.

If one or more PCPs satisfy all criteria, continue.

If no PCPs satisfy all criteria, assign the No Auto PCP Found ID from the member’s line of business record to the member’s eligibility record and set an internal flag indicating that PCP Auto Assignment has been performed for this record.

3. Evaluate potential PCPs against the Preference Rules (in order), for the member’s line of business.

If one or more PCPs satisfy a Preference Rule, proceed to evaluate these PCPs against the Auto Assign Rules. Do not evaluate any further Preference Rules.

If no PCPs satisfy any Preference Rules, evaluate all potential PCPs against the Auto Assign Rules.

4. Evaluate potential PCPs against the Auto Assign Rules.

If only one PCP satisfies all Auto Assign Rules, assign the PCP on the record to the current member eligibility record. Also update the PCP’s PROVE enrollment count and set internal flags to reflect that PCP Auto Assignment has been performed for this member eligibility record and the date it was performed.

If more than one PCP satisfies all Auto Assign Rules, apply the Tiebreaker Rules for the member’s line of business to the PCPs on the qualifying records. If a single PCP is selected using the tiebreakers, assign the PCP to the member’s eligibility record. If a single PCP is not selected by the Tiebreaker Rules, use a random process to pick a single PCP and then make the assignment. Also update the submitted and assigned PCPs PROVE enrollment counts and set internal flags to reflect that PCP Auto Assignment has been performed for this member eligibility record and the date it was performed.

If no PCPs satisfy all Auto Assign Rules, assign the No Auto PCP Found ID from the member’s line of business record to the member’s eligibility record. Also set internal flags to reflect that PCP Auto Assignment has been performed for this member eligibility record and the date it was performed
