CHECK Stop/Void Menu Command Field Definitions

Field Name

Req

Description

Source

Comments

Action

R

The status change made when CVSUP Action, Post, is selected.

After selection of an action, prior to running CVSUP, an interim status of "In Process" is shown for the check number affected.

Selected from predefined list. Depending on the initial status of the check, different options available.

Valid values are:

  • Reissue

Initial status must be "Stop Confirmed" or "Voided." Changes status to "Reissued" and removes claim detail payment data and accounts payable transactions associated with the check. The status, reason and date associated with the check prior to reissue are written to PSCLM, INCLM, or DNCLM notes.

  • Reverse

Initial status must be "Stop Confirmed or "Voided." Changes status to "Reversed" and creates offsetting claim detail payment data and accounts payable transactions.

  • Stop Requested

Available for checks whose status is "Posted." Changes check status to "Stop Requested."

  • Stop Confirmed

Initial status must be "Stop Requested." Changes check status to "Stop Confirmed."

  • Stop/Reissue

Initial status must be "Stop Requested." Combines "Stop Confirmed" and "Reissue." Removes claim detail payment data and accounts payable transactions. The status, reason and date associated with the check prior to stop and reissue are written to PSCLM, INCLM, or DNCLM notes.

  • Stop/Reverse

Initial status must be "Stop Requested." Combines "Stop Confirmed" and "Reverse." Creates offsetting claim detail payment data and accounts payable transactions.

  • Void

Available for checks whose status is "Posted." Changes check status to "Void."

  • Void/Reissue

Initial status must be "Posted." Combines "Void" and "Reissue." Removes claim detail payment data and accounts payable transactions. The status, reason, and date associated with the check prior to void and reissue are written to PSCLM, INCLM, or DNCLM notes.

  • Void/Reverse

Initial status must be "Posted." Combines "Void" and "Reverse." Creates offsetting claim detail payment data and accounts payable transactions.

Date

R

Effective date of the status change.

System generated, may be manually overridden.

 

From (Check)

O

Beginning check number for the range requiring status update.

System generated, may be manually overridden.

 

Reason

R

The source or reason for the status change.

Selected from predefined list

Reason codes are maintained in keyword REASN, with a type CK.

To (Check)

O

Ending check number for the range requiring status update.

System generated, may be manually overridden.
