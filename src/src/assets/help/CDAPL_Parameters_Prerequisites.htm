CDAPL Parameters & Prerequisites

A claim must match an authorization at the header level in order for authorization/claim detail linking and pricing to be performed.

The following is only a brief listing and description of the parameters that affect this keyword. See PARAM Record Map for links to more information about each parameter. How DIAMOND Thinks documents may also give further details on how to configure these parameters for your particular needs.

Parameter

Description

Comments

CDAPLWCPSYSG

Specifies the system generated procedure code used when all the following apply:

  • The claim line is Whole Claim Priced (WCP)

  • An additional adjustment is required to balance the claim total payable

  • A Claim Detail Authorization Procedure Link (CDAPL) rule is used with a Pay Action of Price Using the Auth Price
