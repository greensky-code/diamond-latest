Matrix Determinant (MATRX) defines premium billing rate criteria for groups using the S Subscriber or W Whole Family (Member-by-Member) premium billing tiering methods (GRUPB > Tiering Mthd). The matrix determinant defines the criteria DIAMOND uses to assign a rate to a subscriber (if using the S-Subscriber tiering method) or to each member in a family up to the maximum number of dependents specified (if using the W-Whole Family tiering method).

You enter rate criteria for each matrix determinant and description in X (column) and Y (row) matrix determinant cells. The X (column) determinants can categorize members based on age bands, gender, salary bands, and Medicare status. The Y (row) determinants can categorize members based on family size, age bands, gender, and whether or not the subscriber has a spouse or domestic partner.

DIAMOND uses the data on MATRX to build the Matrix Premium Rate Entry (PREMM) where you enter the actual rates for each combination of member or subscriber unit characteristics.

 

During premium billing calculations, DIAMOND takes a snapshot of subscriber information (for subscriber billing) or subscriber and dependent information (for whole family billing) including age, gender, family size, salary, medicare status, and whether or not the subscriber has a spouse or domestic partner. DIAMOND then compares this snapshot of information against each X determinant (line by line) until it finds one where all determinants match. Next, DIAMOND compares the snapshot of information against the Y determinants (line by line) until it finds one where all determinants match. If a determinant is left blank (or .00 in a numeric field) it is not compared to the snapshot of information. The exception to this is the spouse flag. If a family has a spouse or domestic partner (defined in the relationship field of MELIG as S for Spouse or E for Domestic Partner) and the family matches all of the other criteria defined in a Y determinant, then DIAMOND checks that the Spouse Flag of the Y determinant is selected. Also, if you run billing for a family that includes a spouse and your matrix does not include any Y determinants, you will receive an error. In this situation, you must include at least one Y determinant row that has the spouse flag selected.

DIAMOND must find both an X and a Y determinant that matches the snapshot. Otherwise, it will display an error.

On the Group Billing (GRUPB) window you select the S-Subscriber tiering method and this PREMM record as the matrix definition. During premium billing calculation, DIAMOND compares the column and row determinants entered on the MATRX window against each eligible subscriber record and assigns the premium rate where the applicable column and row determinants intersect.

Thus in this example DIAMOND assigns the following premium rates for the following eligible subscribers of either gender, any age, any salary, and who:

  • are single, family size = 1 - $11

  • have a spouse or domestic partner, family size = 2 - $21

  • have one dependent, family size = 2 - $31

  • have a spouse or domestic partner and dependent(s), family size = 3-99 - $41

  • have two or more dependents, family size = 3-99 - $51
