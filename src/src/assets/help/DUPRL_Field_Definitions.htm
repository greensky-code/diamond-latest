DUPRL Field Definitions

Field Name

Req

Description

Source

Comments

Action

O

Automatic action taken by DIAMOND on a subsequent claim matches the criteria rules when compared to an earlier claim.

Selected from a predefined list

Hard coded values are:

Warn Operator receives a warning that the claim may be a duplicate of an earlier claim.

Hold Operator receives a warning message and a hold reason code is assigned to the claim.

Deny Operator receives a warning message and a denied reason code is assigned to the claim.

Blank No action is taken.

Billed

O

Tells DIAMOND whether to consider duplicates according to the amount billed on the detail line.

Selected by check box.

If selected, the total amount billed is included as a potential duplicate characteristic.

Claim Type

R

General category of claim to be subjected to the Duplicate Rule.

Selected from a predefined list

Valid selections are

P Professional

I Institutional

D Dental

Days After DOS

C

Number of days after the primary date of service that defines the range of dates within which a second claim will be evaluated as a potential duplicate.

Manually entered.

Optional if Primary DOS is selected; not available for entry if Primary DOS is cleared.

Days After DOS

C

Number of days after the detail date of service that defines the range of dates within which a second claim will be evaluated as a potential duplicate.

Manually entered.

Optional if Detail DOS is selected; not available for entry if Detail DOS is cleared.

Days Before DOS

C

Number of days prior to the primary date of service that defines the range of dates within which a second claim will be evaluated as a potential duplicate.

Manually entered.

Optional if Primary DOS is selected; not available for entry if Primary DOS is cleared.

Days Before DOS

C

Number of days prior to the detail date of service that defines the range of dates within which a second claim will be evaluated as a potential duplicate.

Manually entered.

Optional if Detail DOS is selected; not available for entry if Detail DOS is cleared.

Desc

R

Description of the duplicate rule.

Manually entered.

 

Detail DOS

O

Indicates whether the date of service on the detail line is considered in checking for a duplicate claim.

Selected by check box.

If checked, the detail DOS is one of the characteristics of the claim that may indicate a duplicate.

Dup Rule

R

Unique identifier of the duplicate rule.

Manually entered.

The Duplicate Rule ID is the link to the LINBS record.

Med Def

O

Tells DIAMOND whether to consider duplicates according to the Medical Definition on the detail line.

Selected by check box.

If selected, the Medical Definition is included as a potential duplicate characteristic.

Proc Modifier

O

Tells DIAMOND whether to consider duplicates according to the procedure code + modifier on the detail line.

Selected by check box.

This check box is available only if Procedure Code is selected. If selected, the procedure code modifier is included as a potential duplicate characteristic.

Place of Service

O

Tells DIAMOND whether to consider duplicates according to the place of service.

Selected by check box.

If selected, the place of service is included as a potential duplicate characteristic.

Primary DOS

O

Tells DIAMOND whether to consider duplicates according to the primary date or service.

Selected by check box.

If selected, the primary date of service is included as a potential duplicate characteristic.

Procedure Code

O

Tells DIAMOND whether to consider duplicates according to the procedure on the detail line.

Selected by check box.

If selected, the procedure code is included as a potential duplicate characteristic.

Provider

O

Tells DIAMOND whether to consider duplicates according to the rendering or institutional provider.

Selected by check box.

If selected, the rendering provider or institution is included as a potential duplicate characteristic.

Reason Code

C

Reason code which is assigned by DIAMOND depending on the action entered above

Selected from a predefined list

Selectable reason codes are consistent with the Action selected. If Hold is the action, only reason codes with a type HD are shown; if Warning, only type WN are shown; if Deny, only type DN are shown.

Reason codes are created and maintained in REASN

Same Tooth

O

Indicates if DIAMOND should search all dental-specific fields for a match.

Selected by check box.

If selected, DIAMOND checks the following fields for a match:

T#: Same tooth number

Surf: Surface

A: Arch

Qdr: Quadrant

Oral Cav: Oral Cavity

Total Billed

O

Tells DIAMOND whether to consider duplicates according to the total amount billed.

Selected by check box.

If selected, the total amount billed is included as a potential duplicate characteristic.
