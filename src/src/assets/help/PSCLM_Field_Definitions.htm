Audit Status

O

The results of the claim audit (GMIS) filter. DIAMOND determines whether the claim is excluded from the claim audit.

System generated

Values are:

  • Y - Claim detail line has passed the Claim Audit Filter.

  • N - Claim detail line has failed the Claim Audit Filter or Claim Audit Filter exclusions have not been setup for the line of business associated with the claim.

  • Blank - Claim detail line was not evaluated against the Claim Audit Filter because parameter CLAIMAUDIT is not Y.

  • C - Completed ClaimCheck.

  • A - ClaimCheck Added Line.

  • E - Expanded line processed by ClaimCheck.

  • S - Single line claim processed by ClaimCheck.

For more information, see keyword CAUDT (Claim Audit Filter).

Audit Status

O

The results of the claim audit (GMIS) filter. DIAMOND determines whether the claim is excluded from the claim audit.

System generated

Values are:

  • Y - Claim header has passed the Claim Audit Filter.

  • N - Claim header has failed the Claim Audit Filter or Claim Audit Filter exclusions have not been setup for the line of business associated with the claim.

  • Blank - Claim header was not evaluated against the Claim Audit Filter because parameter CLAIMAUDIT is not Y.

  • C - Completed ClaimCheck.

For more information, see keyword CAUDT (Claim Audit Filter).

Auth No

O

The authorization number linked to this claim.

Manually entered or system generated

You can type an authorization number or after you enter the member ID, the Lookup dialog box will display a list of authorizations for the member you can select.

If the members line of business is set up for automatic authorization/claim matching, then one of the following occurs when you first display the detail window:

  • If there is one matched authorization for the member, then the authorization number automatically displays.

- or -

  • If there are multiple matched authorizations for the member, then the Select Matched Authorizations dialog box displays. It displays a list of authorizations for the member that you can select.

A line of business is set up for automatic authorization claim matching when the Auth/Claim Match check box is selected in keyword LINBS (Line of Business).

COB Pat Liab

D

The reduction in patient liability resulting from COB, if any.

System generated

DIAMOND may truncate the display of the currency amount.

Calculated as the Total Patient Liabilities amount before COB minus the Total Patient Liabilities amount after COB.

Only calculated if all the following apply:

  • COB Calculation method Non-Duplication of Benefits or Integration of Benefits was applied during automatic adjudication

  • Total Pt Liabilities after COB is less than Total Pt Liabilities before COB

  • BENEF detail stipulates that benefit accumulators are not adjusted for other carrier paid amounts

Date

R

First date of service pertaining to this detail line.

Manually entered

During adjudication, DIAMOND validates that this (from) Date of service falls within the date range of one of the members eligibility periods. If it does not, DIAMOND denies the line, sets the Claim Status list to D Denied, sets the Allowed Amt box and Total Not Cov box to 0.00, and assigns the Reason code associated with Reason code type ME as the Not Cov Rsn code. Reason code ME (only one is allowed) must exist in keyword REASN.

DIAMOND ID

O

Unique Identifying Number

There is one identifier for every instance of a person in the Member Master records

Manually entered

For more information, see Dual Coverage, How DIAMOND Thinks.

Dx 1-4

O

Members diagnosis codes associated with this claim.

Manually entered

You can type up to 30 alphanumeric characters. Unless the insertion point is in this box, then only 7 characters display.

The primary diagnosis code must go in Dx 1. Entered codes are validated against the members gender/age and a warning is issued if the code is flagged for case management. DIAMOND displays an error message if codes do not pass validation.

Diagnosis codes are maintained in keyword DIAGN (Diagnosis Code).

Dx Ptr 1-4

O

Indicates which Dx box on the claim header window contains the diagnosis code that most pertains to this claim line.

Manually entered

Valid values of 1, 2, 3, and 4 can be typed in these four boxes. For example, type 1 to point to the Dx 1 box on the Claim header window.

During claims adjudication, DIAMOND uses the pointer in the first Dx Ptr box.

If only one diagnosis code was entered in the Dx 1 box on the claim header window, then 1 displays in the first Dx Ptr box.
