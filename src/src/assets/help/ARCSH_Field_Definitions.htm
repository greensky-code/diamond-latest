ARCSH Field Definitions

Field Name

Req

Description

Source

Comments

Amount

R

Dollar amount of the transaction, typically the check amount.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Batch ID

R

Unique identifier of a cash batch.

System generated for new batches, or selected from a predefined list for existing batches

F5 lookup is available.

Batch Status

O

Display of batch status.

System generated

Valid values are:

Open: A new batch was entered and saved.

Closed: A batch with either a previous status of Open or of Error (subsequently corrected) was closed using menu command Batch Close.

Error: A closed batch was posted using menu command Cash Post, but error(s) within the batch resulted in no receipts being posted. Errors must be corrected; batch must be closed again, and posted again.

Posted: A batch that was closed was posted using menu command Cash Post.

Batch Total

R

Total dollar amount of all items in the batch.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Company Code

R

General ledger company to which the cash receipt will be posted.

Selected from predefined list

General ledger companies are maintained in keyword COMPF.

Customer ID

R

Unique identifier of the customer.

Selected from predefined list

Customer IDs are created by the execution of GPBIL, or in some circumstances, manually in keyword ARCUS.

Customer Type

R

Category of customer.

Selected from predefined list

Valid values are:

Annual Billing

Combined

Group

Individual

Description

O

Text description of the transaction.

Manually entered

 

GL Ref Code

R

Cross reference to the debit and credit accounts to which the cash receipt will be posted.

Selected from predefined list

General ledger reference codes are maintained in keyword GLREF.

Invoice Number

D

Automatically assigned control number associated with each billing period and customer.

System generated

 

Item Count

R

Number of items in the batch.

Manually entered

 

Post Date

D

Date the cash batch was posted.

System generated as a result of the Cash Post menu command

 

Received Date

R

Date the transaction, typically the check, was received; default is the system date.

Manually entered

 

Statement Status

D

Cash receipts may be printed on customer invoices. This field displays the availability of this transaction for printing.

System generated

Valid values are:

Not reflected on bill: Although available to print on an invoice, it has not yet been printed.

Reflected on bill—Unposted: The transaction is available to print because GPBIL Calculate has run, but that run has not yet been posted.

Reflected on bill—Posted:

The transaction is available to print because both GPBIL Calculate and GPBIL Post have been run.

Transaction Date

R

Date of the transaction, typically the date on the check.

Manually entered

 

Transaction Number

R

Identifier of the transaction, typically the check number.

Manually entered

You can type up to 35 alphanumeric characters.

Transaction Type

R

Category of transaction.

Selected from predefined list

Standard values are

Cash

Card

EFT

Check

Transaction types can be updated in CODES, TRANSTYPE.
