APTAX Field Definitions

Field Name

Req

Description

Source

Comments

Action

R

Next action steps to perform in current 1099 reporting process.

Selected from predefined list

Valid values depend on the job status.

Job status and corresponding actions are:

  • New: Calc, Cancel

  • Successful: Calc, Print, Cancel

  • Posted: Print

  • Calculate: One 1099 record will be calculated for each vendor within tax reporting entity

  • Print: Creates 1099 records for viewing, printing, and electronic IRS reporting

  • Post: Finishes processing of the job; once posted, it cannot be rerun

  • Cancel: Stops the run and removes the job ID from list of existing jobs

Comments

O

Description of the job.

Manually entered

 

Include Offsets

O

Indicates whether or not claims and admin fees that were offset toward advance payments should be included in 1099 reporting.

Manually entered

  • Checked: include claims and admin fees that were offset toward advance payments in 1099 reporting.

  • Blank: do not include claims and admin fees that were offset toward advance payments in 1099 reporting.

Dollar Limit

R

Minimum dollar amount of total vendor payments included in the 1099 run.

Manually entered

You can type up to 16 characters before the decimal and 2 characters after the decimal.

Job ID

R

Unique identifier of a 1099 job.

System generated or manually entered.

 

Report Type

R

Medium of the 1099 report.

Selected from predefined list

Default is:

View Data: Displays all vendor 1099s online. A hard copy of the vendor 1099 can be printed from this display.

Request Date

D

System date.

System generated

 

Request Type

R

Indicates when job will be run.

Selected from predefined list

Values are:

  • Immediate

  • Deferred (future use)

Request User

D

Logon ID of the user requesting the job.

System generated

 

Sort Options

R

Order in which the report is sorted.

Selected from predefined list

Values are:

  • Vendor Name: Sorted by vendor name.

  • Zip Code/Tax Entity Name: Sorted by vendor name within ZIP code.

  • IRS Tax ID/Tax Entity Name: Sorted by tax entity within IRS tax ID.

  • Tax Entity Name: Sorted by tax entity name.

Status

D

Job processing status.

System generated

Values are

  • New: Job ID is saved but records have not been calculated

  • Successful: Records have been calculated

  • Posted: Calculated records have been posted

Tax Rep Entity

R

Valid tax reporting entity.

Selected from predefined list

Tax reporting entities are maintained in keyword TRENT.

Tax Year

 

R

Check date year of the transactions included in the 1099 job.

Manually entered
