This topic provides processing logic information about:

About price rules

A price rule comprises of records added in keywords PRULE (Price Rule), PRULD (Price Rule Detail), and PRULS (Price Rule Detail Selection). These sets of records together can do any or all of the following:

  • Specify different pricing methods and the circumstances under which they are used

  • Calculate allowed amounts and assign reason codes

  • Hold a claim line and assign a hold reason code

  • Send a message to the claims processor

  • Add units to base procedure units

  • Override the allowed factor

  • Prompt the claims processor to enter an allowed factor

  • Apply provider-specific "% of allowed"

About price rule detail and selection

Add a price rule detail record in keyword PRULD to enter details for a high-level Price rule added in keyword PRULE. Price rule details can be set up specific to procedure code modifiers.

You can enter one modifier code for pricing institutional claims in keyword INCLM (Institutional Claims).

You can enter four modifier codes for pricing professional services claims in keyword PSCLM (Professional Services Claims). However, only the modifier codes entered in the first or second Mods box are used for pricing or adjudication.

Price rule detail records do the following:

  • Define pricing differences within a single price rule based on either procedure modifier code or a specific calculation method

  • Define processing rules, including a reason code, default, and override values for each calculation method within a price rule detail record

Add price rule detail selection records in Keyword PRULS to specify what price rule detail records can be used for particular procedure codes or ranges of procedure codes. This flexibility allows different calculation methods and processing rules to be applied depending on the medical service.

Keyword PRULD also includes the Add to Base Units field. This field is used when the Calculation Method list is set to either Lesser Unit Value or Pct of Billed or Unit Value. The value in the Add to Base Units box is added to the number of units DIAMOND retrieves from the Unit Value record in keyword UNITV (Unit Value). In keyword PRULD, The unit value is selected in the Unit Value Type list, and tells DIAMOND which UNITV field to retrieve: total, anesthesia, professional, or technical. The units retrieved are referred to as "base units."

How DIAMOND finds price rule details

Keyword PRULD contains the inputs to the pricing calculation. Different pricing calculations apply to different modifier values, and therefore DIAMOND first looks at the modifier code in the first Mods box field to identify whether the price rule detail record applies to the claim line item. (Of the four modifier codes in keyword PSCLM, DIAMOND only uses the modifier code entered in the first or second Mods box in pricing and adjudication. Institutional claims include only a single Modifier box.) Typically, for example, different calculations are required to price a surgery claim than those required to price the anesthesia for that surgery. The modifier code of the surgical procedure code that indicates anesthesia tells DIAMOND to look for the calculation method on a price rule detail record with the anesthesia modifier code. For more information on pricing anesthesia time, see Pricing Anesthesia Time, How DIAMOND Thinks.

Beyond differences due to modifier codes, you may have pricing schemes that require different calculation methods for different claim situations. These different calculations are added in keyword PRULD, and DIAMOND must find the correct method for each detail claim line depending on the claim situation. DIAMOND uses a fall-through logic to search for the correct (matching) price rule detail record, reading the records and stopping as soon as it finds a match. Because DIAMOND searches the methods in the order you specify, you can define the order to ensure DIAMOND will find the correct rule, stop searching, and select it.

To define this search order for price rule detail records, you associate a search level and at least one sequence number to each record. (Note, search levels are sometimes referred to as "calculation methods.") You may have up to five search levels, and hundreds of sequences within each level. Because DIAMOND will search until it finds a matching rule, stop searching and apply that rule, you must make sure that specific rules fall in order before more general rules.

Multiple sequence numbers are used when different pricing characteristics apply to different procedure codes for the same calculation method (level). For example, calculation method sequence 1 can refer to general surgery, and sequence 2 can refer to organ transplants. To distinguish these different procedures a PRULS record is used to define the procedure code range or ranges to which the price rule detail record applies.

DIAMOND searches within each level in numeric sequence number order, and then moves to the next highest level, e.g. level 1/sequence 1, level 1/ sequence 2, level 2/sequence 1, level 2/sequence 2, and so forth. Therefore, use level 1/sequence 1 to define the method which applies to the most specific circumstance. Use levels 2 - 5 and their associated sequences to define increasingly more generally applicable methods.

Make sure that you have a general method to catch those claim line situations that do not match specific rules, otherwise an error message displays when no pricing method is found.

Some health plans establish a level 1/sequence 1 record with a calculation method of "None" to apply to procedures that are not covered.
