CHECK Field Definitions

Field Name

Req

Description

Source

Comments

Amount

D

The amount of the transaction.

Displayed

 

Bank Account

R

ID of the bank account on which the transaction was drawn; entered as a selection criteria to limit the display of transactions to be updated or viewed.

Selected from predefined list

Bank accounts are maintained in keyword BANKA.

Check Status

D

The status or nature of the check or EFT transaction.

Displayed

Values are:

  • Alignment: Check was used during print setup in CKPRT to align the printer.

  • Cleared: Transaction was printed (CKPRT), posted (CKPST) and cleared by the bank.

  • In Process: Transaction has been updated using CHECK menu command Stop/Void but not yet updated using CVSUP.

  • Posted: Transaction was printed (CKPRT) and posted (CKPST).

  • Reissued: After printing and posting, check was stopped or voided and reissued.

  • Reversed: After printing and posting, check was stopped or voided and reversed.

  • Stop Requested: After printing and posting, stop was requested.

  • Stop Confirmed: A requested stop was subsequently confirmed.

  • Voided: After printing and posting, transaction was voided.

Check/EFT Date

D

The transaction date of the selected check or EFT transaction.

Displayed

 

Check/EFT No. From and Through

O

Specify the beginning and ending check or EFT numbers to display.

Selected from predefined list

Tip:

To display a single check or EFT transaction, type the number in both the from and through fields.

If you leave the through field blank, the system will display all transactions starting with the number you entered in the from field.

Check/EFT Type

O

A limit on the display of transactions to a category.

Selected from predefined list

Valid values are:

  • Check

  • EFT

  • Blank (shows both checks and EFTs)

Comments

O

Text comments explaining the transaction.

Manually entered

 

Cross Reference

O

The number of a transaction associated with the transaction being displayed or updated.

Manually entered

Used primarily to associate a reissued transaction with the original transaction.

Job ID

D

The ID of the CKPRT job associated with the displayed transaction.

Displayed

 

Last Action

D

The most recent action taken on the transaction.

Displayed

 

Last Action Dt

D

Date of Last Action.

Displayed

 

Payment Date

O

A limit on the display to a specific payment date or a range beginning with that date.

Manually entered

 

Reason

D

The reason code for the Check Status shown.

Displayed

Reason codes are maintained in keyword REASN.

Status

O

A limit on the display to a specific status.

Selected from predefined list

Valid values are:

  • Cleared

  • Posted

  • Reissued

  • Reversed

  • Stop Confirmed

  • Stop Requested

  • Voided

Status Dt

D

Effective date of the Check Status shown.

Displayed

 

Vendor

O

A limit on the display to a specific vendor.

Selected from predefined list

The vendor address associated with the check or EFT transaction.
