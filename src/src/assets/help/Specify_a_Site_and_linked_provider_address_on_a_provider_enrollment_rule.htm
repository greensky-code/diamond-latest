Specify a Site and linked provider address on a provider enrollment rule

You can only specify a site and linked provider address on a provider enrollment rule when adding a new rule. You cannot change a provider enrollment rule once you save it.

If you want to add a site and linked provider address to an existing Provider ID/Effective Date/Term Date/LOB/Panel/IPA/Address combination, you must first terminate the existing record and then add a new record.

The Site field is only available when parameter MEMSITEVAL Parameter 1 = Y .

To specify a Site and linked provider address on a provider enrollment rule

  1. Press F3, type PROVE, and then press ENTER.

  2. In the Provider ID field, type a Provider ID. F5 lookup is available.

  3. Press CTRL+N to add a provider enrollment rule.

  4. In the Effective Date field, type the effective date of this rule.

  5. In the LOB field, type the line of business for this rule. F5 lookup is available. When the line of business has the Site check box selected, the Site field below is required. Else, Site is optional.

  6. In the Panel field, type the provider Panel for which this rule applies. F5 lookup is available.

  7. In the IPA field, type the provider IPA for which this rule applies. F5 lookup is available.

  8. In the Site field, type a Site code. F5 lookup is available. If you type a Site code, it must satisfy Site Validation criteria. The F5 lookup only displays Site codes that satisfy Site Validation criteria. On the Lookup window, you can also change the way a blank IPA or Panel is treated by selecting the Match All Contracts check box For more information, see Site Validation, How DIAMOND Thinks section Entering a Site code on a provider enrollment rule.   9. When you type or select a Site, DIAMOND logic displays the corresponding Panel, IPA, site name, Cert check box value, and in the Address field, the linked provider address.

  10. Fill in the remaining fields on the keyword PROVE window. For more information, see PROVE Field Definitions.
