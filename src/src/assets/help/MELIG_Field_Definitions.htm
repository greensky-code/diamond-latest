MELIG Field Definitions

Field Name  Req Description Source Comments

Ben Start Date

O

Entered on the subscriber’s record only, but can pertain to accumulations of benefits for the family.

Manually entered

This field, in combination with entries in keyword BRULE (Benefit Rule), overrides DIAMOND’s use of the group’s contract effective date (in GRUPC). See Benefit Rules and Packages user documentation for additional detail.

CS Ind

O

Customer Service indicator noting member has confidential or sensitive information.

Selected from predefined list

Valid Values:

CO - Confidential

VP - V.I.P

Date of Death

O

Member's date of death

Manually entered

Unavailable if Termination Date is blank. Else, optional.

If a Termination Date is entered, when you enter a date of death DIAMOND logic validates it according to the LINBS Date of Death Rule.

Dept.

O

Code representing the department where a member works.

Selected from predefined list

Department codes are maintained in keyword CODES (System Codes Maintenance) with a type MELIGDEP.

DIAMOND ID

O

Unique Identifying Number

There is one identifier for every instance of a person in the Member Master records

Manually entered

Use of a DIAMOND ID allows tracking of more than one instance of coverage for a person in DIAMOND.

If this person has more than one insurance coverage, the other instances of the person appear in a popup and the appropriate one can be selected from the list.

For more information, see Dual Coverage, How DIAMOND Thinks.

Eff Date

R

First date the member is eligible for the coverage described in the eligibility section.

Manually entered

DIAMOND does not allow overlapping eligibility periods.

Additional edits are performed against keyword MEMMC Medicare Part A and Part B effective dates if enabled in system parameters MEMMEDAEFF AND MEMMEDBEFF.

Elig Sts

R

Indicates whether the member is eligible during the period described in this eligibility record.

System generated or manually entered

If a new member record, DIAMOND defaults to the first character of the ELIGSTAT record in keyword PARAM (Parameter).

If the entry in this field matches an entry in ELIGSTAT parameter, member is eligible. If the entry is not found in ELIGSTAT, the member is ineligible.

This field can be manually entered/overridden for existing members.

Group ID

R

Member’s group.

Manually entered

Selected from keyword GROUP.

Hire Date

O

Member’s date of hire.

Manually entered

 

IPA

O

Member’s Independent Practice Association.

Manually entered

Selected from keyword IPAVA.

LOB

O

Line of business associated with the group/plan in which the member is enrolled.

System generated

 

Location

O

Code representing the location of the member.

Selected from predefined list

Location codes are maintained in keyword CODES with a system code type MELIGLOC.

MCareSt

O

Member’s Medicare status.

Manually entered

Valid values include codes for the following:

  • State Code

  • County Code

  • Medicaid

  • Disability

  • Hospice

  • Institution

  • ESRD

  • Working aged

OtherSt

O

Second status code.

Manually entered

This field is not edited.

Panel ID

O

Code associated with a panel of providers which are associated with the group in which the member is enrolled..

Manually entered

 

PCP ID

C

ID of the member’s primary care provider.

Manually entered

Required if required by the line of business otherwise, optional.

PCP Chng Rsn

C

The reason code for this PCP change.

Manually entered

 

PEC End Date

O

The last day of the pre-existing condition restriction.

Manually entered

It is only necessary to enter a date in this field when the member’s pre-existing condition end date is different than the group end date.

PEC Waived

O

Indicator that the pre-existing condition restriction is waived.

Selected from check box

When PEC Waived is selected, DIAMOND skips all pre-existing condition (PEC) logic, including all member and group PEC logic.

Plan Code

R

Code representing a benefit plan available for this group and member.

Manually entered

Selected from keyword GRUPD.

Privacy On

Y

Designates whether the member prohibits release of personal health information to outside entities.

Selected from check box

If checked, member’s information may not be released to outside organizations.

Prov 2

O

ID of a secondary provider associated with the member, such as an OB/GYN for a female adult member.

Manually entered

 

Rate Type

O

An optional code to indicate a specific rate type to be used during premium billing rate calculation.

Selected from a predefined list after user defined values are set up using CODES.

Rate Type codes are maintained in CODES. Examples of user defined rate types might be:

Rate0

Rate1

Rate2

Rate3

See the following online documentation for more information:

Premium Billing>How DIAMOND Thinks>Member-by-Member (Whole Family) Family Billing

Premium Billing>Rate Matrix Examples>Member-by-Member Family Rates by Member Age/Rate Type

Reason

C

Hold reason code to be applied to the claim detail line when the PEC end date is different than the group end date

 

Required if there is a date entered in the PEC End Date field.

Reason codes are maintained in keyword REASN (Reason).

Rel Code

R

Indicates member’s relationship to the subscriber.

Manually entered

Valid values are

A Aunt/Uncle

B Brother/Sister

C Child

D Stepchild

E Domestic Partner

F Friend

G Grandchild

H Head of household

I Brother/Sister-in-law

J Foster Child

K Parent-in-law

M Step-parent

N Grandparent

O Other

P Parent

R Boy/Girlfriend

S Spouse

T Cousin

U Guardian

X Niece/Nephew

Riders

O

Codes which represent add-on benefits available for this group, for example, dental, pharmacy, vision.

Manually entered

 

Salary

O

Member’s salary.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Some groups use salary as a parameter for premium billing.

Site

C

Unique identifier of a site (facility).

Selected from predefined list

Unavailable if parameter MEMSITEVAL Parameter 1 = No or if the corresponding line of business record Site check box is clear.

Required if the corresponding line of business Site check box is selected.

F5 lookup is available. For more information, see Using Site Validation and Site Validation, How DIAMOND Thinks.

Subscriber ID

R

Unique identifier of a subscriber unit.

Manually entered

 

Term Date

O

Last date on which the member is eligible for coverage

Manually entered

DIAMOND does not allow overlapping eligibility periods.

Additional edits are performed if enabled in keyword LINBS Date of Death Rule.

Term Reason

C

Code explaining the reason for termination.

Manually entered

Required if there is a date entered in the Term Date field.

Reason codes are maintained in keyword REASN.

UserDef1-3

O

Implementation specific text fields.

Manually entered

The literals and validation for these fields are defined during implementation using keyword UDTXT (User Defined Fields).
