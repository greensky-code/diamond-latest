Overview

Use the Change Header special function to change any of these claim header data elements:

  • Service Date

  • Thru Date

  • Member ID/Person Number

  • Provider ID

  • Provider Address

  • Provider Zip

  • Vendor ID

  • Vendor Address

When you use the Change Header screen, the system reinvokes all claims processing for member eligibility, provider/vendor relationship validation, contract search, pricing, and adjudication. You can access this screen only from the claim header and only if the claim is not finalized. You cannot use Change Header if the claim header Audit Status is C, Complete.

Changes you make in the Change Header screen are tracked by Audit Trail.

Changes to Provider and Vendor Information

You can change the provider or vendor fields or delete the value so a field is blank. However, you cannot make both provider and vendor fields blank, you must specify at least one.  

 

If you change the provider or vendor information, the system performs contract match validation logic:

  • If the system cannot locate a matching contract, it displays a warning message: "No Contract found for Provider and Vendor Indicated. Press Yes to continue to apply Claims Pricing Logic or press Cancel to return to Change Header screen to make additional changes."

  • If you click Yes to apply Claims Pricing logic, the system checks to determine if Covering Provider logic applies. If this logic is applied, the claim is priced.

  • If Covering provider logic is not applied, the system generates an error message: "Unable to Find Provider Contract. Press Yes to continue, claim detail lines will be denied. Or Cancel to return to the Change Header screen to make additional changes.

If you blank out the provider, the system uses Vendor ID/Vendor Address to search for a contract.

  • If the system locates a single match, it populates the provider information and re-prices and re-adjudicates the associated claim detail lines.

  • If the system locates multiple matching contracts, it generates a provider pick list from which you can make a selection.

  • If the system doesn't locate a matching contract, it displays an error message "Unable to determine provider from vendor information, need to specify provider." and you are automatically returned to the Change Header screen.

If you blank out the vendor, the system uses Provider ID/Provider Address to search for a contract.

  • If the system locates a single match, it populates the provider information and re-prices and re-adjudicates the associated claim detail lines.

  • If the system locates multiple matching contracts, it generates a vendor pick list from which you can make a selection.

  • If the system doesn't locate a matching contract, it displays an error message "Vendor ID is Required" and you are automatically returned to the Change Header screen.

Affects on Pay to Subscriber or Pay to Dependent Flag

When you use the Change Header screen to change claim information, the Pay to Subscriber and Pay to Dependent flags are automatically reset to No. Once you press OK after changing data on the Change Header screen, the system checks these flags and if set to yes, resets them to No. If either flag was set to yes prior to using the Change Header function, the system provides a message: "Pay to Subscriber Flag Reset from Y to N." or "Pay to Dependent Flag reset from Y to N." You must manually reset the flag.

Top

Affects on EOB Indicator and Other Carrier Information

If you change the Subscriber ID or Person Number, the system resets the EOB Indicator and Other Carrier Code to blank/None. Also, when you re-price and re-adjudicate the claim detail line, the system sets the OC Allowed and OC Paid Amt fields to zero.

Top

Affects on the Print Flag

If the print flag was manually changed and the status is M, the print flag will not be recalculated.

Top

VENDORFIRST Parameter

The VENDORFIRST parameter does not work with the Change Header screen. This means that your cursor will not automatically move to the vendor or provider field on the Change Header screen based on the VENDORFIRST parameter setting. For specific information about how provider contract search works with either provider or vendor information, see the Changes to Provider and Vendor Information section of this topic.
