Setting Up the Default Provider

Restrictions

Instructions

Restrictions

None.

Instructions

Step

Comment

1. In keyword PROVF, create the default provider (entry values represent examples).

  1. Provider ID = DEFAULTP (clearly identifies the default provider)

  2. Last Nm = Default

  3. First Nm = Provider

  4. Provider Type = DEFALT

  5. Default Vendor ID = DEFAULTV

  6. Default Vendor Addr = Address for Default Vendor

  7. Specialty Type = DEFLT

  8. Primary = Yes

  9. Accepts Patients = Yes

You are creating the default provider master record.

Provider types are created using the PTYPE window.

You are linking the default vendor ID you previously created to this default provider.

Specialty types are created using the PTYPE window.

2. Create the default provider contract record using the PROVC window.

  1. Provider ID = DEFAULTP (the default provider)

  2. Def Contr Type = P-Payable

  3. Enter an Eff(ective) Date for the default contract record.

  4. Par Flag = N-NonPar

  5. Vendor = DEFAULTV (the default vendor)

  6. Vendor Address = Address for the default vendor

The default provider contract can be "global" (no line of business, panel, or IPA) or you can create multiple contract records with different pricing rules depending on your business needs.

This example is a global contract record; that is, all out-of-area, non-participating, or one-time providers will be paid under this contract.

You will override the default vendor address when you pay a claim from an out-of-area, non-participating, or one-time provider.

3. Navigate to the PROVC>Special menu>Contract Pricing Details window and enter the default contract’s pricing data.

Default pricing is usually set up as 100% of either the billed or the allowed amount depending on your business needs.

4. Open a new PROVF window.

  1. Use the F5 lookup function to look up the DEFAULTP (default) provider.

  2. Write down the default provider’s Seq Prov ID displayed in the lookup window.

You will need the Seq Prov ID when you turn on Default Provider Logic.
