About COBOR

Use COBOR to build COB Order rules. COB Order rules can be used to automatically determine the other carrier coverage order - primary, secondary, tertiary or other - on a Member COB History (COBHS) record.

To use COB Order rules to automatically assign COB Order, when you enter a COBHS record accept the COB Order field default value of O - Other and attempt a save. The COB Order rules are then automatically applied to determine a value for the COB Order field.

Note:

When you enter, modify or delete a COBOR rule and save, you must log out of DIAMOND and then log back in for the rule change to take effect.

You can also use the COBHS Special menu command Determine COB Order to reapply COB Order of liability determination logic and overwrite the COB Order field. In addition, you can manually override an automatic COB Order assignment.

Common types of rules include the Birthday Rule and the Gender Rule. For example, a COBOR Birthday Rule can be set up to accomplish the following.

If the following are true:

  • Relationship code of the member = C - Child

  • COBHS Relationship code = P - Parent

  • COBHS Dependent Determination Rule = B - Birthday

  • GRUPD Dependent Determination Rule = B - Birthday

  • Date of birth of the COBHS Subscriber is earlier than the date of birth of the DIAMOND subscriber

Then:

COB Order is P - Primary. 

See Coordination of Benefits -- How DIAMOND Thinks for more information on setting up COB Order rules.
