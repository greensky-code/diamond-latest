Field Name

Req

Description

Source

Comments

A/R Adjustment ID

R

Unique identifier of an adjustment.

System generated for new adjustments, or selected from a predefined list for existing adjustments

The Statement Status is indicated with a code:

Not reflected on bill

Reflected on bill, unposted

Reflected on bill, posted

Comments

O

Description of the transaction.

Manually entered

 

Company Code

R

General ledger company to which the adjustment will be posted.

Selected from predefined list

General ledger companies are maintained in keyword COMPF.

Customer ID

R

Unique identifier of the customer.

Selected from predefined list

 

Customer Type

R

Category of customer associated with the adjustment.

Selected from predefined list

Valid values are

Annual Billing

Group

Individual

Combined

GL Ref Code

R

Cross-reference to the debit and credit accounts to which the adjustment will be posted.

Selected from predefined list

General ledger reference codes are maintained in keyword GLREF.

Invoice Number

D

Automatically assigned control number associated with each billing period and customer.

System generated

 

Post Date

D

Date the adjustment was posted.

System generated

Generated as a result of Adjustment Post.

Reason Code

O

Code explaining the reason the adjustment is being applied.

Selected from predefined list

Accounts Receivable reason codes are maintained in keyword REASN, with a type AR.

Statement Status

D

Adjustments may be printed on customer invoices. This field displays the availability of this transaction for printing.

System generated

Values are:

Not reflected on bill: Although available to print on an invoice, it has not yet been printed.

Reflected on bill—Unposted: The adjustment is available to print because GPBIL Calculate has run, but that run has not yet been posted.

Reflected on bill—Posted:

The adjustment is available to print because GPBIL Calculate has run and GPBIL Post has been run.

Transaction Amt

O

The dollar amount of the adjustment, either positive or negative.

Manually entered

You can type up to 16 digits before the decimal and 2 digits after the decimal.

Negative amounts (credits to the customer’s account) are indicated with a minus sign (-) before the dollar amount.

Transaction Date

R

Date of the adjustment.

Manually entered, or the default, the system date, can be accepted.
