VADPY Commands on the Menu Bar

 

Command            Menu  Description

Acc Pay Vendor Display

S

Display keyword APVDS (Accounts Payable Vendor Display).

Adv Pay Account

T

Display keyword VADPY (Vendor Advance Payment Account).

Adv Pay Priority

T

Display keyword VADPP (Advance Payment Offset Priority)

Adv Pay Rules

S

Display keyword VADPR (Vendor Advance Payment Rules)

Advance Payment Approval

S

This special function is available only if system parameter VENDSECAPP is set to yes. Use this special function to approve a payment/entry before the advance payment will be considered for processing.

This process requires one individual to enter the information and a different person to approve the record.

Fields:

  • Vendor Id – Displayed from the Account being added/updated

  • Vendor Address – Displayed from the Account being added/updated

  • Adv Pay Account – Displayed from the Account being added/updated

  • Approve Status

  • Deny Reason - Required if the Adv Pay Status is Not Approved. Deny reasons are defined in REASN.  

  • By - The Approver’s ID. The ID must be different than the ID of the individual who entered the Account.

  • Date - Defaults to the system date and can be changed by the user.

  • User Defined 1 and 2 - Use of these fields is determined by your organization.

  • User Date 1 and 2 - Use of these fields is determined by your organization.

  • O.K. button - The information is saved and the user is taken back to the primary screen.

  • Cancel button - The information is not saved and the user is taken back to the primary screen.

Line of Business Selection

S

Use to include one or more lines of business in the advance payment account.

Provider Relationships

S

Display the providers associated with this vendor ID, in each of the following categories:

  • Providers who use this vendor ID for their default vendor.

  • Providers who use this vendor for payment of capitation (PROVC, Type=C, Cap Vend ID).

  • Providers who use this vendor for payment of fee-for-service claims (PROVC, Type=Payable).

Vendor Address

T

Display keyword VENDA (Vendor Address).

Vendor Credit

T

Display keyword VENCR (Vendor Credit).

Vendor Lookup

S

Opens the Vendor Lookup screen.

Vendor Master

T

Display keyword VENDR (Vendor Master).

View Credit Balance

S

Displays the Transaction ID, File Type, and Net Amount of the vendors credit balance. If the vendor no longer has a credit balance, the screen displays the Paid Net Amount.
