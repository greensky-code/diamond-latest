SUBSC Field Definitions

Field Name

Req

Description

Source

Comments

Job ID

R

Unique Subscriber Update job identifier.

System generated

This field can be manually overridden with a unique ID.

Request User

D

User ID of the person entering this job information.

System generated

 

Action

R

Action being requested.

Selected from predefined list

Hard coded values are:

Run Run this currently identified job.

Cancel The currently identified and previously run job.

Important: When you run a premium billing job using the Premium Billing (GPBIL) window, Diamond automatically runs an SUBSC job (prior to the GPBIL job) if there are outstanding membership transactions that have not been included in a previous premium billing run.

To run Subscriber Update without running GPBIL, select Run Job (CTRL+J) from the File Menu.

Status

D

Current job status.

System generated

Values are:

New: The job has been saved, but has not yet run.

Successful Run: The job has run without errors.

Error: The job has run, but due to error(s) no records were updated.

Cancelled: The job was cancelled (see Action above).

Request Type

R

Nature of the job request; there is only one valid value for this field currently.

Selected from predefined list

Hard coded values are:

Immediate: Run the job immediately after File-Run Job is selected.

Deferred: This choice is not implemented at this time.

Comments

O

Description or comments about the job.

Manually entered
