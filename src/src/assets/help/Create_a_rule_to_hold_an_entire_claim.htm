Create a rule to hold an entire claim

Restrictions

Instructions

Use the following procedure to create a rule to hold an entire claim. The system administrator or another person with administrator permissions normally defines rules. When the data on a claim header satisfies a CLHLD Hold rule, DIAMOND holds the entire claim so that Accounts Payable can select no claim detail line items for payment.

Requirements

None.

Instructions

Step

Comment

1. Determine the conditions under which you want to hold entire claims based on information related to claim header data.

Claims held by CLHLD rules require the hold to be released manually.

2. In keyword CLHLD, enter all required data and save the rule.

CLHLD allows you to use any claim header field (except date fields) as a rule determinant, for example, Vendor ID.

3. Enter institutional, professional services, and dental claims online, via Claims Batch or via Claims API.

DIAMOND automatically applies user-defined claim header Hold rules upon the initial save of a claim header. You can also manually apply these rules in keyword INCLM/PSCLM/DNCLM using the Re-apply Claim Holds command on the Special menu.

Newly entered or changed CLHLD header Hold rules will not be applied to existing claims. Existing claims, including claims currently on hold, will not automatically be subject to the new or revised CLHLD header hold rules unless you use the INCLM/PSCLM/DNCLM Special menu command Re-apply Claim Holds.
