Restrictions

You should run the Provider Enrollment Utility when release R5.1 is installed. Do not run the utility during Member or Claims Batch processing.

Instructions – Creating Enrollment Records Using the Enrollment Utility

Step

Comment

1. Preparation for activating provider/member enrollment functionality for the first time using the Provider/Member Enrollment Utility (see Provider/Member Enrollment user documentation):

  • Install DIAMOND Release 5.1.

  • Activate the PROVENROL parameter (see PARAM Reference documentation).

  • Run the Provider/Member Enrollment Utility scripts as outlined in D950C/S R5.1 Software Installation Guide.

As a result of running the prove_create_util.sh script, you:

  • Select the enrollment rule record creation criteria when prompted by DIAMOND.

And, DIAMOND:

  • Creates a generic PROVE record, based on the PROVE record creation criteria selected, for each PCP provider (PROVC > PCP Flag check box must be checked).

As a result of running the mem_enroll_util.sh script, DIAMOND:

  • Associates the record with the provider’s primary address (see Provider/Member Enrollment user documentation, section Provider/Member Enrollment Utility > PROVE Record Creation).

  • Associates existing, eligible members with their assigned PCP and corresponding criteria based PROVE record.

  • Increments the associated provider’s enrollment count (PROVE > Current Enrollment) in all appropriate PROVE records.

During online member enrollment, when you select a member’s PCP (MEMBR > PCP ID), DIAMOND evaluates the associated provider enrollment rule(s) before assigning the PCP to the member. DIAMOND updates the associated PROVE record enrollment count when the PCP is assigned to the member.

During Member Batch processing, DIAMOND evaluates the provider’s PROVE rule based on the provider’s primary address.

2. After running the Provider/Member Enrollment Utility and as required for each provider, use PROVE to:

  • Create additional provider contract/LOB/IPA/panel/provider address enrollment rule records for existing providers.

  • Create new provider contract/LOB/IPA/panel/provider address records for newly added providers.

  • Change override edits using Special menu > Override Enrollment Edits.

  • Change the current enrollment count using the Special menu > Current Member Enrollment.

  • Change Enrollment Utility created (generic) enrollment rule criteria using the Special menu > Modify Enrollment Criteria. For example, clearing the Accepts New Patients check box to indicate that the provider does not accept new patients and/or changing the Gender from Both to Male.

  • Terminate or unterminate an enrollment rule record using Special menu > Term/Unterminate Provider Enrollment.

You use MEMBR, Special menu – Provider Change to reassign existing members (previously associated with a different PROVE record) to new provider or additional provider address PROVE records.

The provider’s PROVC > PCP Flag check box must be selected on each contract record before you create an associated PROVE record.

You must have access to the Modify Enrollment Criteria and the Current Member Enrollment commands (see SFUNC Reference documentation). You use these commands to change/correct the Enrollment Utility created PROVE generic records. In the future, if a provider’s enrollment rule changes (based on the provider’s contract record), you should terminate the existing PROVE record and create a new PROVE record based on the new criteria.

 

 

Instructions – Creating Enrollment Records Manually

Step

Comment

1. Install DIAMOND Release 5.1

See R5.1 Software Installation Guide.

2. Activate the PROVENROL parameter (see PARAM reference documentation).

When R5.1 is installed, the PROVENROL parameter defaults to No = not activated.

3. Use PROVE to manually create enrollment rules, one-by-one, on each valid PCP/provider contract/LOB/IPA/panel/provider address basis.

The provider’s PROVC > PCP Flag check box must be selected on each contract record before you create an associated PROVE record.

As this is a manual process, the initial PROVE > Current Enrollment count should be set by the user using the Special menu – Current Member Enrollment.

During online member enrollment, when you select a member’s PCP (MEMBR > PCP ID), DIAMOND evaluates the associated provider enrollment rule(s) before assigning the PCP to the member. DIAMOND updates the associated PROVE record enrollment count when the PCP is assigned to the member.

During Member Batch processing, DIAMOND evaluates the provider’s PROVE rule based on the provider’s primary address.

4. After you create all of the initial enrollment rules manually, continue to use PROVE to:

  • Create additional provider contract/LOB/IPA/panel/provider address enrollment rule records for existing providers.

  • Create new provider contract/LOB/IPA/panel/provider address records for newly added providers.

  • Change override edits using Special menu > Override Enrollment Edits.

  • Change the current enrollment count using the Special menu > Current Member Enrollment.

  • Correct enrollment rule criteria using the Special menu > Modify Enrollment Criteria. For example, un-checking the Accepts New Patients check box to indicate that the provider does not accept new patients and/or changing the Gender from Both to Male.

  • Terminate or unterminate an enrollment rule record using Special menu > Term/Unterminate Provider Enrollment.

You use MEMBR, Special menu > Provider Change to reassign existing members (previously associated with a different PROVE record) to new provider or additional provider address PROVE records.

The provider’s PROVC > PCP Flag check box must be selected on each contract record before you create an associated PROVE record.

You must have access to the Modify Enrollment Criteria and the Current Member Enrollment commands (see SFUNC Reference documentation). You use these commands to change/correct the Enrollment Utility created PROVE generic records. In the future, if a provider’s enrollment rule changes (based on the provider’s contract record), you should terminate the existing PROVE record and create a new PROVE record based on the new criteria.
