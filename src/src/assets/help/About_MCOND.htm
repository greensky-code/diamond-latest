About MCOND

Use keyword MCOND (Member Conditions) to add or change member condition records for individual members. Member condition records include such criteria as a condition type and a range of diagnoses. These criteria reflect a specific member condition.

DIAMOND logic supports two types of member conditions:

  • Privacy - a keyword MCOND (Member Conditions) member Condition Type of Privacy represents "privacy by diagnosis" functionality

  • Pre-Existing Condition - a keyword MCOND member Condition Type of Pre-Existing, Rated-up, or Ridered-out represents "pre-existing condition" functionality

Each type of member condition enables you to trigger special processing logic when the data on a claim or authorization satisfy the criteria on a member condition record. For example, when the data on a claim header satisfies a member condition record of condition type Privacy, the claim header Privacy Applies box is set to Y .
