PCP Auto Assignment Examples

DIAMOND performs PCP Auto Assignment when you run a PCPJB (PCP Job Setup) batch job. Job particulars you must specify include:

  • Lines of business included

  • Date range of member eligibility record insert dates included

  • Member eligibility record insert processes (batch, online) included

The following sections detail PCP Auto Assignment rules and processing logic.

PCP Auto Assignment Rules

PCP Auto Assignment makes and validates PCP assignments according to sets of rules you define. You define individual PCPAA rules once and then combine them into sets with appropriate values for the applicable lines of business. PCPAA is a supplement to the existing Provider Enrollment Rules (PROVE) functionality.

PCP Auto Assignment Rule Types are summarized in the following table.

PCP Auto Assignment Rule Type

Use

10 = Preference Rule

Use to most specifically define a subset of preferred potential PCP providers that meet the criteria for the selected member eligibility record. Used when either:

  • No PCP is submitted

  • Submitted PCP does not pass the Basic Screening rules

  • Submitted PCP is the Default PCP identified on the Member’s line of business record

Providers must also pass Auto Assignment rules to be eligible for PCP assignment to a member eligibility record.

20 = Auto Assignment Rule

Use to identify a subset of potential PCP providers that meet the criteria for the selected member eligibility record when either:

  • No PCP is submitted

  • Submitted PCP does not pass the Basic Screening rules

  • Submitted PCP is the Default PCP identified on the Member’s line of business record

Preference Rules, if any, apply before Auto Assignment Rules.

30 = Basic Screening Rule

Use to validate a submitted or requested PCP selection by a member. Basic Screening Rules are generally not as specific or limiting as Auto Assignment Rules since a typical business approach is to allow the Member to be assigned to their chosen PCP whenever possible.

90 = Tie Breaker Rule

Use when more than one PCP provider qualifies as the PCP for a selected member record. A Tie Breaker Rule dictates the method of selecting between multiple providers that meet all applicable PCP Auto Assignment criteria.

 

PCP Auto Assignment rules use fields from the following screens as attributes:

  • PROVE  (Provider Enrollment Rules)

  • PROVF  (Provider Master)

  • MEMBR  (Member Master)

  • MELIG  (Member Eligibility Record)

  • LINBS  (Line of Business)

Available attributes depend on the Screen and Rule Type entered. The following table summarizes the available PCP Auto Assignment rule criteria and describes how DIAMOND uses each.

Proximity (Address/Zip Code)

All but Rule Type 90

The distance between the member’s MEMBR address and the provider’s PROVE address must be less than or equal to the Urban To Miles or Rural To Miles on the rule, whichever is applicable.

DIAMOND uses a third party tool to determine the longitude and latitude of the addresses. DIAMOND then calculates the distances in miles. To determine whether to use the value in Urban To Miles or Rural To Miles, DIAMOND uses the ZIPRF Zip Code Qualifier field value (Urban or Rural) for the member’s ZIP code. If the member’s ZIP code is not available, DIAMOND uses the Urban To Miles value.

If you select this attribute when entering a rule in PCPRL, you must specify an Urban To Miles and Rural To Miles.

See Example of PCPRL Rule Attribute Proximity for an example of how this rule is used.

Provider’s PROVF Language 1, Language 2, or Language 3 must match Member’s MEMBR Language (Lang) to satisfy this rule.

MELIG

Reinstated Member

All but Rule Type 90

Selects the PCP from the member’s latest prior eligibility record for possible auto assignment to the current selected member eligibility record.

MEMBR

Member with a DIAMOND ID

All but Rule Type 90

Uses DIAMOND ID to locate other member master records for the member (that is, with the same DIAMOND ID but a different Subscriber ID). If found, selects the PCP from this other member record’s most recent eligibility record for possible auto assignment to the current selected member eligibility record.

MEMBR

Member with a Family Affiliation

All but Rule Type 90

For the member’s line of business, uses the LINBS Family Affiliation field value to locate all family members. If one or more family members are found, locates the most recent eligibility record. If the most recent eligibility record is effective for the member’s current selected eligibility period, selects the PCP from this family member’s eligibility record for possible auto assignment to the current selected member eligibility record.

The Family Affiliation indicator can have one of the following two values, which are used to locate family member eligibility records as follows:

  • Subscriber ID - Uses the Seq_subscriber_ID to locate other eligibility records using the same Seq_subs_ID, excluding records where the seq_membr_id is equal to the incoming member’s seq_membr_id.

  • User Defined field in MEMBR - Uses the specified MEMBR Master User Defined field to find any other family members with the same value in the same user defined field.

LINBS (PCPSI)

Excl Selected Provider Specialties for the specified age

All but Rule Type 90

Excludes from PCP Auto Assignment the primary provider specialties by age as specified in the Specialty Restrictions by Age section box of the PCPSI record for the member’s line of business. That is, if a member’s age falls in an age range defining an excluded specialty, a provider with that excluded primary specialty cannot be assigned to the member.

Warning: If this attribute is selected for a rule and the member’s LINBS record has no value established for this on PCPSI, all providers meet the criteria.

LINBS (PCPSI)

Excl Selected Provider Specialties for the specified gender

All but Rule Type 90

Excludes from PCP Auto Assignment the primary provider specialties by gender as specified in the Specialty Restrictions by Gender section box of the PCPSI record for the member’s line of business. That is, if a member’s gender corresponds to a PCPSI record defining an excluded specialty, a provider with that excluded primary specialty cannot be assigned to the member.

Warning: If this attribute is selected for a rule and the member’s LINBS record has no value established for this on PCPSI, all providers meet the criteria.

LINBS (PCPSI)

Use the Ped Specialty Indicator

All but Rule Type 90

Use to specify an age range defining a child for the purposes of assignment to a PCP with a pediatric specialty. Only one age range should be specified, with one or more specialties.

For a specified member age range, restricts PCP Auto Assignment to a provider with a primary pediatric specialty. Uses the age range/specialty requirements specified in the Pediatric Indicator section box of the PCPSI record for the member’s line of business.

That is, if a member’s age falls in the age range specified in the Pediatric Indicator section box, the member can only be assigned to a PCP with a primary specialty matching one of the pediatric specialties defined for the age range.

Warning: If this attribute is selected for a rule and the member’s LINBS record has no value established for this on PCPSI, all providers meet the criteria.

LINBS (PCPSI)

Use the OB/GYN Specialty Indicator

All but Rule Type 90

Use to specify an age range for female members that is appropriate for assignment to a PCP with a specified OB/GYN specialty. Only one age range should be specified, with one or more specialties.

For a specified member age range, restricts PCP Auto Assignment to a provider with a primary indicated specialty. Uses the age range/specialty requirements specified in the OB/GYN Indicator section box of the PCPSI record for the member’s line of business.

That is, if a member’s age falls in the age range specified in the OB/GYN Indicator section box, the member can only be assigned a PCP with a primary specialty matching one of the indicated specialties defined for that age range.

Warning: If no entries exist in the OB/GYN Indicator section box of the PCPSI record for the member’s line of business and this attribute is selected in a rule, no providers will meet the specified criteria.

PROVE

Closest Proximity (Address/Zip Code)

Only available for Rule Type 90, Tie Breaker

Uses proximity as a tiebreaker if more than one provider satisfies the rules for PCP Auto Assignment to a member. Proximity is the distance between the member’s MEMBR address and each provider’s PROVE address.

DIAMOND uses a third party tool to determine longitude and latitude of the addresses, and then calculates the distances in miles. DIAMOND then assigns the provider that is closest to the Member as the PCP. If more than one PCP provider has the same distance, DIAMOND evaluates any other Tie Breaker rules. If no further Tie Breaker rules remain to be evaluated, DIAMOND uses an Oracle random selection routine to make the assignment.

PROVE

Least Recent Assigned to a Member

Only available for Rule Type 90, Tie Breaker

Uses least recent PCPAA assignment as a tiebreaker if more than one provider satisfies the rules for PCP Auto Assignment to a member.

Determines which provider has a PROVE record with the most recent PCPAA assignment. If two or more providers have the same Least Recent Assigned Date to a Member, DIAMOND evaluates any other Tie Breaker rules. If no further Tie Breaker rules remain to be evaluated, DIAMOND uses an Oracle random selection routine to make the assignment.

Fixed PCP Auto Assignment Criteria

In addition to the user-defined rules defined above, PCP Auto Assignment logic always applies certain fixed key criteria. To be eligible for PCP assignment, a provider must satisfy the following fixed criteria:

  • Have a PROVC record with the PCP Flag selected

  • Have a PROVE record effective for the member eligibility record Effective date

  • This matching PROVE record must have a LINBS+IPA+PANEL combination that matches the member eligibility record

  • This matching PROVE record must have the Available for Auto Assign check box selected

PCP Auto Assignment Restrictions

  • PCP Auto Assignment makes no changes to eligibility dates on member eligibility records.

  • DIAMOND uses the enrollment threshold percentage (LINBS Ind PCP/LINBS Threshold) only in PCP Auto Assignment. This enrollment threshold percentage is not otherwise enforced at the PROVE, PROVF or LINBS levels.

  • You can use the current PROVE Special menu command Override Enrollment Edits to override the enrollment limit maximum for an individual PROVE record. However, this manual override enrollment limit command is not available for the PROVF and LINBS enrollment limits and PCP Auto Assignment.

  • When the Member’s LINBS requires a PCP and the submitted batch or API member eligibility record does not include a submitted or valid PCP, DIAMOND generates a critical error. DIAMOND does not automatically use the member’s LINBS Default PCP in such instances. You must resubmit such a record using the member’s LINBS Default PCP in order to add the record to DIAMOND. You can then run a PCP Auto Assignment batch job to automatically assign a PCP.

  • The Address and Zip Code fields on Member and Provider are not required fields in DIAMOND. However, to use the PROVE Proximity or Closest Proximity attributes in the PCP Auto Assignment process, you must enter valid Zip codes for both the Member and Provider records.

  • It is recommended that all Member batch and API transactions be completed (no longer running) when a PCPAA batch job is run.

  • PCP Auto Assignment does not include the MEMBR Prov2 (Provider 2) field.

  • When you use PCP Auto Assignment with member Site Validation and the member's line of business requires Site, you must use keyword MELIG (Member Eligibility History) to add Site to the member eligibility records assigned a PCP. For more information, see Specifying a Site on an existing member eligibility record.
